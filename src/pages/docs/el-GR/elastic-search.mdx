## <h2 hidden>Τι είναι η Ελαστική Αναζήτηση?</h2>

Η ελαστική αναζήτηση είναι μια διανεμημένη, RESTful μηχανή αναζήτησης και ανάλυσης που χρησιμοποιείται συνήθως για τη ευρετηρίαση και την αναζήτηση μεγάλων όγκων δεδομένων. Στην Illa, μπορείτε να χρησιμοποιήσετε την πηγή δεδομένων Elasticsearch για να συνδεθείτε σε εμφανίσεις Elasticsearch και να εκτελέσετε ερωτήματα και συγκεντρώσεις στα δεδομένα σας.

### Δημιουργία Ελαστικής Αναζήτησης

Υπάρχουν δύο τρόποι για να δημιουργήσετε έναν πόρο στην Illa μετά την είσοδό σας στο λογαριασμό Illa.

-   Δημιουργία σε πόρους

Συνδεθείτε στον Illa λογαριασμό σας, επιλέξτε `**Πόροι**` στην κορυφή της σελίδας και κάντε κλικ στο κουμπί `**Δημιουργία Νέου**`.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Επιλέξτε **`Elastic Search`** από τη λίστα της βάσης δεδομένων.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Συνδεθείτε στη βάση δεδομένων με τις απαιτούμενες παραμέτρους που περιγράφονται στις παρακάτω ρυθμίσεις σύνδεσης.

Κάντε κλικ στο κουμπί `**Δοκιμή Σύνδεσης**` για να δείτε αν μπορούμε να συνδεθείτε με επιτυχία στη βάση δεδομένων. Εάν ναι, κάντε κλικ στο **`Αποθήκευση πόρων`**, αλλιώς, ελέγξτε ξανά το όνομα κεντρικού υπολογιστή, τη θύρα, το όνομα χρήστη και τον κωδικό πρόσβασης είναι σωστά.

Μετά τη δημιουργία ενός πόρου, η έτοιμη Ελαστική Αναζήτηση θα εμφανίζεται όπως εμφανίζεται.

![ela_ex_ready](https://cdn.illacloud.com/official-website/img/docs/ela_ex_ready.png)

-   Δημιουργία στο Builder

Συνδεθείτε στον Illa λογαριασμό σας, δημιουργήστε ένα έργο στο Illa Builder στη σελίδα `**App**`, και πλοηγηθείτε στο **`Λίστα Ενεργειών`** στο κάτω μέρος της σελίδας. Κάντε κλικ στο κουμπί `**new**`, στη συνέχεια επιλέξτε **`Ελαστική Αναζήτηση`** από τη λίστα της βάσης δεδομένων. Στη συνέχεια, συνδεθείτε στη βάση δεδομένων με τις απαιτούμενες παραμέτρους που περιγράφονται στις "Ρυθμίσεις Σύνδεσης" παρακάτω.

Κάντε κλικ στο κουμπί `**Δοκιμή Σύνδεσης**` για να δείτε αν μπορούμε να συνδεθείτε με επιτυχία στη βάση δεδομένων. Εάν ναι, κάντε κλικ στο **`Αποθήκευση πόρων`**, αλλιώς, ελέγξτε ξανά το όνομα κεντρικού υπολογιστή, τη θύρα, το όνομα χρήστη και τον κωδικό πρόσβασης είναι σωστά.

| Ιδιότητες         | Απαιτείται  | Περιγραφή                                                                                                                   |

| Όνομα             | απαιτείται  | Το όνομα του πόρου κατά τη δημιουργία ενεργειών στο ILLA.                                                                   |
| Hostname          | απαιτείται  | Η διεύθυνση URL ή IP για τη βάση δεδομένων σας                                                                              |
| Θύρα              | απαιτείται  | Ο αριθμός θύρας του διακομιστή που θα πρέπει να χρησιμοποιήσετε για να συνδεθείτε.                                          |
| Βάση Δεδομένων    | απαιτείται  | Το όνομα της βάσης δεδομένων                                                                                                |
| Όνομα Χρήστη      | απαιτείται  | το όνομα χρήστη που θέλετε να χρησιμοποιήσετε κατά τη σύνδεση στον εξυπηρετητή Ελαστικής Αναζήτησης.                        |
| Κωδικός           | απαιτείται  | Χρησιμοποιήστε αυτόν τον κωδικό πρόσβασης για ταυτοποίηση.                                                                  |
| Επιλογές SSL      | προαιρετικό | αποφασίζει πόσο υψηλή προτεραιότητα έχει μια ασφαλής σύνδεση SSL TCP/IP κατά τη διαπραγμάτευση με το διακομιστή.            |

### Δημιουργία Ενεργειών

Έχουμε δημιουργήσει έναν Ελαστικό πόρο Αναζήτησης, μπορούμε να προσθέσουμε την ενέργεια επιλέγοντας Ελαστική Αναζήτηση από λίστα ενεργειών και επιλέγοντας το κουμπί **`Δημιουργία δράσης`**.

![ela_resource_list](https://cdn.illacloud.com/official-website/img/docs/ela_resource_list.png)

Τώρα έχουμε προσθέσει τον Elastic Search server ως μια ενέργεια στη σελίδα μας κτίριο.

![ela](https://cdn.illacloud.com/official-website/img/docs/ela.png)

### Ρύθμιση Ελαστικής Αναζήτησης

#### Επισκόπηση

| Όνομα Μεθόδου   | Περιγραφή                                                                                        |
| --------------- | ------------------------------------------------------------------------------------------------ |
| Τύπος Ενέργειας | αναζήτηση, λήψη εγγράφου, εισαγωγή εγγράφου, ενημέρωση εγγράφου, διαγραφή εγγράφου               |
| Μετασχηματιστής | μετατρέποντας τα δεδομένα σε στυλ που σας αρέσει χρησιμοποιώντας JavaScript                      |

#### εισαγωγή ενός εγγράφου

αποστολή ενός αιτήματος HTTP PUT στο σύμπλεγμα Elasticsearch με ένα έγγραφο JSON που περιέχει τα δεδομένα που θα εισαχθούν.

** Input **

| Ιδιότητες    | Απαιτείται  | Περιγραφή                                                              |
| ------------ | ----------- | ---------------------------------------------------------------------- |
| Σώμα         | προαιρετικό | ένα έγγραφο JSON που περιέχει τα δεδομένα που πρόκειται να εισαχθούν   |
| Δείκτης      | προαιρετικό | όνομα του ευρετηρίου όπου πρέπει να εισαχθεί το έγγραφο                |

Για παράδειγμα, για να εισάγετε `value1`, `value2`, και `value3` στο **`field1`**, **`field2`**, και **`field3`** στη βάση δεδομένων Elastic αναζήτησης, βάλτε τον κώδικα snippet κάτω στο `**Body`** και `insert2` για **`Index`\*\*.

```jsx
{
	"field1": "value1",
	"field2": "value2",
	"field3": "value3"
}
```

** Έξοδος **

Μήνυμα επιτυχίας του κατά πόσον το έγγραφο εισάγεται επιτυχώς και πληροφορίες του εισαχθέντος εγγράφου.

![ela_insert_data](https://cdn.illacloud.com/official-website/img/official-site/ela_insert_data.png)

Χρησιμοποίησε το `{{insert.data}}` για να πάρεις το μήνυμα.

![ela_insert_code](https://cdn.illacloud.com/official-website/img/official-site/ela_insert_code.png)

#### ενημέρωση ενός εγγράφου

στέλνοντας ένα αίτημα HTTP **`POST`** ή **`PUT`** στο σύμπλεγμα Elasticsearch με το αναγνωριστικό του εγγράφου που θα πρέπει να ενημερωθεί, και ένα έγγραφο JSON που καθορίζει τις αλλαγές που πρέπει να γίνουν.

** Input **

| Ιδιότητες    | Απαιτείται  | Περιγραφή                                             |
| ------------ | ----------- | ----------------------------------------------------- |
| Σώμα         | προαιρετικό | Η νέα τιμή που θα αντιστοιχιστεί στο πεδίο.           |
| Δείκτης      | προαιρετικό | Το όνομα του ευρετηρίου όπου αποθηκεύεται το έγγραφο. |
| Id           | προαιρετικό | Το μοναδικό αναγνωριστικό για το έγγραφο.             |

Για παράδειγμα, για να ενημερώσετε τα αρχεία `value1`, `value2`, και `value3` για **`field1`**, **`field2`**, και **`field3`** στη βάση δεδομένων Elastic αναζήτησης, βάλτε τον παρακάτω κώδικα στο **`Body,`** `insert2` για **`Index`** και αντιγράψτε το έγγραφο **`Id`** από το μήνυμα εισαγωγής.

```jsx
{"doc":{
	"field1": "value11",
	"field2": "value2",
	"field3": "value3"
}}
```

** Έξοδος **

Μήνυμα επιτυχίας του κατά πόσον το έγγραφο ενημερώνεται με επιτυχία και πληροφορίες του επικαιροποιημένου εγγράφου.

![ela_update_data](https://cdn.illacloud.com/official-website/img/official-site/ela_update_data.jpg)

Χρησιμοποίησε το `{{update.data}}` για να πάρεις το μήνυμα.

![ela_update_code](https://cdn.illacloud.com/official-website/img/official-site/ela_update_code.jpg)

#### Αναζήτηση

αναζήτηση και ανάκτηση εγγράφων από ένα ευρετήριο

** Input **

| Ιδιότητες    | Απαιτείται  | Περιγραφή                                                                                          |
| ------------ | ----------- | -------------------------------------------------------------------------------------------------- |
| Ερώτημα      | προαιρετικό | καθορίστε το ερώτημα αναζήτησης που χρησιμοποιείται για την αντιστοίχιση εγγράφων σε ένα ευρετήριο |
| Δείκτης      | προαιρετικό | προσδιορίζει το όνομα του ευρετηρίου ή των δεικτών που θα πρέπει να αναζητηθούν                    |

Για παράδειγμα, για να αναζητήσετε τη βάση δεδομένων για να βρείτε έγγραφα με ζεύγος κλειδιών `"field1": "value1"`, βάλτε το κομμάτι κώδικα παρακάτω στο **`Query` .**

```jsx
{
	"query": {
		"match": {
			"field1": "value1"
		}
	}
}
```

** Έξοδος **

Μήνυμα επιτυχίας με τον αριθμό των hits/documents που βρίσκει ότι ταιριάζει με το ερώτημα και τις πληροφορίες για κάθε χτύπημα.

![ela_search_data](https://cdn.illacloud.com/official-website/img/official-site/ela_search_data.jpg)

Χρησιμοποίησε το `{{search.data}}` για να πάρεις το μήνυμα.

![ela_search_code](https://cdn.illacloud.com/official-website/img/official-site/ela_search_code.jpg)

#### πάρε ένα έγγραφο

αποστολή ενός αιτήματος HTTP **`GET`** στο σύμπλεγμα Elasticsearch με το αναγνωριστικό του εγγράφου που θα πρέπει να ανακτηθεί.

** Input **

| Ιδιότητες    | Απαιτείται  | Περιγραφή                                             |
| ------------ | ----------- | ----------------------------------------------------- |
| Δείκτης      | προαιρετικό | Το όνομα του ευρετηρίου όπου αποθηκεύεται το έγγραφο. |
| Id           | προαιρετικό | Το μοναδικό αναγνωριστικό για το έγγραφο.             |

Για παράδειγμα, για να λάβετε τις πληροφορίες του εγγράφου που μόλις ενημερώσαμε, έβαλε το `insert2` για το **`Index`**, και αντιγράψτε το έγγραφο **`Id`** από το ενημερωμένο μήνυμα.

** Έξοδος **

επιστρέφει το έγγραφο με το καθορισμένο αναγνωριστικό ή ένα **`404 Δεν βρέθηκε`** σφάλμα εάν το έγγραφο δεν υπάρχει.

![ela_get_data](https://cdn.illacloud.com/official-website/img/official-site/ela_get_data.jpg)

Χρησιμοποίησε το `{{get.data}}` για να πάρεις το μήνυμα.

![ela_get_code](https://cdn.illacloud.com/official-website/img/official-site/ela_get_code.jpg)

#### διαγραφή ενός εγγράφου

αποστολή ενός αιτήματος HTTP DELETE στο σύμπλεγμα Elasticsearch με το αναγνωριστικό του εγγράφου που θα πρέπει να διαγραφεί.

** Input **

| Ιδιότητες    | Απαιτείται  | Περιγραφή                                             |
| ------------ | ----------- | ----------------------------------------------------- |
| Δείκτης      | προαιρετικό | Το όνομα του ευρετηρίου όπου αποθηκεύεται το έγγραφο. |
| Id           | προαιρετικό | Το μοναδικό αναγνωριστικό για το έγγραφο.             |

Για παράδειγμα, για να διαγράψετε έγγραφο, βάλτε `insert2` για **`Index`,** και αντιγράψτε επικολλήστε το έγγραφο **`Id`** από το ενημερωμένο μήνυμα.

** Έξοδος **

επιστρέφει το έγγραφο με το καθορισμένο αναγνωριστικό ή ένα **`404 Δεν βρέθηκε`** σφάλμα εάν το έγγραφο δεν υπάρχει.

![ela_delete_data](https://cdn.illacloud.com/official-website/img/official-site/ela_delete_data.jpg)
