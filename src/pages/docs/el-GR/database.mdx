Η ILLA συνδέεται με τις περισσότερες βάσεις δεδομένων, συμπεριλαμβανομένων των PostgreSQL, MySQL, TiDB, και MongoDB. Παρακαλώ ελέγξτε αυτή τη λίστα εδώ για τις πηγές δεδομένων που υποστηρίζει αυτή τη στιγμή η ILLA. Οι επερχόμενες πηγές δεδομένων που σχεδιάζουμε να υποστηρίξουμε παρατίθενται εδώ. Εάν το ILLA λείπει ορισμένοι πόροι δεδομένων που χρησιμοποιείτε, παρακαλώ συμπληρώστε τη φόρμα εδώ.
Οι λεπτομέρειες σύνδεσης που απαιτούνται από την ILLA εξαρτώνται από τον τύπο βάσης δεδομένων που θέλετε να συνδεθείτε. Αυτές οι πληροφορίες περιλαμβάνουν γενικά μια υποδοχή, θύρα, διαπιστευτήρια χρήστη και συμβολοσειρά σύνδεσης. Παρακαλούμε ανατρέξτε στη διαμόρφωση ή τεκμηρίωση της βάσης δεδομένων σας για να το πάρετε αυτό.

1.  Δημιουργήστε έναν πόρο βάσης δεδομένων
    Οι πόροι αντιπροσωπεύουν συνδέσεις με πηγές δεδομένων και είναι αυτό που θέτετε στις εφαρμογές ILLA σας. Συνδεθείτε στο ILLA και μεταβείτε στην καρτέλα Πόροι > κάντε κλικ στην επιλογή Δημιουργία νέας βάσης δεδομένων στην οποία θέλετε να συνδεθείτε. Ανοίγει τη ρύθμιση παραμέτρων ενός τρόπου πόρων όπου προσθέτετε πληροφορίες σύνδεσης.

Αφού εισάγετε τα στοιχεία σύνδεσής σας, κάντε κλικ στο στοιχείο Δοκιμή σύνδεσης για να ελέγξετε ότι η ILLA μπορεί να συνδεθεί επιτυχώς. Μόλις ολοκληρωθεί, κάντε κλικ στην επιλογή Αποθήκευση πόρου για να αποθηκεύσετε τον πόρο. 
Επιπλέον, μπορείτε να δημιουργήσετε πόρους στη λίστα ενεργειών κάτω από τον επεξεργαστή εφαρμογών. 
2\. Ερώτηση της βάσης δεδομένων σας
Στον επεξεργαστή εφαρμογών, κάντε κλικ στο κουμπί + Νέα στη Λίστα Ενεργειών> επιλέξτε τύπο πόρου και επιλέξτε έναν υπάρχοντα πόρο ή δημιουργήστε έναν νέο πόρο > κάντε κλικ στην επιλογή Δημιουργία ενέργειας και ενημερώστε τις ενέργειες για να ανακτήσετε κάποιες πληροφορίες από έναν πίνακα στη βάση δεδομένων σας.

Κάντε κλικ στο κουμπί Αποθήκευση και, στη συνέχεια, κάντε κλικ στο κουμπί Εκτέλεση. Τα αποτελέσματα του ερωτήματος εμφανίζονται αυτόματα μετά την ολοκλήρωση του ερωτήματος. 
Υπάρχουν κάποιες προηγμένες ρυθμίσεις, συμπεριλαμβανομένων της μετονομασίας, των ρυθμίσεων εκτέλεσης, της μετατροπής δεδομένων, και της ρύθμισης του χειρισμού της επιτυχίας ή αποτυχίας εκτέλεσης, όπως φαίνεται παρακάτω.
