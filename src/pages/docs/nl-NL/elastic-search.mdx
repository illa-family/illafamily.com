## <h2 hidden>Wat is Elastic Search?</h2>

Elastic search is een verspreide, RESTful search en analytics engine die vaak wordt gebruikt om te indexeren en te zoeken naar grote hoeveelheden gegevens. In Illa kunt u de data bron van Elasticsearch gebruiken om verbinding te maken met Elasticsearch instanties en query's en aggregaties op uw gegevens uitvoeren.

### Elastisch zoeken maken

Er zijn twee manieren om een bron in Illa te maken nadat je bent ingelogd op je Illa account.

-   Maak in Bronnen

Log in op je Illa account, selecteer `**Resources**` bovenaan de pagina, en klik op de `**Maak Nieuw**` knop.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Selecteer **`Elastic Search`** uit de database lijst.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Maak verbinding met de database met de vereiste parameters beschreven in `Connectie Instellingen` hieronder.

Klik op `**Test Connection**` om te zien of we met succes verbinding kunnen maken met de database. Zo ja, klik **`Resources`**, anders controleer je de hostnaam, port, gebruikersnaam en wachtwoord correct.

Na het maken van een grondstof, wordt het Elastisch Zoeken weergegeven zoals getoond.

![ela_ex_ready](https://cdn.illacloud.com/official-website/img/docs/ela_ex_ready.png)

-   Maak in Bouwer

Log in op je Illa account, maak een project aan in Illa Builder in de `**App**` pagina en navigeer naar de **`Action List`** onderaan de pagina. Klik `**nieuw**`, en selecteer dan **`Elastic Search`** uit de database lijst. Maak vervolgens verbinding met de database met de vereiste parameters beschreven in `Connectie Instellingen` hieronder.

Klik op `**Test Connection**` om te zien of we met succes verbinding kunnen maken met de database. Zo ja, klik **`Resources`**, anders controleer je de hostnaam, port, gebruikersnaam en wachtwoord correct.

### Verbindingsinstellingen

Hier moeten we informatie verstrekken om verbinding te maken met de Elastische zoekdatabase.

![ela_config](https://cdn.illacloud.com/official-website/img/docs/ela_config.png)

| Eigenschappen     | vereist   | Beschrijving                                                                                                 |
| ----------------- | --------- | ------------------------------------------------------------------------------------------------------------ |
| naam              | verplicht | De naam voor resource bij het maken van acties in de ILLA.                                                   |
| Hostname          | verplicht | De URL of het IP-adres van uw database                                                                       |
| Poort             | verplicht | Het poortnummer van de server host dat u moet gebruiken om te verbinden.                                     |
| Database          | verplicht | De naam van de database                                                                                      |
| Gebruikersnaam    | verplicht | de gebruikersnaam die u wilt gebruiken bij het inloggen op de Elastic Search server.                         |
| Wachtwoord        | verplicht | Gebruik dit wachtwoord voor verificatie.                                                                     |
| SSL opties        | optioneel | Bepaal hoe hoog een veilige SSL TCP/IP verbinding prioriteit heeft tijdens het onderhandelen met de server.  |

### Configureer Elastisch zoeken

#### Overzicht.

| Methode Naam  | Beschrijving                                                                                          |
| ------------- | ----------------------------------------------------------------------------------------------------- |
| Actie Type    | zoeken, een document ophalen, een document invoegen, document bijwerken, verwijderen van een document |
| Transformator | data omzetten naar de stijl die je leuk vindt met behulp van JavaScript                               |

#### document invoegen

stuurt een HTTP-PUT-verzoek naar de Elasticsearch-cluster met een JSON-document dat de gegevens bevat die moeten worden toegevoegd.

** Input **

| Eigenschappen  | vereist   | Beschrijving                                                          |
| -------------- | --------- | --------------------------------------------------------------------- |
| Lichaam        | optioneel | een JSON-document dat de gegevens bevat die moeten worden ingevoegd   |
| Indexeren      | optioneel | naam van de index waar het document moet worden ingevoegd             |

Bijvoorbeeld, om `value1` in te voegen, `value2`, en `value3` in **`field1`**, **`field2`**, en **`field3`** in onze Elastic search database, zet de code snippet hieronder in de `**Body`** en `insert2` voor **`Index`\*\*.

```jsx
{
	"field1": "value1",
	"field2": "value2",
	"field3": "value3"
}
```

** Uitvoer **

Succes bericht of het document met succes is ingevoegd en informatie van het ingevoegd document.

![ela_insert_data](https://cdn.illacloud.com/official-website/img/official-site/ela_insert_data.png)

Gebruik `{{insert.data}}` om het bericht te krijgen.

![ela_insert_code](https://cdn.illacloud.com/official-website/img/official-site/ela_insert_code.png)

#### document bijwerken

zend een HTTP **`POST`** of **`PUT`** verzoek naar de Elasticsearch cluster met de ID van het document dat moet worden bijgewerkt, en een JSON-document dat aangeeft welke wijzigingen moeten worden aangebracht.

** Input **

| Eigenschappen  | vereist   | Beschrijving                                                       |
| -------------- | --------- | ------------------------------------------------------------------ |
| Lichaam        | optioneel | De nieuwe waarde die toegewezen moet worden aan het veld.          |
| Indexeren      | optioneel | De naam van de index waar het document is opgeslagen.              |
| Id             | optioneel | De unieke id voor het document.                                    |

Bijvoorbeeld, om `value1`, `value2` en `value3` voor **`field1`**, **`field2`**, en **`field3`** bij te werken in onze Elastische zoekdatabase, zet de code snippet hieronder in het **`Body,`** `insert2` voor **`Index`** en kopieer en plak het document **`Id`** in het invoegen bericht.

```jsx
{"doc":{
	"field1": "value11",
	"field2": "value2",
	"field3": "value3"
}}
```

** Uitvoer **

Succes bericht van of het document met succes wordt bijgewerkt en informatie over het bijgewerkte document.

![ela_update_data](https://cdn.illacloud.com/official-website/img/official-site/ela_update_data.jpg)

Gebruik `{{update.data}}` om het bericht te krijgen.

![ela_update_code](https://cdn.illacloud.com/official-website/img/official-site/ela_update_code.jpg)

#### Zoeken

query en documenten ophalen uit een index

** Input **

| Eigenschappen     | vereist   | Beschrijving                                                                   |
| ----------------- | --------- | ------------------------------------------------------------------------------ |
| Zoekopdracht      | optioneel | Geef de zoekquery op die wordt gebruikt om documenten in een index te koppelen |
| Indexeren         | optioneel | Specificeer de naam van de index of indices die gezocht moeten worden          |

Bijvoorbeeld, om te zoeken in de database naar documenten met sleutelwaarde paar `"field1": "value1"`, zet de code snippet hieronder in de **`Query`.**

```jsx
{
	"query": {
		"match": {
			"field1": "value1"
		}
	}
}
```

** Uitvoer **

Succes bericht met het aantal hits/documenten dat gevonden wordt dat overeenkomt met de query en de info voor elke slag.

![ela_search_data](https://cdn.illacloud.com/official-website/img/official-site/ela_search_data.jpg)

Gebruik `{{search.data}}` om het bericht te krijgen.

![ela_search_code](https://cdn.illacloud.com/official-website/img/official-site/ela_search_code.jpg)

#### een document krijgen

stuurt een HTTP **`GET`** verzoek naar de Elasticsearch cluster met het ID van het document dat moet worden opgehaald.

** Input **

| Eigenschappen  | vereist   | Beschrijving                                          |
| -------------- | --------- | ----------------------------------------------------- |
| Indexeren      | optioneel | De naam van de index waar het document is opgeslagen. |
| Id             | optioneel | De unieke id voor het document.                       |

Bijvoorbeeld om de documentinformatie te krijgen voor het document dat we zojuist hebben bijgewerkt, zet `insert2` voor **`Index`**, en kopieer en plak het document **`Id`** in het bijgewerkte bericht.

** Uitvoer **

retourneert het document met de opgegeven ID, of een **`404 Not Found`** fout als het document niet bestaat.

![ela_get_data](https://cdn.illacloud.com/official-website/img/official-site/ela_get_data.jpg)

Gebruik `{{get.data}}` om het bericht te krijgen.

![ela_get_code](https://cdn.illacloud.com/official-website/img/official-site/ela_get_code.jpg)

#### verwijder een document

verzenden van een HTTP DELETE verzoek naar de Elasticsearch cluster met het ID van het document dat moet worden verwijderd.

** Input **

| Eigenschappen  | vereist   | Beschrijving                                          |
| -------------- | --------- | ----------------------------------------------------- |
| Indexeren      | optioneel | De naam van de index waar het document is opgeslagen. |
| Id             | optioneel | De unieke id voor het document.                       |

Om bijvoorbeeld document te verwijderen, plaats `insert2` voor **`Index`,** en kopieer en plak het document **`Id`** van het bijgewerkte bericht.

** Uitvoer **

retourneert het document met de opgegeven ID, of een **`404 Not Found`** fout als het document niet bestaat.

![ela_delete_data](https://cdn.illacloud.com/official-website/img/official-site/ela_delete_data.jpg)
