## <h2 hidden>Wat is Redis?</h2>

Redis in Illa is een database-integratie waarmee je verbinding en interactie met een Redis-database kunt maken. Redis is een populair open-source database beheer systeem dat bekend staat om zijn snelheid, betrouwbaarheid en gebruiksgemak.

Met de Redis-integratie in Illa kunt u gegevens opzoeken, invoegen, bijwerken en verwijderen uit een Redis-database met behulp van SQL commando's. U kunt ook aangepaste SQL-query's maken en uitvoeren binnen Illa. Dit maakt het makkelijk om data-driven applicaties en dashboards te bouwen die real-time gegevens weergeven uit een Redis-database.

### Redis aanmaken

Er zijn twee manieren om een bron in Illa te maken nadat je bent ingelogd op je Illa account.

-   Maak in Bronnen

Log in op je Illa account, selecteer `**Resources**` bovenaan de pagina, en klik op de `**Maak Nieuw**` knop.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Selecteer **`Redis`** uit de database lijst.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Maak verbinding met de database met de vereiste parameters beschreven in `Connectie Instellingen` hieronder.

Klik op `**Test Connection**` om te zien of we met succes verbinding kunnen maken met de database. Zo ja, klik **`Resources`**, anders controleer je de hostnaam, port, gebruikersnaam en wachtwoord correct.

Na het aanmaken van een bron, worden de gerealiseerde Redis weergegeven zoals getoond.

![redis_ex_ready](https://cdn.illacloud.com/official-website/img/docs/redis_ex_ready.png)

-   Maak in Bouwer

Log in op je Illa account, maak een project aan in Illa Builder in de `**App**` pagina en navigeer naar de **`Action List`** onderaan de pagina. Klik op `**nieuw**`, en selecteer dan **`Redis`** uit de database lijst. Maak vervolgens verbinding met de database met de vereiste parameters beschreven in `Connectie Instellingen` hieronder.

Klik op `**Test Connection**` om te zien of we met succes verbinding kunnen maken met de database. Zo ja, klik **`Resources`**, anders controleer je de hostnaam, port, gebruikersnaam en wachtwoord correct.

### Verbindingsinstellingen

Hier moeten we informatie verstrekken voor het verbinden met de Redis-database.

![redis_config](https://cdn.illacloud.com/official-website/img/docs/redis_config.png)

| Eigenschappen        | vereist   | Beschrijving                                                                                                                                |
| -------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| naam                 | verplicht | De naam voor resource bij het maken van acties in de ILLA.                                                                                  |
| Hostname             | verplicht | De URL of het IP-adres van uw database                                                                                                      |
| Poort                | verplicht | Het poortnummer van de server host dat u moet gebruiken om te verbinden. Als je geen poort opgeeft, is de standaard poort '6379'.           |
| Database index       | optioneel | een gegevensstructuur die wordt gebruikt om de prestaties van query's die gegevens ophalen uit een Redis-database te optimaliseren          |
| Database             | verplicht | De naam van de database                                                                                                                     |
| Gebruikersnaam       | verplicht | de gebruikersnaam die je wilt gebruiken bij het inloggen op de Redis-server.                                                                |
| Wachtwoord           | verplicht | Gebruik dit wachtwoord voor verificatie.                                                                                                    |
| SSL opties           | optioneel | Bepaal hoe hoog een veilige SSL TCP/IP verbinding prioriteit heeft tijdens het onderhandelen met de server.                                 |

### Acties aanmaken

We hebben een Redis-bron gemaakt, we kunnen de actie toevoegen door op Redis van actielijst te klikken en te kiezen voor de **`Create action`** knop.

![redis_resource_list](https://cdn.illacloud.com/official-website/img/docs/redis_resource_list.png)

Nu hebben we de Redis-server toegevoegd als actie aan onze bouwpagina.

![redis](https://cdn.illacloud.com/official-website/img/docs/redis.png)

### Redis configureren

| Eigenschappen  | Beschrijving                                                            |
| -------------- | ----------------------------------------------------------------------- |
| vraag          | commando's om gegevens uit de database op te halen                      |
| Transformator  | data omzetten naar de stijl die je leuk vindt met behulp van JavaScript |

Voorbeeld gebruik:

-   BELANGRIJK

Dit commando haalt een lijst van alle sleutels in de Redis-database. Je kunt jokertekens gebruiken om de sleutels op patroon te filteren.

```sql
KEYS *
KEYS user:*
```

-   KRIJG

Dit commando haalt de waarde op die is gekoppeld aan een bepaalde sleutel.

```sql
GET user:12345
```

-   HGETALL

Dit commando haalt alle velden en hun waarden op van een Redis-hash-gegevensstructuur.

```sql
HGETALL user:12345
```

-   ZRANGE

Dit commando haalt een bereik van waarden op uit een Redis gesorteerde gegevensstructuur, gebaseerd op hun scores.

```sql
ZRANGE high_scores 0 9 WITHSCORES
```

-   LRANGE

Dit commando haalt een bereik van waarden op van een Redis-lijst data-structuur.

```sql
LRANGE my_list 0 4
```

-   BESTELINGEN

Deze opdracht controleert of een bepaalde sleutel bestaat in de Redis-database.

```sql
EXISTS user:12345
```

-   TTL

Dit commando haalt de tijd-naar-live (TTL) waarde op die gekoppeld is aan een sleutel, wat aangeeft hoe lang de sleutel in de database blijft voordat deze verloopt.

```sql
TTL user:12345
```
