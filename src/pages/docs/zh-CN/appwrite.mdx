## <h2 hidden>Appwrite Database</h2>

Appwrite提供了用户身份验证和帐户管理、用户偏好、数据库和存储持久性、云功能、本地化、图像处理等多种服务。 

ILLA Cloud与Appwrite的结合，即服务端低代码与前端低代码的结合，将真正实现低代码全栈开发。

[点此](https://appwrite.io/)了解更多关于Appwrite的信息。

### 在ILLA Cloud中配置Appwrite资源

#### 两种方式来创建资源

第一种方式时进入“资源”标签页 > 点击“新建”并选择Appwrite 

<img src='https://cdn.illacloud.com/official-website/img/docs/assemble/appwrite1.png' alt="" />

另一种方式是进入应用编辑页面 > 点击“+新建” 并选择Appwrite > 点击“新建资源”来添加新配置。 

<img src='https://cdn.illacloud.com/official-website/img/docs/assemble/appwrite2.png' alt="" />

#### 通过输入下述字段来配置Appwrite

在连接一个Appwrite 资源之前，请确保您已经在本地机器或云服务提供商上部署了Appwrite。 [点击这里](https://appwrite.io/docs/installation) 了解更多关于安装 Appwrite的信息。 

| 属性          | 是否必填     | 描述                                                         |
| ----------- | -------- | ---------------------------------------------------------- |
| Name        | required | 在ILLA中创建Action时可以看到的资源名称。 您可以自定义名称以便于识别。 您可以自定义一个名称以便于识别。  |
| Host        | required | 主机名或IP                                                     |
| Database ID | required | 数据库 ID                                                     |
| Project ID  | required | Project ID                                                 |
| Api key     | required | 令牌                                                         |

<img src='https://cdn.illacloud.com/official-website/img/docs/assemble/appwrite3.png' alt="" />

### 在应用程序编辑器中创建Action

#### 通过以下方式创建Action

在应用的编辑页中点击 “+ 新建“ > 选择Appwrite > 选择已有的Appwrite资源或创建新资源 > 点击“创建Action”以创建新Action。 

<img src='https://cdn.illacloud.com/official-website/img/docs/assemble/appwrite4.png' alt="" />

创建Action后，从下述方法中选择一个您预期执行的方法： 列出文档、创建文档、更新文档、获取文档和删除文档。 每种方法的配置如下。 

#### 列出文档

下面是一个简单的配置示例：列出$id为5的文档。 

<img src='https://cdn.illacloud.com/official-website/img/docs/assemble/appwrite5.png' alt="" />

| 属性            | 必填项 | 描述                                                                                                                                               |
| ------------- | --- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| Collection ID | 必填  | 输入一个集合id                                                                                                                                         |
| Filter        | 可选的 | 支持多个过滤器。 过滤器由Key、Value和Condition组成。 1. key：输入一个属性 2. condition：选择一个条件，包括equal, notEqual, lessThan, lessThanEqual, greaterThan等。3.value：输入值 值：输入值 |
| Order by      | 可选的 | 支持按多个属性排序。 1. 输入一个属性到排序方式 2。 选择asc(默认) 或desc                                                                                                     |
| Limit         | 可选的 | 返回的记录数量                                                                                                                                          |

下面举例说明了如何在Action中使用组件中输入的数据：

1.  在画布中添加一个input组件，让用户可以输入自己想要查找的文档id。  
2.  设置filter属性：将Key字段设置为$id，并在Value字段中输入{{ input1.value }}。 

<img src='https://cdn.illacloud.com/official-website/img/docs/assemble/appwrite6.png' alt="" />
