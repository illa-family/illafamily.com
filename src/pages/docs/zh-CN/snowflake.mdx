## <h2 hidden>什么是Snowflake？</h2>

在Illa, 你可以使用 Snowflake 查询编辑器来执行SQL 查询你的Snowflake 数据仓库。 您可以通过输入您的Snowflake账户凭据连接到 Snowflake，包括账户名称、用户名、密码和仓库名称。 一旦您连接到 Snowflake, 您可以使用查询编辑器写入并执行 SQL 查询以检索、操纵和分析您的数据。

### 创建Snowflake

在登录到您的 Illa 帐户后，有两种方式在Illa 中创建资源。

-   在资源中创建

登录到您的 Illa 帐户，在页面顶部选择 `**资源**` 并点击 `**创建新的**` 按钮。

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

从数据库列表中选择 **`Snowflake`** 。

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

使用下面`链接设置`描述的必填参数连接到数据库。

点击 "**测试连接**" 查看我们是否可以成功地连接到数据库。 如果是，请点击 **\`保存资源** \*，请重新检查主机名、端口、用户名和密码是否正确。

创建资源后，即将显示准备好的雪花。

![snow_ex_ready](https://cdn.illacloud.com/official-website/img/docs/snow_ex_Ready.png)

-   在构建器中创建

登录您的 Illa 帐户，在`**App**` 页面中创建一个项目 并导航到页面底部的 **`Action List` **。 点击`**新**`，然后从数据库列表中选择 **`Snowflake`** 。 然后，用下面`链接设置`描述的参数连接到数据库。

点击 "**测试连接**" 查看我们是否可以成功地连接到数据库。 如果是，请点击 **\`保存资源** \*，请重新检查主机名、端口、用户名和密码是否正确。

### 连接设置

这里我们需要提供连接到 Snowflake 数据库的信息。

![snow_config](https://cdn.illacloud.com/official-website/img/docs/snow_config.png)

| 属性           | 必填  | 描述                                                                                                                                   |
| ------------ | --- | ------------------------------------------------------------------------------------------------------------------------------------ |
| 名称           | 必填  | 在 ILLA 中创建动作时的资源名称。                                                                                                                  |
| 帐户名称         | 必填  | 分配给每个Snowflake帐户的唯一标识符                                                                                                               |
| 仓库           | 必填  | 用于处理查询和运行分析工作量的一组计算资源                                                                                                                |
| 数据库          | 必填  | 数据库名称                                                                                                                                |
| 图案           | 可选的 | 一个容器储存数据库对象，如表、视图和程序                                                                                                                 |
| 作用           | 可选的 | 一个命名的权限集可以分配给一个或多个用户                                                                                                                 |
| 认证           | 必填  | 在 Snowflake 数据仓库中安全访问您的数据和资源 (基本认证或密钥配对 (只支持加密的私钥)                                                                                   |
| 用户名          | 必填  | 登录到 Snowflake 服务器时您想要使用的用户名。                                                                                                         |
| 密码           | 必填  | 使用此密码进行身份验证。                                                                                                                         |

### 创建动作

我们已经创建了一个Snowflakes资源，我们可以通过从动作列表中选择Snowflakes来添加动作，并选择 **`创建动作` ** 按钮。

![snow_resource_list](https://cdn.illacloud.com/official-website/img/docs/snow_resource_list.png)

现在我们已经将 Snowflake 服务器添加到我们的建筑页面作为动作。

![snow](https://cdn.illacloud.com/official-website/img/docs/snow.png)

### 配置 Snowflake

| 属性       | 描述                                                               |
| -------- | ---------------------------------------------------------------- |
| SQL 查询   | 要获取的 sql 命令 (选择), 插入, 更新, 删除数据库中的数据                              |
| 转换器      | 将数据转换为你喜欢使用 JavaScript 的样式                                       |

示例用法：

-   选择

```sql
选择* 从用户排序按姓排列；
```

-   Insert

```sql
INSERT INTO users

	(firstname, lastname, email)

VALUES

	(

		{{ NewUser.data.firstNameInput }},

		{{ NewUser.data.lastNameInput }},

		{{ NewUser.data.emailInput }}

	);
```

-   更新

```sql
UPDATE users
  SET firstname = '{{form1.updatedProfile.first}}'
  WHERE id = {{ form1.updatedProfile.uid}};
```

-   删除

```sql
DELETE FROM users WHERE id = {{ form1.updatedProfile.uid }};
```
