## <h2 hidden>时间范围是什么？</h2>

时间范围组件是一个用户可以从24小时界面选择范围的界面。 通常用于用户需要指定开始和结束时间范围的表格或数据输入接口。

当一个时间范围组件添加到画布时，它会同时显示两个24小时接口。 一个接口用于选择开始时间，另一个则用于选择结束时间。 用户可以通过移动鼠标光标或使用上下箭头来调整时间。

### Properties

时间范围可用的属性。 JavaScript 可以被写入阅读或更改组件信息。

| 属性     | 描述                                                               |
| ------ | ---------------------------------------------------------------- |
| 开始时间   | 组件的默认起始时间值                                                       |
| 结束时间   | 组件的默认结束时间值                                                       |
| 格式     | 有效的日期格式字符串。 见 https://day.js.org/docs/en/parse/string-formation. |
| 启动占位符  | 当开始时间输入字段为空时，将显示该值。                                              |
| 终端占位符  | 当结束时间输入字段为空时，值将显示。                                               |
| 步骤大小   | 以分钟为单位的步长                                                        |
| 标签     | 显示给用户的字段名称                                                       |
| 标题     | 详细描述字段的标题                                                        |
| 隐藏标签   | 设置是否显示标签                                                         |
| 位置     | 设置标签相对于组件的位置                                                     |
| 对齐     | 设置标签的对齐(对齐到左或右)                                                  |
| Width  | 当标签位于组件左侧时，设置标签的宽度。                                              |
| 事件处理程序 | 响应组件事件触发查询、控制组件或调用其他API。                                         |
| 已禁用    | 控制组件是否被禁用的状态。 当组件被禁用时，它不能被修改或聚焦。                                 |
| 工具提示   | 用户可以在此输入组件工具提示。 工具提示将在其焦点时显示。 支持Markdown 格式。                     |
| 显示清除按钮 | 控制是否在输入字段中显示清除按钮                                                 |
| 必填字段   | 只有当开启开关时才有效。                                                     |
| 自定义规则  | 在此创建您的验证逻辑。 规则应该在 JavaScript 中制定，并且由 {{}} 涵盖。                    |
| 隐藏验证消息 | 当输入值不正确时，您可以通过切换隐藏状态来隐藏错误消息。 您可以用JavaScript动态地更改隐藏状态。            |
| 表单数据键  | 在构建数据属性时指定包装表单组件的键值。                                             |
| Hidden | 动态控制组件是否隐藏。 您可以通过动态布尔值更改隐藏状态。                                    |
| 主题颜色   | 允许用户指定按钮的背景颜色和不透明度                                               |

### Methods

您可以使用其他组件来控制组件。 我们支持以下两种方法：

-   **setStartTime**

要设置起始日期值，例如 {{"value1"}}

| 属性 | 描述      |
| -- | ------- |
| 值  | 输入起始时间值 |

-   **setEndTime**

要设置结束日期值，例如 {{"value2"}}

| 属性 | 描述      |
| -- | ------- |
| 值  | 输入结束时间值 |

-   **clearValue**

清除选中组件的值 

-   **validate**

验证输入信息是合法的

-   **清除验证**

清除验证消息

### 事件处理程序

| 事件 | 描述                       |
| -- | ------------------------ |
| 更改 | 当用户更改所选时间值时，执行用户定制的特定操作。 |

### 使用大小写

接下来，我们将演示如何绘制数据源到时间范围的数据，并用按钮将结束值设置为当前时间。 

#### 步骤 1 添加动作

让我们在REST API 中创建一个叫做`promoteCodes`** 包括6列：**`code`**、**`id`**、**`createdAt`**、**`expireedAt`**、**`startedAt`、`updatedAt`\*\*。 我们将使用启动和过期的时间进行采样使用。 

然后我们可以从行动列表中创建一个新的 REST API 动作，列出促销代码中的所有数据，并命名restapi1。 

点击 **\`保存** 和 **`Run`** 激活此操作。

#### 步骤 2 添加组件

接下来，我们可以在画布中添加 **`时间范围` ** 组件和 **`按钮` **。 对于日期范围组件的默认起始时间值，我们使用开始时间。当第一次推广来自其余api数据的代码时。

```sql
{{restapi1.data.promoteCodes[0].startedAt}}
```

同样，对于默认的结束时间值，我们使用过期时间。当第一次推广来自其他api数据的代码时。

```sql
{{restapi1.data.promoteCodes[0].expiredAt}}
```

我们还将按钮标为“现在”。

![time_range_0](https://cdn.illacloud.com/official-website/img/docs/time_range_0.png)

#### 步骤 3 配置组件

在 **`按钮`** 组件中，我们可以配置它来设置结束日期值在 **`时间范围`** 组件中设置到今天的日期。

1.  在编辑事件处理器中，选择 **`控制组件` ** 正在执行中。 选择你想要更新的 **`时间范围` ** 组件作为事件的目标。
2.  选择 **`setEndTime` ** 动作并设置 **`结束时间` ** 为`{{currentUserInfo.updatedAt}}` 

![time_range_config](https://cdn.illacloud.com/official-website/img/docs/time_range_config.png)

#### 步骤4 测试

现在当您点击“现在”按钮时，结束时间应该更改为今天的日期。 至于这里, 我们有 20:03:05, 但在您测试时可能不同。

![time_range_1](https://cdn.illacloud.com/official-website/img/docs/time_range_1.png)
