## <Heading hidden>快速上手</Heading>

### 1.  部署ILLA Builder

在开始使用 ILLA Builder之前，您应该选择使用 ILLA 云端或自助ILLA 来托管您的应用程序。 我们为您提供ILLA CLI, Docker Compose, 和 K8s以便在5分钟内快速部署ILLA 。 稍后我们将为您提供ILLA 云，帮助您简单地开始使用ILLA Builder。

<img
  src={require('@/img/docs/qs_c_1.png').default}
  alt=""
/>

### 2.  获取数据

注册并登录到ILLA Builder，然后输入资源选项卡。 点击“创建新”按钮，然后您可以在此选择资源类型。 我们为您提供REST API 和 MySQL 来连接您的数据。 Postgres、Redis和MongoDB将于本月启用。

<img
  src='https://cdn.illacloud.com/official-website/img/docs/qs_e_2.png'
  alt=""
/>

### 3.  拖拽组件

当您的数据准备好时，您可以返回应用选项卡并创建一个应用程序。 插入列表在页面右侧。 您可以选择其中一个并将其拖动到框架内。 然后点击在检查面板中插入并修改其参数。 我们为您提供了Markdown模式来编辑每个插入的标签。 如果您熟悉Markdown语法，您可以快速调整文本大小和粗细。 同时，您可以使用 {{double brackets}} 在文本输入框中调用动作数据，在ILLA Builder中取得更多结果。

### 4.  编写查询

需要通过Action功能将数据从资源转换为插入数据。 我们称之为Action 您可以在操作中写一个查询来从您的资源获取数据。 在此基础上，您可以在执行数据处理指令的动作中写入 JavaScript ，如过滤器和合并。 MySQL 示例如下：

<img
  src='https://cdn.illacloud.com/official-website/img/docs/qs_e_3.png'
  alt=""
/>

1.  在应用编辑器中创建 MySQL 操作。 命名它是action1。

2.  在配置面板中输入查询语句，例如 SELECT \* FROM Users。

3.  运行此操作，查询结果将显示在底面板上。

### 5.  展示数据

当操作被成功创建时，您需要将操作结果绑定到组件中。 您应该将检查面板上的值变量替换为操作 {{action1.data}} 的结果。 然后组件数据将与Action结果绑定。
