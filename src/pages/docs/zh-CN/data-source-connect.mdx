学习如何连接到您的数据源，并在您的 ILLA Builder 中呼叫它。

ILLA 允许您将包括数据库和API在内的广泛数据源连接并整合到您的组织中，作为资源。

## 数据来源

数据来源是储存和处理数据的地点。 数据来源的例子包括PostgreSQL数据库、Twilio REST API和Amazon S3 buckets。 在 ILLA 中，您可以添加多个数据来源并让用户可以查询它们。

### **集成**

ILLA 为许多需要最小配置的热门数据源提供内置集成，如SQL数据库和 Stripe。 这些集成，也称为资源，允许您使用Retool连接和查询数据源。 如果特定类型的数据源不被内置集成支持， 您可以使用REST、GraphQL或 SOAP 集成连接到几乎任意API。 这使您能够从ILLA构建器中访问广泛的数据源。

## 资源

ILLA 中的资源是一个已保存的配置，它指定了ILLA 如何连接到数据源。 要在 ILLA中使用数据源，您必须为此创建一个资源。 您可以在写查询时选择访问数据的适当资源。

ILLA 中的每个资源都有自己的配置选项来决定ILLA 如何与相应的数据源连接。 这些选项可以包括身份验证凭据和流程、数据库连接设置和 API 设置，如基本的 URL 和默认头部。

### **资源如何工作**

ILLA 资源存储安全连接到您的数据源的必要设置。 这使得您可以轻松地查询数据，而不会直接向用户暴露您的数据源。 当一个查询被运行到资源时， 重新工具使用资源的配置设置来代理服务器上数据源的请求。 这确保只有Retool才能直接访问数据源。

ILLA 简化了为您的用户连接到数据源的过程。 他们只需在撰写查询时选择相关资源，而不必担心数据来源的细节。 您也可以添加多个相同类型的资源，例如多个Amazon S3桶来区分它们。

这种方法将您的数据源隐藏在您的用户身上，并确保与数据源的所有交互都能安全地处理。 ILLA 避免将客户端连接到可能暴露敏感信息的数据源，并在所有互动中使用同样的条件。 此外，资源启用查询缓存，这可以减少向数据源提出请求的次数，并可能降低费用。
