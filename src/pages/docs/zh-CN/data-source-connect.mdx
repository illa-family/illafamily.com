学习如何连接到您的数据源，并在您的 ILLA Builder 中调用它。

ILLA 允许您连接到多种数据库和API，并将其作为数据源直接使用。

## 数据源

数据源是储存和处理数据的地点。 数据源包括但不限于PostgreSQL、REST API、Amazon S3 buckets等 在 ILLA 中，您可以添加多个数据来源并让用户可以通过APP中的Action运行它们。

### **集成**

ILLA 为许多热门数据源提供内置集成，只需要填入最少的配置，即可连接。如MySQL等。 这些集成，也称为资源，允许您使用ILLA连接和查询数据源。 如果特定类型的数据源不被内置集成支持， 您可以使用REST、GraphQL集成连接到任意API。 这使您能够从ILLA Builder中访问广泛的数据源。

### 连接到数据库

ILLA支持连接到大多数数据库

### 连接到 APIs

连接到 REST 和 GraphQL和多数常见API

## 资源

ILLA 中的资源是一个已保存的配置，它指定了ILLA 如何连接到数据源。 要在 ILLA中使用数据源，您必须为此创建一个资源。 您可以在写查询时选择访问数据的适当资源。

ILLA 中的每个资源都有自己的配置选项来决定ILLA 如何与相应的数据源连接。 这些选项可以包括身份验证凭据和流程、数据库连接设置和 API 设置，如基本的 URL 和默认头部。

### **资源如何工作**

ILLA 的资源将存储安全连接到您的数据源的必要设置。 这使得您可以轻松地查询数据，而不会直接向用户暴露您的数据源。 当一个Action被运行时， ILLA使用资源的配置设置来代理服务器上数据源的请求。 这确保只有ILLA才能直接访问数据源。

ILLA 简化了您连接到数据源的过程。 您只需在配置Action时选择相关资源，而不必担心数据连接的细节。 您也可以添加多个相同类型的资源，例如多个Amazon S3桶来区分它们。

这种方法将您的数据源从用户侧隐藏，并确保与数据源的所有交互都能被安全地处理。 ILLA 避免将客户端连接到可能暴露敏感信息的数据源，并在所有互动中使用同样的连接方式。 此外，资源的查询缓存，也可以减少向数据源提出请求的次数，并可能降低费用。
