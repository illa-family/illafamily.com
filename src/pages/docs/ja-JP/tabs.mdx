## <h2 hidden>タブとは?</h2>

ILLA Cloudのタブコンポーネントは、タブレイアウトでコンテンツを整理し、表示することができる汎用性の高い機能です。 これにより、単一のコンテナ内に複数のタブを作成することができ、ユーザーはコンテンツの異なるセクションを簡単に切り替えることができます。 format@@0コンポーネントは、関連情報をナビゲートするためのユーザーフレンドリーなインターフェースを提供し、全体的なユーザーエクスペリエンスを向上させます。 カスタマイズ可能なスタイルオプションと、テキスト、画像、埋め込みコンポーネントなど、さまざまな種類のコンテンツを追加する機能を備えています。 ILLA クラウドのタブコンポーネントは、柔軟性を提供し、データの組織とプレゼンテーションを強化します。

### プロパティー

| プロパティー              | 説明                                                                                                                                           |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| コンテナへのリンク           | タブコンポーネントを特定のコンテナにリンクさせ、タブと関連付けられたコンテンツ間をシームレスにナビゲーションすることができます。                                                                             |
| イベントハンドラ：           | クリック、キープレス、値の変更など、特定のユーザーアクションやイベントを検出し、応答します。                                                                                               |
| 無効                  | 特定のタブを無効にし、ユーザーとのやり取りを妨げます                                                                                                                   |
| ツールチップ              | 各タブの追加情報やコンテキストについて                                                                                                                          |
| レイアウト               | レイアウトオプションが異なるため、タブをデザイン設定や利用可能なスペースに基づいて水平方向または垂直方向に配置できます。                                                                                 |
| Align               | 左、中央、または右の配置など、タブコンポーネントを親コンテナ内の異なる位置に揃えます。                                                                                                  |
| Hidden              | プログラムで特定のタブを隠すんだ                                                                                                                             |
| テキストの色              | カスタマイズ可能なテキストカラーを使用すると、アプリケーションのデザインやブランドに合わせて、異なる色でタブラベルのスタイルを設定できます。                                                                       |

### データ

コンポーネントには一般的に使用されているデータがいくつか含まれており、アプリで **`{{componentName.propertyName}}`** を介して呼び出すことができます。

| プロパティ名                | 説明                                                                                                      |
| --------------------- | ------------------------------------------------------------------------------------------------------- |
| align                 | 左、中央、または右の配置などのコンポーネント内のタブの配置                                                                           |
| colorScheme           | コンポーネント内のテキストの色を定義します。                                                                                  |
| currentIndex          | は、タブリスト内の現在選択されているタブのインデックスを表します。                                                                       |
| currentKey            | 現在選択されているタブの一意の識別子またはキー                                                                                 |
| displayName           | は、コンポーネントに割り当てられた名前またはラベルを表します。                                                                         |
| hidden                | タブコンポーネントを表示または非表示にするかを決定するブール値                                                                         |
| linkWidgetDisplayName | 特定のタブに関連付けられているリンクウィジェットの名前またはラベルを表示する                                                                  |
| navigateContainer     | タブの内容を表示または移動するコンテナまたは保存先                                                                               |
| tabList               | 表示名、キー、その他の関連情報を含むタブのリストです。                                                                             |
| tabPosition           | タブストリップの位置 (上、下、左、右など)                                                                                  |
| tooltipText           | ユーザーが特定のタブに移動したときにツールチップとして表示されるテキストです。                                                                 |
| viewList              | 各タブに関連付けられたビューまたはコンテンツ、選択したタブに基づいて異なるコンテンツのレンダリングを有効にします                                                |

例: {{tabs1.displayName}}

### 大文字小文字を区別する

tabs コンポーネントと container コンポーネントを組み合わせて、3 つのビューを表示する方法を説明します。

#### ステップ 1 コンポーネントの追加と設定

**`container`** コンポーネントと **`tabs`** コンポーネントを追加します。 tabs コンポーネントの `inspect` ページで、 **\`Link to a Container** をオンにし、 \`Container\*\* にリンクしたいコンテナを選択します。

-   表示 1:

これがホームページですので、コンテナの左上隅に **`text`** コンポーネントと "Home" を配置します。

**`text`** コンポーネント"Hi! 私の個人的なウェブサイトへようこそ」と、その下には「私についてもっと知る」というラベルの付いた**`button`** コンポーネントがあります。

**`button`** コンポーネントでは、クリックされた場合、次のビュー (view2) に移動するように設定できます。

1.  **`button`** コンポーネントをクリックして検査ページを開きます。 **`Event handler`**の下で `*** 新しい**` をクリックします。
2.  Edit event handlerで**`Control component`** を実行中に選択します。 をクリックし、更新する **`container`** コンポーネントをイベントのターゲットとして選択します。
3.  **`showNextView`**メソッドを選択します。

"Invite" **`button`** コンポーネントでは、クリックされた場合にフォームに情報を送信するように設定できます。

1.  **`button`** コンポーネントをクリックして検査ページを開きます。 **`Event handler`**の下で `*** 新しい**` をクリックします。
2.  Edit event handlerで**`Control component`** を実行中に選択します。 をクリックし、更新する **`form`** コンポーネントをイベントのターゲットとして選択します。
3.  **`submit`** メソッドを選択します。

**`form`** コンポーネントについては、送信された場合に通知を表示するように設定できます。

1.  **`form`** コンポーネントをクリックして検査ページを開きます。 **`Event handler`**の下で `*** 新しい**` をクリックします。
2.  Edit event handlerで、Event で **`submit`** を選択し、 **`action`** で **`Show notification`** を選択します。 タイトルに「送信成功」、説明に「フレンドリクエスト送信済み」、\*\*`タイプ`の`成功`を選択します。 通知を2秒間表示させて、継続時間に`{{2000}}`を入れてください。
