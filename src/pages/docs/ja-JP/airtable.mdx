## <h2 hidden>エアラブルとは何ですか？</h2>

### エアテーブルのバックエンドを設定

`Airtable` にアカウントを作成し、必要に応じてベースを作成します。

ベースを ILLA Cloud に接続するには、`developer hub` に移動します。

![air_developer_hub](https://cdn.illacloud.com/official-website/img/official-site/intergration/air_developer_hub.png)

`Personal Access Token`のページで、`Create new Token`をクリックします。

![air_create_new](https://cdn.illacloud.com/official-website/img/official-site/intergration/air_create_new.jpeg)

これにより、作成しているPATの設定オプションが開きます。

最初に PAT という名前を付け、この PAT にアクセスさせたいスコープを選択します。 最後に、PATのアクセスレベルを選択します。 最後に、PATのアクセスレベルを選択します。 最後に、PATのアクセスレベルを選択します。

![air_new_token](https://cdn.illacloud.com/official-website/img/official-site/intergration/air_new_token.jpeg)

**`create token`**ボタンをクリックすると、PAT接続の準備ができました。

### ILLA Cloud にエアラブルを作成

Illaアカウントにログインした後、Illaにリソースを作成するには2つの方法があります。

-   リソースに作成

Illaアカウントにサインインし、ページの上部にある`**リソース**`を選択し、`**新規作成**`ボタンをクリックします。

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

![action_list_3](https://cdn.illacloud.com/official-website/img/official-site/intergration/action_list_3.jpeg)

### 接続設定

ここでは、Airtable API に接続するための情報を提供する必要があります。

![air_config](https://cdn.illacloud.com/official-website/img/official-site/intergration/air_config.jpeg)

| プロパティー     | 必須 | 説明                                                                                                                                                  |
| ---------- | -- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| 名前         | 必須 | ILLAでアクションを作成する際のリソース名です。                                                                                                                           |
| アクセストークン   | 必須 | サードパーティーのアプリケーションまたはサービスの許可を与える一部の情報 Airtable accountのデータにアクセスし、代わってアクションを実行するための情報                                                                 |

### アクションを作成

Airtable リソースを作成しました。アクションリストから Airtable を選択し、**`Create action`** ボタンを選択してアクションを追加できます。

![air_resource_list](https://cdn.illacloud.com/official-website/img/official-site/intergration/air_resource_list.jpeg)

これで、ビルドページに「Airtable API」を追加しました。

![air](https://cdn.illacloud.com/official-website/img/official-site/intergration/air.jpeg)

### Airtable の設定

| プロパティー    | 説明                                                                                                                             |
| --------- | ------------------------------------------------------------------------------------------------------------------------------ |
| ベースID     | エアテーブルデータベースの一意の識別子 ベースURLからベースIDを取得できます。 通常は"app"で始まる文字列です。                                                                   |
| テーブル名     | は、選択したベース内の特定のテーブルを指定します。                                                                                                      |
| アクションタイプ  | 指定されたテーブルで統合がどのような操作を実行するか                                                                                                     |
| トランス      | JavaScriptを使用して好きなスタイルにデータを変換します                                                                                               |

#### レコードの一覧

Airtable で指定されたテーブルからレコードのリストを取得します。

** Input **

| プロパティー            | 必須   | 説明                                                                                 |
| ----------------- | ---- | ---------------------------------------------------------------------------------- |
| 表示                | 省略可能 | 特定の基準に基づいてレコードを整理して表示する                                                            |
| フィールド             | 省略可能 | フィールドは、テーブル内の列を参照します                                                               |
| 数式でフィルター          | 省略可能 | は、取得または対話するレコードをフィルタリングする数式を定義します。                                                 |
| 並べ替え              | 省略可能 | レコードが表示されたり取得された順序です                                                               |
| 最大レコード            | 省略可能 | 1つのリクエストでインテグレーションを取得またはやり取りするレコードの数を制限します。                                        |
| ページサイズ            | 省略可能 | 多くの場合、複数のバッチ(ページ)でデータを取得する必要があるシナリオで使用されます                                         |
| オフセット             | 省略可能 | は、特定のページ内のレコードを取得するための開始点を指定します。                                                   |
| セルの書式             | 省略可能 | 特定のフィールド内でデータがどのように表示されるか                                                          |
| タイムゾーン            | 省略可能 | 日付と時刻の計算とプレゼンテーションが正確で一貫性があることを保証します                                               |
| ユーザーの場所           | 省略可能 | ユーザーの地理的位置や好みに基づいて統合出力を調整します                                                       |
