## <Heading hidden>クエリ</Heading>

リソースを作成したら、リソース内のデータを抽出する必要があります。 データをマージし、論理要件に従ってInsertsに入れます。 私たちは、このデータ処理プロセスをアクションと呼んでいます。 アクションは、データとコンポーネントの間の架け橋であり、アプリケーションを構築する上で不可欠な要素です。 行動とトランスフォーマーの二つのカテゴリーに分かれています。

### SQL アクション

#### アクションを作成

SQLアクションを使用するプロセスは、次のステップに分かれています。

1.  まず、アプリエディタでアクションリストを見つけて「追加」ボタンをクリックする必要があります。
2.  次に、リソース選択ボックスでデータ ソースの種類を選択します。 作成されたリソースは、対応するリソースリストで見つけることができます。ここで新しいリソースを作成することもできます。 作成したリソースを選択し、「確認」ボタンをクリックします。 SQL アクションは正常に作成されます。
3.  作成されたアクションがアクションリストに表示されます。 アクションエディタは、アクションをキックした後、フレームの下に表示されます。 アクションのリソースを変更したり、アクションステートメントを書いたり、レスポンスコンテンツを追加したり、変換したり、イベントハンドラを追加したりすることができます。
4.  アクションリストを入力した後、"Preview" ボタンをクリックして、作成したアクションが有効かどうかを確認できます。 設定パネルにアクションの結果が表示されます。 アクションが編集されたら、「保存」ボタンをクリックして処理を終了します。 情報を保存せずに別のアクションに切り替えると、システムは変更を保存するように求められます。

#### アクションの破壊

1.  Action: Action モジュールに Code Mirror エディタを追加し、アクション文のハイライトとコード補完を有効にしたアクション文を入力できます。 その後のアップデートで アクションステートメントに慣れていないユーザーがデータアクションを完了できるよう、アクションステートメントを記述するためのGUI入力モードを提供します。
2.  Transformer: 変換器内の JavaScript 文でアクションロジックを絞り込むことができます。 例えば、action文のwhere関数の代わりにJSでフィルタ関数を使用するなどです。
3.  イベントハンドラ: イベントハンドラを使用すると、ユーザーの操作に応じてアクションのトリガー、挿入制御、およびその他のアクションを実行できます。 カスタム JavaScript ステートメントを追加することなく、イベントに応答するために複数のイベントハンドラを追加できます。 例えば、ユーザーがボタンをクリックすると、(1) 実行するアクションをトリガーできます。 (2) 複数のドロップダウン リストとテキスト入力の状態をリセットします。
4.  Response: Response モジュールでは、アクションが不正な結果を返す条件をカスタマイズできます。 行動の正確性を確保するための方法です
5.  Advanced:高度なオプションで、データベースのタイムアウト応答時間、イベントプロセッサの遅延などのアクションの詳細設定を調整します。

### APIアクション

#### アクションを作成

API を使用してデータを操作するプロセスは、SQL に似ています。

1.  まず、 ユーザーは、アプリケーションの編集ページの左下にあるアクションリストを見つけて、アクションリストの右上にあるプラス記号をクリックする必要があります。
2.  次に、表示されるリソース選択ボックスで対応するデータソースリストを選択します。 作成されたリソースは、対応するデータソースリストで見つけることができます。また、ここで新しいリソースを作成することもできます。 作成したリソースを選択し、「作成」をクリックすると、APIアクションが正常に作成されます。
3.  正常に作成されたアクションは、下部パネルのformat@@0リストに表示され、クリックすると、アクションのformat@@1タブがフレームの下に表示されます。 ILLA 内の任意の HTTP API にリクエストを行うことができます。 データリクエストメソッドについては API ドキュメントを参照してください。
4.  あなたの行動に記入したら、 「プレビュー」ボタンをクリックすると、記述したアクションが有効であるかどうかを確認でき、設定パネルに結果が表示されます。 アクションの編集が完了したら、format@@0をクリックして終了し、処理を終了します。 保存せずに別の操作に切り替えると、保存操作を完了するようポップアップが表示されます。

#### アクションの破壊

1.  Transformer: 変換器内の JavaScript 文でアクションロジックを絞り込むことができます。 例えば、action文のwhere関数の代わりにJSでフィルタ関数を使用するなどします。
2.  イベントハンドラ: イベントハンドラを使用すると、ユーザーの操作に応じてアクションのトリガー、挿入制御、およびその他のアクションを実行できます。 カスタム JavaScript ステートメントを追加することなく、イベントに応答するために複数のイベントハンドラを追加できます。 例えば、ユーザーがボタンをクリックすると、(1) 実行するアクションをトリガーできます。 (2) 複数のドロップダウン リストとテキスト入力の状態をリセットします。
