## <Heading hidden>Consulta</Heading>

Después de crear el recurso, necesita extraer los datos en el recurso, seleccionar, filtrar, fusionar los datos y luego colocarlos en los Insertos de acuerdo con los requisitos lógicos. Llamamos a este proceso de procesamiento de datos una Acción. Las acciones son el puente entre datos y componentes y son una parte esencial de la construcción de una aplicación. Hemos dividido las acciones en dos categorías: acciones y transformadores.

### Acción SQL

#### Crear acción

El proceso de uso de acciones SQL se divide en los siguientes pasos:

1.  Primero, tienes que encontrar la lista de acciones en el editor de aplicaciones y hacer clic en el botón "Añadir".
2.  Luego, seleccione un tipo de fuente de datos en la casilla de selección de recurso. Puede encontrar los recursos creados en la lista de recursos correspondiente, también puede crear nuevos recursos aquí. Seleccione el recurso creado y haga clic en el botón "Confirmar". Una acción SQL se creará correctamente.
3.  La acción creada se mostrará en la lista de acciones. Un editor de acción se mostrará debajo del marco después de expulsar la acción. Puede cambiar el recurso de acciones, escribir sentencias de acción, contenido de respuesta, añadir transformadores, gestores de eventos.
4.  Después de completar la lista de acciones, puede hacer clic en el botón "Vista previa" para comprobar si la acción que ha escrito es válida. El panel de configuración mostrará el resultado de la acción. Después de editar la acción, puede hacer clic en el botón "Guardar" para finalizar el proceso. Si cambia a otra acción sin guardar la información, el sistema le pedirá que guarde la modificación.

#### Desglose de acciones

1.  Acción: Puede introducir sus sentencias de acción en el módulo Acción donde hemos añadido el editor Code Mirror para habilitar el resaltado de sentencias de acción y la finalización de código. En actualizaciones posteriores, proveeremos un modo de entrada GUI para escribir sentencias de acción de modo que los usuarios que no estén familiarizados con las sentencias de acción también puedan completar acciones de datos.
2.  Transformador: Puede refinar la lógica de acción con las sentencias JavaScript en el transformador, por ejemplo usando la función de filtro en JS en lugar de la función donde se encuentra la sentencia de acción.
3.  Controlador de eventos: Los manejadores de eventos le permiten activar acciones, controlar insertos y otras acciones en respuesta a la interacción del usuario. Puede agregar varios controladores de eventos para responder a un evento sin añadir sentencias JavaScript personalizadas. Por ejemplo, cuando un usuario hace clic en un botón, puede (1) disparar una acción para ejecutarse, mientras (2) se restablece el estado de múltiples listas desplegables y entradas de texto.
4.  Respuesta: En el módulo de Respuesta, puede personalizar las condiciones bajo las cuales la acción devuelve resultados incorrectos, de tal forma que se garantice la exactitud de la acción.
5.  Avanzado: Refina la configuración detallada de acciones tales como tiempos de espera de la base de datos, retrasos en el procesador de eventos, etc. dentro de las opciones avanzadas.

### Acciones API

#### Crear acción

El proceso para accionar datos usando la API es similar al de SQL.

1.  En primer lugar, el usuario necesita encontrar la lista de acciones en la parte inferior izquierda de la página de edición de la aplicación y hacer clic en el signo más en la parte superior derecha de la lista de acciones.
2.  A continuación, seleccione la lista de fuentes de datos correspondiente en el cuadro de selección de recursos que aparece. Puede encontrar el recurso creado en la lista de fuentes de datos correspondiente, o puede crear un nuevo recurso aquí. Seleccione el recurso creado, haga clic en "Crear" y una acción de la API se creará correctamente.
3.  La acción creada con éxito se mostrará en la lista de Acciones en el panel inferior y la pestaña General de la acción aparecerá debajo del marco cuando se haga clic. Puede hacer una solicitud a cualquier API HTTP en ILLA . Por favor, consulte la documentación de las APIs para los métodos de solicitud de datos.
4.  Una vez que hayas completado tu acción, puede hacer clic en el botón "Vista previa" para ver si la acción que ha escrito es válida y el panel de configuración le dará los resultados. Una vez que haya terminado de editar su acción, puede hacer clic en Guardar y salir para terminar el proceso. Si cambia a otra operación sin guardar, una ventana emergente le preguntará para completar la operación de guardar.

#### Desglose de acciones

1.  Transformador: Puede refinar la lógica de acción con sentencias JavaScript en un transformador, por ejemplo usando la función de filtro en JS en lugar de la función donde se encuentra una sentencia de acción.
2.  Controladores de eventos: Los manejadores de eventos le permiten activar acciones, controlar insertos y otras acciones en respuesta a la interacción del usuario. Puede agregar varios controladores de eventos para responder a un evento sin añadir sentencias JavaScript personalizadas. Por ejemplo, cuando un usuario hace clic en un botón, puede (1) disparar una acción para ejecutarse, mientras (2) se restablece el estado de múltiples listas desplegables y entradas de texto.
