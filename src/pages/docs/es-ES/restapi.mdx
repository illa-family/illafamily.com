## <h2 hidden>¿Qué es REST API?</h2>

En ILLA Cloud, la funcionalidad del API REST permite a los desarrolladores interactuar con sistemas externos, , y fuentes de datos utilizando el estilo arquitectónico de Transferencia Estatal (REST). Con la API REST de ILLA Cloud, los desarrolladores pueden integrar perfectamente sus aplicaciones de código bajo con otras aplicaciones, bases de datos o servicios.

La API REST en ILLA Cloud permite crear puntos finales personalizados que pueden manejar peticiones HTTP, GET, POST, PUT y DELETE. Esto permite a los desarrolladores realizar varias operaciones, incluyendo recuperar datos, enviar datos, actualizar registros y eliminar información.

### Crear API REST

Hay dos maneras de crear un recurso en Illa después de iniciar sesión en tu cuenta de Imax.

-   Crear en Recursos

Inicia sesión en tu cuenta de Imax, selecciona `**Recursos**` en la parte superior de la página y haz clic en el botón `**Crear nuevo**`.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Selecciona **`API REST`** de la lista de bases de datos.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Conecte a la base de datos con los parámetros requeridos descritos en `Configuración de Conexión` a continuación.

Haga clic en `**Probar conexión**` para ver si podemos conectar con éxito a la base de datos. En caso afirmativo, haga clic en **`Guardar Recursos`**, lo demás, compruebe dos veces el nombre de host, el puerto, el nombre de usuario y la contraseña son correctos.

Después de crear un recurso, el API REST listo se mostrará como mostrado.

![rest_ex_ready](https://cdn.illacloud.com/official-website/img/official-site/components/rest_ex_ready.jpg)

-   Crear en el Constructor

Inicia sesión en tu cuenta de Imax, crea un proyecto en Illa Builder en la página `**App**`, y navegar a la **`Lista de Acciones`** en la parte inferior de la página. Haz clic en `**nuevo**`, luego selecciona **`API REST`** de la lista de bases de datos. Luego, conecte a la base de datos con los parámetros requeridos descritos en `Configuración de Conexión` a continuación.

Haga clic en `**Probar conexión**` para ver si podemos conectar con éxito a la base de datos. En caso afirmativo, haga clic en **`Guardar Recursos`**, lo demás, compruebe dos veces el nombre de host, el puerto, el nombre de usuario y la contraseña son correctos.

### Configuración de conexión

Aquí tenemos que proporcionar información para conectarse a la base de datos de la API REST.

![rest_config](https://cdn.illacloud.com/official-website/img/official-site/components/rest_config.jpg)

| Propiedades        | Requerido | Descripción                                                                                              |
| ------------------ | --------- | -------------------------------------------------------------------------------------------------------- |
| Nombre             | requerido | El nombre del recurso al crear acciones en el ILLA.                                                      |
| URL base           | requerido | La URL o dirección IP de su base de datos                                                                |
| Parámetros de URL  | opcional  | proporcionar información adicional o modificar el comportamiento de las solicitudes API                  |
| Encabezados        | opcional  | metadatos adicionales sobre la solicitud o respuesta de la API                                           |
| Cookies            | opcional  | pequeños pedazos de datos que se almacenan en el lado del cliente y se envían con cada petición API      |
| Autenticación      | opcional  | verificar la identidad del usuario o aplicación haciendo la solicitud API                                |

### Crear acciones

Hemos creado un recurso REST API; podemos añadir la acción seleccionando REST API de la lista de acciones y eligiendo el botón **`Crear acción`** .

![rest_resource_list](https://cdn.illacloud.com/official-website/img/official-site/components/rest_resource_list.jpg)

Ahora hemos añadido el servidor REST API como una acción a nuestra página de construcción.

![rest](https://cdn.illacloud.com/official-website/img/official-site/components/rest.jpg)

### Configurar API REST

| Propiedades       | Descripción                                                                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| Tipo de acción    | GET, POST, PUT, HEAD, PATCH, DELETE, OPCIONES                                                                                               |
| Parámetro URL     | proporcionar información adicional o modificar el comportamiento de las solicitudes API si no se especifica en la configuración de conexión |
| Encabezados       | metadatos adicionales sobre la solicitud o respuesta de la API si no se especifica en la configuración de conexión                          |
| Transformador     | transforma los datos en el estilo que te gusta usando JavaScript                                                                            |

#### RECOGER

recuperar o recuperar datos de un servidor. Cuando se realiza una solicitud GET, el cliente especifica la URL del recurso al que quiere acceder, y el servidor responde con los datos solicitados.

#### POST

crear un nuevo recurso en el servidor. Permite a los clientes enviar datos al servidor para ser procesados y almacenados. Al enviar una solicitud POST, el cliente incluye los datos en el cuerpo de la solicitud.

#### PUT

actualizar o reemplazar un recurso existente. Se utiliza típicamente para modificar el estado o el contenido de un recurso identificado por una URL específica.

#### LIMPIO

similar al método GET, pero con una diferencia clave: el método HEAD sólo recupera las cabeceras de la respuesta, sin el cuerpo de la respuesta.

#### PATCH

actualizar un recurso específico. Se usa típicamente para hacer modificaciones parciales a un recurso existente sin reemplazar todo el recurso.

#### LIMPIO

solicitar la eliminación de un recurso específico o datos identificados por una URL determinada

### Caso de uso

Demostraremos cómo autorrellenar la información del formulario de datos de la API REST.

#### Paso 1 Añadir una acción

Vamos a crear una tabla en la API REST con las columnas `id`, `name` y `age`. El documento id 1 es lo que queremos mostrar en el componente de formulario.

![rest_get_all](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_all.jpg)

Entonces podemos crear una nueva acción para la API REST desde la lista de acciones de la aplicación que queremos construir, listando todos los datos en el cubo y llamado restapi1.

Selecciona `GET` para `**Tipo de acción**`. En parámetros de URL, pon `id` para la clave y `1` para el valor**.**

Haz clic en **`Guardar`** y **`Ejecutar`** para activar esta acción.

![rest_get_1](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_1.jpg)

#### Paso 2 Añadir componentes

Añade un componente **`form`** con el texto predeterminado "Forma" y un botón "Enviar" a lienzo.

Añade un componente **`input`**, un componente **`number input`** y un componente **`button`** de `**Insert**` en el formulario.

Etiqueta el componente de entrada a ser "Nombre", el componente de entrada de número a ser "Edad" y el componente de botón a ser "Auto Completar".

Establezca el marcador de posición del componente de entrada a "Por favor ponga su nombre aquí" y el valor por defecto de la entrada de número a 0 como se muestra a continuación.

![rest_layout](https://cdn.illacloud.com/official-website/img/official-site/components/rest_layout.jpg)

#### Paso 3 Configurar el componente

Para los componentes de "Auto relleno" **`button`** podemos configurarlo para establecer el valor en el componente **`input`**.

1.  Haga clic en el componente **`button`** para abrir su página de inspección. Bajo **`Controlador de eventos`**, haga clic en `**+ Nuevo**`.
2.  En el controlador de edición de eventos, seleccione **`componente de control`** en acción, y selecciona el **`input`** componente que quieres actualizar como objetivo del evento.
3.  Elige la acción **`setValue`** y establece el valor como `{{restapi1.data[0]["name"]}}`

![rest_get_input](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_input.jpg)

Del mismo modo, para los componentes **`button`** "Auto Fill" para establecer el valor en el componente **`number input`**.

1.  Bajo **`Controlador de eventos`**, haga clic en `**+ Nuevo**`.
2.  En el controlador de edición de eventos, seleccione **`componente de control`** en acción, y selecciona el **`número de entrada`** componente que quieres actualizar como objetivo del evento.
3.  Elige la acción **`setValue`** y establece el valor como `{{restapi1.data[0]["age"]}}`

![rest_get_num](https://cdn.illacloud.com/official-website/img/official-site/components/rest_get_num.jpg)

Para el componente "Enviar" **`botón`** podemos configurarlo para que envíe el formulario si el formulario está completado

1.  Haga clic en el componente **`button`** para abrir su página de inspección. Bajo **`Controlador de eventos`**, haga clic en `**+ Nuevo**`.
2.  En el controlador de edición de eventos, seleccione **`componente de control`** en acción, y seleccione el componente **`form`** que desea actualizar como objetivo del evento.
3.  Elegir el método **`submit`**

![rest_submit_config](https://cdn.illacloud.com/official-website/img/official-site/components/rest_submit_config.jpg)
