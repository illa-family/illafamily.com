## <h2 hidden>¿Qué es Redis?</h2>

Redis in Illa es una integración de base de datos que te permite conectar e interactuar con una base de datos Redis. Redis es un sistema de gestión de bases de datos relacionales de código abierto conocido por su velocidad, fiabilidad y facilidad de uso.

Con la integración de Redis en Illa, puede consultar, insertar, actualizar y eliminar datos de una base de datos Redis usando comandos SQL. También puede crear consultas SQL personalizadas y ejecutarlas dentro de Illa. Esto hace que sea fácil construir aplicaciones y tableros basados en datos que muestren datos en tiempo real desde una base de datos Redis.

### Crear Redis

Hay dos maneras de crear un recurso en Illa después de iniciar sesión en tu cuenta de Imax.

-   Crear en Recursos

Inicia sesión en tu cuenta de Imax, selecciona `**Recursos**` en la parte superior de la página y haz clic en el botón `**Crear nuevo**`.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Selecciona **`Redis`** de la lista de bases de datos.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Conecte a la base de datos con los parámetros requeridos descritos en `Configuración de Conexión` a continuación.

Haga clic en `**Probar conexión**` para ver si podemos conectar con éxito a la base de datos. En caso afirmativo, haga clic en **`Guardar Recursos`**, lo demás, compruebe dos veces el nombre de host, el puerto, el nombre de usuario y la contraseña son correctos.

Después de crear un recurso, los Redis listos se mostrarán como mostrados.

![redis_ex_ready](https://cdn.illacloud.com/official-website/img/docs/redis_ex_ready.png)

-   Crear en el Constructor

Inicia sesión en tu cuenta de Imax, crea un proyecto en Illa Builder en la página `**App**`, y navegar a la **`Lista de Acciones`** en la parte inferior de la página. Haz clic en `**nuevo**`, luego selecciona **`Redis`** de la lista de bases de datos. Luego, conecte a la base de datos con los parámetros requeridos descritos en `Configuración de Conexión` a continuación.

Haga clic en `**Probar conexión**` para ver si podemos conectar con éxito a la base de datos. En caso afirmativo, haga clic en **`Guardar Recursos`**, lo demás, compruebe dos veces el nombre de host, el puerto, el nombre de usuario y la contraseña son correctos.

### Configuración de conexión

Aquí tenemos que proporcionar información para conectarse a la base de datos Redis.

![redis_config](https://cdn.illacloud.com/official-website/img/docs/redis_config.png)

| Propiedades             | Requerido | Descripción                                                                                                                                   |
| ----------------------- | --------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Nombre                  | requerido | El nombre del recurso al crear acciones en el ILLA.                                                                                           |
| Hostname                | requerido | La URL o dirección IP de su base de datos                                                                                                     |
| Puerto                  | requerido | El número de puerto del servidor que debe utilizar para conectarse. Si no especifica un puerto, el puerto por defecto es '6379'.              |
| Índice de base de datos | opcional  | una estructura de datos que se utiliza para optimizar el rendimiento de las consultas que recuperan datos de una base de datos Redis          |
| Base de datos           | requerido | El nombre de la base de datos                                                                                                                 |
| Usuario                 | requerido | el nombre de usuario que desea utilizar al iniciar sesión en el servidor Redis.                                                               |
| Contraseña              | requerido | Utilice esta contraseña para la autenticación.                                                                                                |
| Opciones SSL            | opcional  | decide hasta qué punto se prioriza una conexión segura TCP/IP SSL mientras se negocia con el servidor.                                        |

### Crear acciones

Hemos creado un recurso Redis, podemos añadir la acción seleccionando Redis de la lista de acciones y eligiendo el botón **`Crear acción`**.

![redis_resource_list](https://cdn.illacloud.com/official-website/img/docs/redis_resource_list.png)

Ahora hemos añadido el servidor Redis como una acción a nuestra página de construcción.

![redis](https://cdn.illacloud.com/official-website/img/docs/redis.png)

### Configurar Redis

| Propiedades    | Descripción                                                          |
| -------------- | -------------------------------------------------------------------- |
| consulta       | comandos para recuperar datos de la base de datos                    |
| Transformador  | transforma los datos en el estilo que te gusta usando JavaScript     |

Ejemplo:

-   TECLAS

Este comando recupera una lista de todas las claves de la base de datos Redis. Puede utilizar comodines para filtrar las teclas por patrón.

```sql
KEYS *
KEYS user:*
```

-   RECOGER

Este comando recupera el valor asociado con una clave determinada.

```sql
GET user:12345
```

-   HGETALL

Este comando recupera todos los campos y sus valores de una estructura de datos de Redis hash.

```sql
HGETALL user:12345
```

-   ZRANGE

Este comando recupera un rango de valores de una estructura de datos ordenada por Redis, basada en sus partituras.

```sql
ZRANGE high_score 0 9 WITHSCORES
```

-   LANGO

Este comando recupera un rango de valores de una estructura de datos de lista Redis.

```sql
LRANGE my_list 0 4
```

-   EXISTOS

Este comando comprueba si existe una clave dada en la base de datos de Redis.

```sql
EXISTS user: 12345
```

-   TTL

Este comando recupera el valor de tiempo en vivo (TTL) asociado con una clave, que especifica cuánto tiempo permanecerá la clave en la base de datos antes de que caduque.

```sql
TTL user:12345
```
