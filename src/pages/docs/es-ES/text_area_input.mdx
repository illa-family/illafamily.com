## <h2 hidden>¿Qué es la entrada de área de texto?</h2>

El componente de entrada de área de texto es un elemento de interfaz de usuario que permite a los usuarios entrar y editar el área de texto en un formulario o campo de entrada.

### Propiedades

| Propiedades                   | Descripción                                                                                                                                                          |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Valor por defecto             | El valor inicial del componente. Puede cambiar dinámicamente el valor inicial escribiendo JavaScript en {{}}.                                                        |
| Marcador de posición          | El valor se mostrará cuando el campo de entrada esté vacío.                                                                                                          |
| Etiqueta                      | El nombre del campo mostrado al usuario                                                                                                                              |
| Leyenda                       | Un título usado para describir el campo en detalle                                                                                                                   |
| Etiqueta oculta               | Establecer si mostrar o no la etiqueta                                                                                                                               |
| Posición                      | Define la posición de la etiqueta relativa al componente                                                                                                             |
| Alineación                    | Establecer el alineamiento (alinear a la izquierda o a la derecha) de la etiqueta                                                                                    |
| Width                         | Cuando la etiqueta esté en el lado izquierdo del componente, establezca la proporción de ancho de la etiqueta.                                                       |
| Manejador del Evento          | Activar consultas, componentes de control o llamar a otras APIs en respuesta a eventos de componentes.                                                               |
| Deshabilitado                 | Controla el estado de si el componente está desactivado. El componente no puede ser modificado o enfocado cuando está desactivado.                                   |
| Sólo lectura                  | Controlar el estado de si el componente es de sólo lectura. Un componente de sólo lectura puede ser seleccionado y enfocado, pero no puede ser modificado.           |
| Mostrar botón de borrar       | controlar si se muestra o no un botón claro en el campo de entrada                                                                                                   |
| Mostrar conteo de caracteres  | controlar si se muestra o no el conteo de caracteres de la entrada                                                                                                   |
| Sugerencia                    | Los usuarios pueden ingresar la descripción del componente aquí. La descripción se mostrará cuando esté enfocada. Formato Markdown soportado.                        |
| Campo requerido               | Válido sólo cuando el interruptor está encendido.                                                                                                                    |
| Regex                         | especificar un patrón de expresión regular que el valor de entrada debe coincidir                                                                                    |
| Longitud máxima               | establece el número máximo de caracteres que se pueden introducir en el campo de entrada.                                                                            |
| Longitud mínima               | especificar el número mínimo de caracteres que el usuario debe introducir en el campo de entrada para que la entrada se considere válida.                            |
| Regla personalizada           | Crea aquí tu lógica de validación. Las reglas deben ser hechas en JavaScript y cubiertas por {{}}.                                                                   |
| Ocultar mensaje de validación | Puede ocultar el mensaje de error cambiando el estado oculto cuando el valor de entrada es incorrecto. Puedes cambiar dinámicamente el estado oculto por JavaScript. |
| Clave de Formulario           | Especifique una clave de un componente de formulario de envoltura al construir el atributo de datos.                                                                 |
| Altura                        | especificar el tamaño o altura vertical del campo de entrada                                                                                                         |
| Hidden                        | Controlar dinámicamente si el componente está oculto. Puede cambiar el estado oculto a través de un valor booleano dinámico.                                         |
| Color del tema                | Permite a los usuarios especificar el color de fondo y opacidad del botón                                                                                            |

### Método

Puede usar otros componentes para controlar el componente. Apoyamos los dos métodos siguientes:

-   **setValue**

Para establecer el valor de entrada del área de texto, por ejemplo, {{'value1'}}

| Propiedades | Descripción |
| ----------- | ----------- |
| Valor       | Input value |

-   **limpiarValor**

Limpiar el valor del componente seleccionado 

-   **focus**

Cuando se llama al método de enfoque, el campo de entrada del área de texto será resaltado y listo para que el usuario comience a escribir sin la necesidad de que el usuario haga clic en el campo de entrada del número

### Controlador de eventos

| Evento  | Descripción                                                                              |
| ------- | ---------------------------------------------------------------------------------------- |
| Cambiar | Cuando un usuario cambia el valor de entrada                                             |
| Enfoque | Cuando un usuario mueve el cursor del ratón sobre el componente de entrada               |
| Blur    | Cuando un usuario ha terminado de introducir valor y dejar de centrarse en el componente |

### Datos

El componente tiene algunos datos comúnmente utilizados, que pueden llamarse a través de **`{{componentName.propertyName}}`** en la aplicación.

| Nombre de la propiedad      | Descripción                                                                                                        |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| valor                       | valor de entrada de usuario                                                                                        |
| esquema de color            | color de fondo del botón                                                                                           |
| regla personalizada         | regla definida por el usuario o lógica de validación que se puede aplicar al componente de entrada de números      |
| deshabilitado               | un valor booleano indica el estado deshabilitado                                                                   |
| nombre de pantalla          | el nombre de este componente (es decir, botón1)                                                                    |
| altura dinámica             | permite que la altura del área de texto se ajuste dinámicamente basado en el contenido introducido por el usuario. |
| eventos                     | generado o activado cuando ocurren ciertas acciones o eventos                                                      |
| formDataKey                 | la clave de datos de formulario de entrada                                                                         |
| hidden                      | estado oculto (verdadero o falso)                                                                                  |
| etiqueta                    | valor de etiqueta                                                                                                  |
| alineación de etiqueta      | alineación de la etiqueta de entrada (izquierda o derecha)                                                         |
| posición de etiqueta        | posición de la etiqueta de entrada (izquierda o derecha)                                                           |
| ancho de etiqueta           | el entero que representa el ancho de la etiqueta.                                                                  |
| longitud máxima             | el valor de longitud máxima                                                                                        |
| minLongitud                 | el valor de la longitud mínima                                                                                     |
| marcador de posición        | valor del marcador                                                                                                 |
| sólo lectura                | un valor booleano indica el estado readyOnly de la entrada.                                                        |
| requerido                   | un valor booleano indica el estado requerido de la entrada.                                                        |
| cambiar tamaño de dirección | le permite controlar el comportamiento de cambio de tamaño del área de texto                                       |
| regex                       | aplicar la validación basada en patrones al texto de entrada en el área de texto                                   |
| tooltipText                 | valor del texto del tooltip                                                                                        |

Ejemplo: {{textarea1.value}}

### Caso de uso

A continuación, demostraremos cómo mapear los datos de la fuente de datos a la entrada del área de texto con un botón. 

#### Paso 1 Añadir una acción

Creemos una tabla en Supabase llamada `Project`.  Hay dos atributos en Project: `id` y `name`. El `nombre` es el valor que queremos asignar a nuestra entrada de área de texto.

Luego podemos crear una nueva acción para Suabase desde la lista de acciones, listando todos los datos en el proyecto y nombres como `supabasedb2.`

Haz clic en **`Guardar`** y **`Ejecutar`** para activar esta acción.

![text_area_data](https://cdn.illacloud.com/official-website/img/official-site/components/text_area_data.jpg)

#### Paso 2 Añadir componentes

A continuación, podemos añadir un componente de **`área de texto`** y un componente **`botón`** al lienzo. 

Establecemos el valor por defecto del área de texto para ser 'Ninguna' y marcamos el botón como 'Establecer valor' como se muestra a continuación

![text_area_look](https://cdn.illacloud.com/official-website/img/official-site/components/text_area_look.jpg)

#### Paso 3 Configurar el componente

Para el componente **`button`**, podemos configurarlo para establecer el valor en el componente **`entrada de área de texto`** a los datos del documento que leemos desde supabasedb2.

1.  En el controlador de edición de eventos, seleccione **`componente de control`** en acción, y seleccione el componente **`textarea1`** que desea actualizar como objetivo del evento.
2.  Elige la acción **`setValue`** y establece el valor como `{{s31.data}}` 

![text_area_config](https://cdn.illacloud.com/official-website/img/official-site/components/text_area_config.jpg)

#### Prueba de paso 4

Ahora que al hacer clic en el botón 'Establecer valor', el valor debe cambiar a los nombres.

![text_area_test](https://cdn.illacloud.com/official-website/img/official-site/components/text_area_test.jpg)
