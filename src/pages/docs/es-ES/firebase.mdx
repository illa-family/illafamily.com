## <h2 hidden>¿Qué es Firebase?</h2>

Firebase es una plataforma completa proporcionada por Google que ofrece una amplia gama de herramientas y servicios para el desarrollo y la gestión de aplicaciones web y móviles. Proporciona a los desarrolladores una infraestructura de backend escalable y segura, junto con un conjunto de funciones y funcionalidades prediseñadas para acelerar el proceso de desarrollo. Con la integración de Firebase en ILLA Cloud, los desarrolladores pueden mejorar sus aplicaciones con soluciones de almacenamiento confiables y escalables, para que se concentren en la creación de experiencias innovadoras y basadas en datos.

### Crear API Firebase

Hay dos maneras de crear un recurso en Illa después de iniciar sesión en tu cuenta de Imax.

-   Crear en Recursos

Inicia sesión en tu cuenta de Imax, selecciona `**Recursos**` en la parte superior de la página y haz clic en el botón `**Crear nuevo**`.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Selecciona **`Firebase`** de la lista de API.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Conecte a la base de datos con los parámetros requeridos descritos en `Configuración de Conexión` a continuación.

Haga clic en `**Probar conexión**` para ver si podemos conectar con éxito a la base de datos. En caso afirmativo, haga clic en **`Guardar Recursos`**, lo demás, compruebe dos veces el nombre de host, el puerto, el nombre de usuario y la contraseña son correctos.

Después de crear un recurso, el Firebase listo se mostrará como mostrado.

![firebase_ex_ready](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase_ex_ready.jpg)

-   Crear en el Constructor

Inicia sesión en tu cuenta de Imax, crea un proyecto en Illa Builder en la página `**App**`, y navegar a la **`Lista de Acciones`** en la parte inferior de la página. Haz clic en `**nuevo**`, luego selecciona **`Firebase`** de la lista de bases de datos. Luego, conecte a la base de datos con los parámetros requeridos descritos en `Configuración de Conexión` a continuación.

Haga clic en `**Probar conexión**` para ver si podemos conectar con éxito a la base de datos. En caso afirmativo, haga clic en **`Guardar Recursos`**, lo demás, compruebe dos veces el nombre de host, el puerto, el nombre de usuario y la contraseña son correctos.

### Configuración de conexión

Aquí tenemos que proporcionar información para conectarse a Firebase API.

![firebase_config](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase_config.jpg)

| Propiedades                      | Requerido | Descripción                                                                                                 |
| -------------------------------- | --------- | ----------------------------------------------------------------------------------------------------------- |
| Nombre                           | requerido | El nombre del recurso al crear acciones en el ILLA.                                                         |
| URL de la base de datos Firebase | opcional  | el endpoint o URL donde la base de datos de tiempo real Firebase está alojada                               |
| ID del proyecto Firestore        | opcional  | un identificador único asignado a cada proyecto Firebase que utiliza el servicio de base de datos Firestore |
| Clave Privada                    | requerido | una credencial de seguridad usada para autenticar y acceder a los servicios de Firebase programáticamente   |

### Crear acciones

Hemos creado un recurso Firebase, podemos añadir la acción seleccionando Firebase de la lista de acciones y eligiendo el botón **`Crear acción`**.

![firebase_resource_list](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase_resource_list.jpg)

Ahora hemos añadido el servidor Firebase como una acción a nuestra página de construcción.

![firebase](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase.jpg)

#### Obtener usuario por UID

recuperar la información del usuario especificando el identificador único del usuario (UID). Le permite obtener detalles del usuario como el nombre de la pantalla, el correo electrónico, el número de teléfono y los atributos personalizados asociados con el usuario.

** Input **

| Propiedades | Requerido | Descripción                                                                                                            |
| ----------- | --------- | ---------------------------------------------------------------------------------------------------------------------- |
| UID         | requerido | "Identificador de usuario." Es un identificador único asignado a cada usuario en el sistema de autenticación Firebase. |

#### Obtener usuario por email

recuperar la información del usuario especificando la dirección de correo electrónico del usuario. Permite recuperar datos de usuario similares al método basado en UID, pero utilizando el correo electrónico como identificador.

** Input **

| Propiedades | Requerido | Descripción                                                                  |
| ----------- | --------- | ---------------------------------------------------------------------------- |
| email       | requerido | el correo electrónico registrado asociado con el usuario que desea recuperar |

#### Obtener usuario por número de teléfono

recuperar la información del usuario especificando el número de teléfono del usuario. Le permite obtener detalles del usuario similares al método basado en UID, pero usando el número de teléfono como identificador.

** Input **

| Propiedades        | Requerido | Descripción                                                               |
| ------------------ | --------- | ------------------------------------------------------------------------- |
| número de teléfono | requerido | el número de teléfono registrado asociado al usuario que desea recuperar  |

#### Crear un usuario

crear un nuevo usuario en el sistema de autenticación Firebase. Puede proporcionar el correo electrónico del usuario, contraseña, nombre para mostrar y otra información relevante durante el proceso de creación del usuario.

** Input **

| Propiedades    | Requerido | Descripción                                    |
| -------------- | --------- | ---------------------------------------------- |
| Objeto usuario | requerido | un objeto que contiene información del usuario |

#### Actualizar un usuario

actualizar los atributos y propiedades de un usuario existente. Puede modificar el nombre del usuario, el correo electrónico, el número de teléfono, la contraseña y otros atributos personalizados asociados con el usuario.

** Input **

| Propiedades    | Requerido | Descripción                                                                                                            |
| -------------- | --------- | ---------------------------------------------------------------------------------------------------------------------- |
| UID            | requerido | "Identificador de usuario." Es un identificador único asignado a cada usuario en el sistema de autenticación Firebase. |
| Objeto usuario | requerido | un objeto que contiene información del usuario                                                                         |

#### Eliminar un usuario

borrar un usuario del sistema de autenticación Firebase. Esta acción elimina permanentemente al usuario y sus datos asociados del sistema.

** Input **

| Propiedades | Requerido | Descripción                                                                                                            |
| ----------- | --------- | ---------------------------------------------------------------------------------------------------------------------- |
| UID         | requerido | "Identificador de usuario." Es un identificador único asignado a cada usuario en el sistema de autenticación Firebase. |

#### Listar usuarios

recuperar una lista de todos los usuarios registrados en el sistema de autenticación Firebase. Proporciona resultados paginados, permitiéndole iterar a través de la lista de usuarios y recuperar detalles específicos del usuario.

** Input **

| Propiedades               | Requerido | Descripción                                                                              |
| ------------------------- | --------- | ---------------------------------------------------------------------------------------- |
| Longitud de la lista      | opcional  | número de usuarios que desea recuperar en una sola solicitud                             |
| Siguiente token de página | opcional  | paginación cuando hay más usuarios de los que pueden ser devueltos en una sola respuesta |

#### Busca Firebase

realizar consultas en la base de datos utilizando varias condiciones y filtros

** Input **

| Propiedades           | Requerido | Descripción                                                                            |
| --------------------- | --------- | -------------------------------------------------------------------------------------- |
| Colección             | requerido | puede usar el menú desplegable para seleccionar o usar un id crudo                     |
| donde                 | requerido | añadir condiciones a su consulta                                                       |
| límite                | opcional  | limitar el número de documentos devueltos por la consulta.                             |
| Ordenar por           | opcional  | especificar el campo por el cual desea ordenar los resultados de la consulta           |
| Dirección del pedido  | opcional  | la dirección del pedido en los resultados de la consulta                               |

#### Insertar documento

crear un nuevo documento en una colección o subcolección dentro de Firestore. Usted proporciona los datos que se almacenarán en el documento, y Firebase le asigna un identificador único (ID del documento).

** Input **

| Propiedades      | Requerido | Descripción                                                                                                                                          |
| ---------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| Colección        | requerido | especifica el nombre de la colección en la que desea insertar el documento. puede usar el menú desplegable para seleccionar o usar un id crudo       |
| ID del documento | opcional  | especifica un identificador personalizado para el documento que se está insertando                                                                   |
| valor            | requerido | los datos reales que desea insertar en el documento                                                                                                  |

#### Actualizar Documento

actualizar un documento existente en Firestore. Especifica el ID del documento y proporciona los datos actualizados, que sobrescribirán los campos del documento existente.

** Input **

| Propiedades      | Requerido | Descripción                                                                                                                                              |

| Colección        | requerido | especifica el nombre de la colección en la que desea actualizar el documento. puede usar el menú desplegable para seleccionar o usar un id crudo         |
| ID del documento | opcional  | especificar un identificador personalizado para el documento que se está actualizando                                                                    |
| valor            | requerido | los datos reales que desea actualizar al documento                                                                                                       |

#### Consulta base de datos

consultar la base de datos Realtime para recuperar datos que cumplen criterios específicos. Puede definir filtros, ordenar y otras condiciones para recuperar un subconjunto de datos de una ubicación particular en la base de datos.

** Input **

| Propiedades          | Requerido | Descripción                                                    |
| -------------------- | --------- | -------------------------------------------------------------- |
| Ref de base de datos | requerido | una referencia a una ubicación específica en la base de datos. |

#### Establecer datos

escribir o reemplazar datos en una ubicación específica en la base de datos en tiempo real

** Input **

| Propiedades           | Requerido | Descripción                                                                                                                     |
| --------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------- |
| Ref de base de datos  | requerido | una referencia a una ubicación específica en la base de datos.                                                                  |
| Objeto a definir      | requerido | contiene los datos que queremos escribir en la base de datos, incluyendo el nombre, la edad y los campos de correo electrónico. |
