## <h2 hidden>Hva er Grunne?</h2>

En cascader er en type inngangskomponent som lar brukere velge hierarkiske data, for eksempel en kategori eller underkategori fra en nedtrekksmeny. Når en bruker velger et alternativ fra nedtrekksmenyen, cascader vil automatisk oppdatere for å vise relevante underkategorier, eller alternativer i en nestet nedtrekksmeny.

Hvis du for eksempel har en saksholder for å velge en produktkategori, den første rullegardinmenyen kan vise alle de tilgjengelige kategoriene som "Elektronikker" og "Books". Når en bruker velger "Elektroniker", vil cascaderen oppdatere for å vise underkategoriene "Phones", "Laptops" og "Tablets". Dette gjør at brukerne raskt og enkelt kan navigere gjennom hierarkiske data og velge det alternativet de trenger.

### Egenskaper

Kanyler tilgjengelige egenskaper. JavaScript kan skrives til å lese eller endre informasjon om komponenter.

| Egenskaper            | Beskrivelse                                                                                                                                                                                                       |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Data kilde            | Hvert alternativ må være JSON-format og inneholde to nøkler, verdi og etikett. Ved hjelp av matriser kan du opprette alternativer på samme nivå. Og underalternativer kan opprettes ved å bruke "barn"-attributt. |
| Standard verdi        | Den opprinnelige verdien for komponenten. Du kan dynamisk endre innledende verdien ved å skrive JavaScript i {{}}.                                                                                                |
| Plassholder           | Verdien vil bli vist når inndatafeltet er tomt.                                                                                                                                                                   |
| Etikett               | Navnet på feltet som vises til brukeren                                                                                                                                                                           |
| Bildetekst            | Bildetekst som brukes til å beskrive feltet i detalj                                                                                                                                                              |
| Skjult etikett        | Angi om etiketten skal vises                                                                                                                                                                                      |
| Posisjon              | Still plasseringen av etiketten i forhold til komponenten                                                                                                                                                         |
| Justering             | Sett justeringen (justert til venstre eller høyre) for etiketten                                                                                                                                                  |
| Width                 | Når etiketten er på venstre side av komponenten, angir du forholdet bredden på etiketten.                                                                                                                         |
| Arrangementshåndterer | Utløse spørringer, kontroll-komponenter eller ringe andre APIer som respons på komponenthendelser.                                                                                                                |
| Deaktivert            | Kontroller statusen for hvorvidt komponenten er deaktivert. Komponenten kan ikke endres eller fokusert når den er deaktivert.                                                                                     |
| Kun lese              | Kontroller statusen for om komponenten bare er skrivebeskyttet. En skrivebeskyttet komponent kan velges og fokuseres men kan ikke endres.                                                                         |
| Vis null-knapp        | kontrollere om en tydelig knapp vises i inntastingsfeltet eller ikke                                                                                                                                              |
| Utvidelse metode      | Angi hvordan cascader blir utvidet (ved å klikke eller hovre)                                                                                                                                                     |
| Verktøytips           | Brukere kan skrive inn komponentens verktøytips her. Verktøytuppen vil bli vist når den er fokusert. Markdown formatet støttes                                                                                    |
| Hidden                | Dynamisk kontrollere om komponenten er skjult. Du kan endre skjult status gjennom dynamisk boolsk verdi.                                                                                                          |

### Metoder

Du kan bruke andre komponenter til å kontrollere komponenten. Vi støtter følgende to metoder:

-   **setValue**

For å sette det valgte alternativet, for eksempel, {{”value1”}}

| Egenskaper | Beskrivelse |
| ---------- | ----------- |
| Verdi      | Input value |

-   **clearValue**

For å fjerne verdien for den valgte komponenten 

Eksempel Bruk:

Lytte til 'onChange'-hendelsen til Cascader av andre komponenter ved hjelp av innebygd hendelsessystem. Sette det opp med å følge disse eksemplene:

1.  Legg til en hendelse som utløser på komponenten du vil lytte til. Hvis du for eksempel vil lytte til **`onChange`** hendelsen til en Radiorruppekomponent, du ville legge til en hendelse utløser til den Radio-gruppe-komponenten.
2.  I Redigeringshendelsesbehandleren velger du **`Kontrollkomponent`** i handlingen, velg **`ascader`** komponenten du vil oppdatere etter som målet til hendelsen.
3.  Velg handlingen \*\*`SetValue`, og velg den riktige verdien for Skolleserkomponenten. Det vil være verdien som Sileserkomponenten har og vil oppdatere når hendelsen aktiveres.
4.  Lagre hendelsesutløserinnstillingene og gjenta prosessen for andre komponenter som du vil lytte til.

Når du har satt opp hendelsen utløsere, oppdateres kataloggerens inngangskomponent automatisk når **`onChange`** hendelsen utløses på de andre komponentene. Dette lar deg opprette dynamiske grensesnitt som svarer på inndata fra bruker i sanntid, gjøre det enklere for brukerne å navigere og samhandle med søknaden.

| -------- | ---------------------------------------------------------------------------------------- |
| Endre    | Når en bruker endrer den valgte alternativverdien                                        |
| Fokus    | Når en bruker flytter musepekeren på cascader-komponenten                                |

### Data

Komponenten har noen vanlig brukte data, som kan kalles via **`{{componentName.propertyName}}`** i appen.

| Egenskap navn   | Beskrivelse                                           |
| --------------- | ----------------------------------------------------- |
| dataSourceMode  | modusen til datakilden i cascader (standard dynamisk) |
| displayNavn     | navnet på denne komponenten (dvs cascader1)           |
| utviderTrigger  | utløserhendelser (klikk)                              |
| verdi           | Valgt verdi                                           |
| etikett         | etikett verdi                                         |
| labelJuster     | justering av kasseetikett (venstre eller høyre)       |
| etikettposisjon | stilling til kasseetikett (venstre eller høyre)       |
| etikettbredde   | heltall som representerer bredden på etiketten        |
| plassholder     | plassholder verdi                                     |

Eksempel: {{cascader1.value[0]}
