## <h2 hidden>Hugging Gesichts-API</h2>

Hugging Face ist der Github der Maschinenlern-Community mit Hunderttausenden vorausgebildeten Modellen und 10.000 Datensätzen. Sie können frei auf Modelle und Datensätze anderer Branchenexperten zugreifen oder Ihre Modelle auf Hugging Face.

Einige Beispiele für Modelle, die in der Hugging Face Bibliothek verfügbar sind, sind:

1.  BERT (Bidirectional Encoder Representations from Transformers): BERT ist ein Transformer-basiertes Modell, das Google für verschiedene NLP-Aufgaben entwickelt hat. Sie hat im Bereich des Sprachverständnisses und der maschinellen Übersetzung hochmoderne Ergebnisse erzielt.
2.  GPT (Generativer Pre-Training Transformer): GPT ist ein weiteres Transformer-basiertes Modell, das von OpenAI entwickelt wurde. Es wird in erster Linie für die Erstellung von Sprachen wie Übersetzung und Zusammenfassung von Texten verwendet.
3.  RoBERTa (Robustly Optimized BERT): RoBERTa ist eine Erweiterung des BERT-Modells, das entwickelt wurde, um die Leistung von BERT bei verschiedenen NLP-Aufgaben zu verbessern.
4.  XLNet (eXtraordinary LanguageNet): XLNet ist ein Transformer-basiertes Modell, das von Google entwickelt wurde, um die Leistung von Transformatormodellen beim Sprachverstehen zu verbessern.
5.  ALBERT (A Lite BERT): ALBERT ist eine Version des BERT-Modells, das entwickelt wurde, um effizienter und schneller trainieren zu können und dabei eine gute Performance bei NLP-Aufgaben zu erhalten.

### Was du mit Hugging Face im ILLA Builder machen kannst

In Hugging Face, sind über 130.000 Machine-Learning-Modelle über die öffentliche API verfügbar, die du kostenlos unter <https://huggingingface.co/models> verwenden und testen kannst. Wenn du eine Lösung für Produktionsszenarien benötigst, kannst du außerdem die Inferenz-Endpunkte von Hugging Face verwenden, die unter <https://huggingingface.co/docs/inference-endpoints/index> eingesetzt werden können.

ILLA Builder bietet Dutzende von gängigen Frontend-Komponenten an, die es Ihnen ermöglichen, verschiedene Frontend-Schnittstellen auf Basis Ihrer spezifischen Bedürfnisse schnell zu erstellen. Gleichzeitig bietet ILLA eine Verbindung zu Hugging Face, die es Ihnen ermöglicht, sich schnell mit der API zu verbinden, Anfragen zu senden und zurückgegebene Daten zu erhalten. Durch das Verbinden der API und der Front-End-Komponenten können Sie die Anforderung implementieren, dass Benutzer Inhalte über das Frontend eingeben und an die API senden können. Die API gibt den generierten Inhalt zurück, der im Frontend angezeigt wird.

### Hugging Face-API-Ressource konfigurieren

| Eigenschaften | Benötigt | Beschreibung                                                                                                |
| ------------- | -------- | ----------------------------------------------------------------------------------------------------------- |
| Name          | benötigt | Legen Sie einen Ressourcennamen fest, der für die Anzeige in der ILLA verwendet wird                        |
| Token         | benötigt | Der Benutzerzugriff oder API-Token. Du kannst es unter https://huggingingface.co/settings/tokens bekommen.  |

### Aktion konfigurieren

| Eigenschaften | Benötigt | Beschreibung                                                                                                                                                                                                                                                    |
| ------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Modell-ID     | benötigt | Modelle suchen: https://huggingingface.co/models                                                                                                                                                                                                                |
| Parametertyp  | benötigt | Der Parametertyp des Endpunkts. Wenn zum Beispiel der Endpunkt eine Texteingabe benötigt, geben Sie den Parameter „Eingabe“ mit Text ein. Wenn Ihr Endpunkt eine JSON-Eingabe benötigt, geben Sie den Parameter „Eingabetaste“ mit JSON oder Schlüsselwert ein. |
| Parameter     | benötigt | Geben Sie Ihren Parameter ein. Verwenden Sie {{ componentName.value }} , um Daten von Komponenten zu verwenden.                                                                                                                                                 |

# Verwendung von Hugging Face im ILLA Builder

## Usecase 1

### Schritt 1: Build UI mit Komponenten auf ILLA Builder

Hier zeigen wir, wie man einen einfachen Text erstellt, der Text-Eingabeaufforderung und eine Text-Frage eingibt, um eine Textantwort auszugeben. 

Sie benötigen zwei Komponenten im Textbereich, die als "Originaltext eingeben" und "Hier die Frage eingeben", und Schaltflächenkomponente als "Ausführen" und als "Anwer" für Ausgabetext bezeichnete Textbereichskomponente. Das folgende Bild ist ein Beispiel wie oben beschrieben.

![hfapi](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi1.png)

### Schritt 2: Erstelle eine Hugging Face Ressource und konfiguriere die Aktionen

Klicken Sie + Neu in der Aktionsliste und wählen Sie Hugging Face Inference API.

![hfapi2](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi2.png)

Füllen Sie das Formular aus, um sich mit Ihrem Hugging Gesicht zu verbinden:

Name: Der Name, der in ILLA angezeigt wird

Token: Hol dir in deinem Hugging Gesicht [Profileinstellungen](https://huggingingface.co/settings/tokens)

![hfapi3](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi3.png)

Bestätigen Sie die Modellinformationen in Hugging Face bevor Sie die Aktionen konfigurieren:

Wähle ein Modell in [Hugging Face Model Page](https://huggingingface.co/models)

Lass uns [deepset/roberta-base-squad2](https://huggingingface.co/deepset/roberta-base-squad2) als Beispiel verwenden. Geben Sie die Detailseite ein > Deploy > Click Inference API

![hfapi4](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi4.png)

Die Parameter nach \`“inputs” sind das, was du ILLA ausfüllen solltest. 

![hfapi5](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi5.png)

Im ILLA Builder sollten wir die Modell-ID und Parameter ausfüllen. Wenn man das obige Modell als Beispiel nimmt, ist das \`“inputs”-Paar ein Schlüssel-Wert-Paar, so dass wir es mit Schlüssel-Wert oder JSON füllen können. 

![hfapi6](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi6.png)

![hfapi7](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi7.png)

Und wir unterstützen auch die Texteingabe und Binäreingabe, die alle Hugging Face Inference API-Verbindungen erfüllen können.

### Schritt 3: Aktionen mit Komponenten verbinden

Um die Benutzereingabe an die API zu übergeben, können Sie {{ verwenden, um die in der Komponente eingegebenen Daten abzurufen. Zum Beispiel ist input2 die Komponente, um die Frage einzugeben und input1 ist die Komponente zum Input-Kontext, wir können `question` und `context` im Schlüssel ausfüllen und `{{ input.value }}` im Wert ausfüllen:

```jsx
{
"Frage": {{input2.value}},
"context": {{input1.value}}
}
```

Bevor die Ausgabedaten der Aktion in der Front-End-Komponente angezeigt werden, sollten wir bestätigen, in welchem Feld die Ausgabe verschiedener Modelle platziert ist. Nehmen Sie immer noch `deepset/roberta-base-squad2` als Beispiel, die Ergebnisse sind wie folgt:

![hfapi8](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi8.png)

Also können wir die Antwort mit `{{ textQuestion.data[0].answer }}`(Die `textQuestion` ist der Name der Aktion). 

![hfapi9](https://cdn.illacloud.com/official-website/img/docs/connect/hfapi9.png)

## Usecase 2

Hier zeigen wir, wie man das Stable Diffusionsmodell thorugh der Hugging Face API in Illa Cloud verwendet.

**Schritt 1: Baue eine Frontend-Schnittstelle**

Mit Hilfe eines Drag-and-Drop Ansatzes stellen wir wesentliche Komponenten wie Eingabefelder, Tasten, Bilder und vieles mehr zusammen. Nach der Anpassung der Stile der Komponenten erhalten wir die folgende vollständige Webseite.

![hugging_layout](https://cdn.illacloud.com/official-website/img/official-site/components/hugging_layout.png)

**Schritt 2: Erstelle Ressourcen und Aktionen**

Wir legen Ressourcen und Aktionen fest, indem wir die Hugging Face Inference API nutzen, um eine Verbindung zum Stable Diffusion Modell herzustellen. Zwei Modelle können verwendet werden: `runwayml/stable-diffusion-v1-5` und `stabilityai/stable-diffusion-2-1`.

Wählen Sie dazu die "Hugging Face Inference API" aus.

![action_list](https://cdn.illacloud.com/official-website/img/official-site/components/action_list.png)

Geben Sie einen Namen für diese Ressource ein und geben Sie Ihren Token von der Hugging Face-Plattform ein.

![hugging_config](https://cdn.illacloud.com/official-website/img/official-site/components/hugging_config.png)

Bitte geben Sie im Aktionen-Konfigurationsfenster die Modell-ID und den Parameter ein. Wir werden das ausgewählte Modell von radioGroup1 abrufen. Füllen Sie also die Model ID als `{{radioGroup1.value}}` aus. Für die Eingabe, da sie aus dem Eingabefeld abgerufen wird, geben Sie den Parameter als `{{input1.value}} ` ein. Die Konfiguration sollte wie im folgenden Bild gezeigt werden.

![hugging_demo](https://cdn.illacloud.com/official-website/img/official-site/components/hugging_demo.png)

Wir versuchen "Ein Mecha-Roboter in einer Favela im expressionistischen Stil" in die "input"-Komponente einzugeben und die Aktion auszuführen. Die resultierende Ausführung ist wie folgt. Vom linken Fenster aus können Sie die verfügbaren Daten beobachten, die aufgerufen werden können, inklusive `base64binary` und `dataURI`.

![hugging_output](https://cdn.illacloud.com/official-website/img/official-site/components/hugging_output.png)

**Schritt 3: Daten auf Komponenten anzeigen**

Um das von Schritt 2 erhaltene Bild anzuzeigen, ändern wir die Bildquelle der Bildkomponente auf `{{generateInput.fileData.dataURI}}`. Dadurch können wir das generierte Bild anzeigen.

![hugging_display](https://cdn.illacloud.com/official-website/img/official-site/components/hugging_display.png)

**Schritt 4: Ausführung der Aktion mit Komponenten**

Um die Aktion auszuführen, die in Schritt 2 erstellt wurde, wenn die Schaltflächenkomponente angeklickt wird, fügen Sie der Schaltflächenkomponente einen Ereignis-Handler hinzu.

![hugging_event_config](https://cdn.illacloud.com/official-website/img/official-site/components/hugging_event_config.png)

**Schritt 5: Testen**

Nach den letzten vier Schritten können Sie weitere Komponenten und Datenquellen nutzen, um weitere Aufgaben zu erledigen und ein umfassenderes Werkzeug zu erstellen. Zum Beispiel können Sie andere Modelle verwenden, um Eingabeaufforderungen zu generieren oder Eingabeaufforderungen in localStorage oder einer Datenbank zu speichern. Werfen wir einen Blick auf das vollständige Ergebnis, wenn alle Schritte umgesetzt werden.

![hugging_test](https://www.youtube.com/watch?v=SwAry_jIXns)

Jetzt kannst du mit ihm herumspielen! Versuchen Sie andere innovative Modelle, die Hugging face durch diese API zur Verfügung stellt. Es gibt noch mehr zu erforschen, wie zum Beispiel stabile Diffusions-Anime-Modelle, Schulung eines Künstlerstils Stil, Sampling Methode für realistische Bilder. Sie können viel mehr tun!
