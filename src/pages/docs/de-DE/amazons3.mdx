## <h2 hidden>Was ist Amazon S3?</h2>

Amazon S3 ist ein hochskalierbarer und sicherer Cloud-Speicherservice, der von Amazon Web Services (AWS) angeboten wird. In Kombination mit ILLA Cloud können Entwickler nahtlos Amazon S3 in ihre Anwendungen integrieren und damit seine robusten Speicherkapazitäten nutzen, um Daten effizient zu speichern und abzurufen. ILLA Cloud vereinfacht den Integrationsprozess und ermöglicht Entwicklern, sich nahtlos mit Amazon S3 zu verbinden und seine leistungsstarken Funktionen zu nutzen. B. Datenverschlüsselung, Versionierung, Zugriffssteuerung und nahtlose Skalierbarkeit. Mit der Integration von Amazon S3 auf ILLA Cloud können Entwickler ihre Anwendungen mit zuverlässigen und skalierbaren Speicherlösungen erweitern. Damit sie sich auf innovative und datengestützte Erfahrungen konzentrieren können.

### Amazon S3 API erstellen

Es gibt zwei Möglichkeiten, nach der Anmeldung in Ihrem Illa-Konto eine Ressource in Illa zu erstellen.

-   In Ressourcen erstellen

Melde dich mit deinem Illa-Konto an, wähle `**Ressourcen**` oben auf der Seite und klicke auf `**Neues erstellen**` Button.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Wähle **`Amazon S3`** aus der API-Liste.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Verbinden Sie sich mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

Nach dem Erstellen einer Ressource wird der bereitgestellte Amazon S3 wie angezeigt angezeigt.

![amazon_ex_ready](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_ex_ready.jpg)

-   Im Builder erstellen

Melde dich bei deinem Illa Account an, erstelle ein Projekt in Illa Builder auf der `**App**` Seite und navigieren Sie zur **`Action List`** am unteren Ende der Seite. Klicke `**new**`, dann wähle **`Amazon S3`** aus der Datenbankliste. Dann verbinden Sie sich mit der Datenbank mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

### Verbindungseinstellungen

Hier müssen wir Informationen zur Verbindung mit Amazon S3 API zur Verfügung stellen.

![amazon_config](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_config.jpg)

| Eigenschaften                     | Benötigt | Beschreibung                                                                                                                                                                                                                                                |
| --------------------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Name                              | benötigt | Der Name für die Ressource beim Erstellen von Aktionen in der ILLA.                                                                                                                                                                                         |
| Bucket-Name                       | optional | ein eindeutiger Identifikator, der einen Speicher-Container in Amazon S3 darstellt. Es bietet eine logische Trennung für die Organisation und den Zugriff auf Ihre Dateien                                                                                  |
| S3 ACL für hochgeladene Dateien   | optional | erlaubt Ihnen die Definition von Zugriffsberechtigungen für Dateien innerhalb des Buckets. Es legt fest, wer bestimmte Operationen wie Lesen, Schreiben oder Löschen von Dateien ausführen kann.                                                            |
| Region                            | benötigt | geografische Position, an der Ihr Amazon S3 Bucket gespeichert ist. Es bestimmt den physikalischen Standort Ihrer Daten und kann Faktoren wie Latenz und Datenübertragungskosten beeinflussen.                                                              |
| Eigener S3 Endpunkt               | optional | aktiviert die Konfiguration einer bestimmten Endpunkt-URL für den Zugriff auf Ihre Amazon S3 Ressourcen. Es kann für Szenarien nützlich sein, bei denen Sie eine eigene Domain verwenden oder mit einem CDN (Content Delivery Network) integrieren möchten. |
| Zugangsschlüssel                  | benötigt | Anmeldeinformationen, die Ihre Anfragen authentifizieren, um auf Amazon S3 zuzugreifen                                                                                                                                                                      |
| Geheimer Zugangsschlüssel         | benötigt | erzeugt, wenn Sie einen IAM (Identity and Access Management) Benutzer oder ein AWS (Amazon Web Services) Zugang zu Schlüsselpaar erstellen.                                                                                                                 |

### Aktionen erstellen

Wir haben eine Amazon S3 Ressource erstellt, können wir die Aktion hinzufügen, indem wir Amazon S3 aus der Aktionsliste auswählen und den Button **`Aktion erstellen`** auswählen.

![amazon_resource_list](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_resource_list.jpg)

Jetzt haben wir den Amazon Server als Aktion auf unserer Bauseite hinzugefügt.

![amazon](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon.jpg)

### Amazon S3 konfigurieren

#### Übersicht

| Methodenname  | Beschreibung                                                                                                                                                              |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Aktionstyp    | alle Objekte in einer Gruppe auflisten, ein Objekt lesen, ein Objekt herunterladen, ein Objekt löschen, mehrere Objekte löschen, Daten hochladen, mehrere Daten hochladen |
| Bucket-Name   | ein eindeutiger Identifikator, der einen Speicher-Container in Amazon S3 darstellt.                                                                                       |
| Transformator | Umwandlung von Daten in den Stil, den Sie mögen, mit JavaScript                                                                                                           |

#### Alle Objekte in einer Gruppe auflisten

eine Liste aller Objekte in einem bestimmten Bucket abrufen.

** Input **

| Eigenschaften                   | Benötigt | Beschreibung                                                                                                    |
| ------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------- |
| Präfix um Ergebnisse zu filtern | optional | filtert die Ergebnisse um nur Objekte einzubinden, deren Schlüssel (Name) mit einem bestimmten Präfix beginnt   |
| Trennzeichen                    | optional | ein Zeichen angeben, das die Hierarchien der Objektschlüssel trennt                                             |
| Signierte URL generieren        | optional | ob eine zeitlich begrenzte URL erzeugt werden soll, die temporären Zugriff auf ein bestimmtes Objekt ermöglicht |
| Max. Tasten                     | optional | die maximale Anzahl der in der Antwort zurückgegebenen Objekte begrenzen                                        |

Zum Beispiel, um alle Objekte in der Datenbank zu erhalten.

** Ausgabe **

gibt ein Array von Objekten zurück, die mindestens eine `objectKey` Eigenschaft enthalten.

![amazon_list_all_data](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_list_all_data1.jpg)

Verwende `{{s31.data}}` um alle zurückgegebenen Daten zu erhalten.

![amazon_list_all_code](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_list_all_code1.jpg)

#### Objekt lesen

den Inhalt eines bestimmten Objekts in einer Gruppe abrufen

** Input **

| Eigenschaften            | Benötigt | Beschreibung                                                                                        |
| ------------------------ | -------- | --------------------------------------------------------------------------------------------------- |
| Objektschlüssel          | benötigt | der Name oder Pfad des Objekts innerhalb der Gruppe                                                 |
| Signierte URL generieren | optional | eine zeitlich begrenzte URL generieren, die temporären Zugriff auf ein bestimmtes Objekt ermöglicht |

Um zum Beispiel den Inhalt des Dokuments '1111.txt' zu erhalten, setze `1111.txt` für **`Object Key`**.

** Ausgabe **

** Input **

| Eigenschaften            | Benötigt | Beschreibung                                                                                        |
| ------------------------ | -------- | --------------------------------------------------------------------------------------------------- |

#### Daten hochladen

lade ein Objekt in eine Gruppe hoch

** Input **

| Eigenschaften                                  | Benötigt | Beschreibung                                                                                       |
| ---------------------------------------------- | -------- | -------------------------------------------------------------------------------------------------- |
| Inhaltstyp                                     | benötigt | MIME Art von Objekt, das Sie hochladen.                                                            |
| Objektname hochladen                           | benötigt | der Name des Objekts, das Sie hochladen.                                                           |
| Daten hochladen                                | benötigt | die Daten, die Sie bei Amazon S3 hochladen möchten.                                                |
| Timeout für das Hochladen einer Datei (Minute) | optional | die Hochladezeit in Minuten, bevor der Hochladevorgang durch einen Timeout-Fehler gestoppt wird    |

Um beispielsweise mehrere Dokumente zu löschen, setze `text` für **`content type`**, `text.txt` für **`upload object name`**, und `text` für **`upload data`**.

** Ausgabe **

die Nachricht zurückgeben, ob das Dokument erfolgreich hochgeladen wurde

![amazon_upload_data1](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_upload_data1.jpg)

![amazon_upload_data2](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_upload_data2.jpg)

#### Mehrere Daten hochladen

eine Liste von Objekten in eine Gruppe hochladen

** Input **

| Eigenschaften                                  | Benötigt | Beschreibung                                                                                        |
| ---------------------------------------------- | -------- | --------------------------------------------------------------------------------------------------- |
| Inhaltstyp                                     | benötigt | MIME Art von Objekt, das Sie hochladen.                                                             |
| Objektnamenliste hochladen                     | benötigt | eine Liste der Namen der Objekte, die Sie hochladen.                                                |
| Datenliste hochladen                           | benötigt | eine Liste der Daten, die Sie bei Amazon S3 hochladen möchten.                                      |
| Timeout für das Hochladen einer Datei (Minute) | optional | die Hochladezeit in Minuten, bevor der Hochladevorgang durch einen Timeout-Fehler gestoppt wird     |

Zum Beispiel um mehrere Dokumente zu löschen, setze `text` für **`content type`** ein, `{{['text.txt', 'text2.txt']}}` für **`Upload Objekt Name Liste`**, und `{{['text', 'text2']}}` für **`Upload Daten Liste`**.

** Ausgabe **

die Nachricht zurückgeben, ob die Dokumente erfolgreich hochgeladen wurden

![amazon_upload_data1](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_upload_data1.jpg)

![amazon_upload_data2](https://cdn.illacloud.com/official-website/img/docs/assemble/amazon_upload_data2.jpg)
