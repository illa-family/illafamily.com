# <Heading hidden>Bereitstellen auf Prämie</Heading>

Self-Hosting ILLA bedeutet das Hosting und Verwalten der ILLA-Plattform auf Ihrem eigenen Server oder Computer, anstatt einen Drittanbieter zu verwenden. Dieser Ansatz wird oft als sicherer und privater, da Sie die volle Kontrolle über Ihre Daten und ihre Verwendung haben.

ILLA bietet eine Cloud-gehostete Version und die Möglichkeit, Ihre Instanz bald auf verschiedenen Plattformen einzurichten.

## Selbstgehostet

Self-Hosting ILLA auf Ihrer eigenen Infrastruktur ermöglicht Ihnen die nahtlose Integration in Ihr bestehendes Setup, ohne eine separate Konfiguration zu erhalten. Damit können Sie die Funktionen und Funktionen Ihrer bestehenden Infrastruktur nutzen und auf die ILLA erweitern. Einige Vorteile des Einsatzes von ILLA für Ihre Infrastruktur beinhalten:

### Sicherheit

Indem Sie ILLA auf Ihre eigene Infrastruktur unterbringen Sie können Ihre Daten vor Ort aufbewahren und sicherstellen, dass Ihre API-Schlüssel oder Ihre Sicherheitsinformationen niemals Ihren Server verlassen. Dies erlaubt Ihnen, die gleichen Sicherheitsregeln auf ILLA anzuwenden, die Sie für jede andere App auf Ihrem Server anwenden. Die Anwendungen, die Sie mit ILLA erstellen, werden auf Ihrer Infrastruktur und in Ihren Unternehmensnetzwerken gehostet. Sie können auch ILLA in internen Netzwerken hinter einer Firewall hosten und so konfigurieren, dass es mit Ihren eigenen Domain- und Sicherheitsrichtlinien funktioniert.

### Interne Datenbank

Durch selbst-Hosting von ILLA können Sie sich mit einer Datenbank verbinden, die auf demselben Rechner oder Intranet läuft und diese zum Erstellen von Apps verwenden. ILLA verwendet MongoDB zur Speicherung von App-Daten und Redis, um Sitzungsinformationen und nicht-kritische Caches zu speichern. Wenn Sie ILLA mit benutzerdefinierten MongoDB oder Redis Servern auf Ihrer Infrastruktur integrieren möchten Sie müssen die Konfiguration von Umgebungsvariablen ändern und auf Ihre eigenen MongoDB/Redis-Instanzen verweisen.

### Updates

Durch selbst-Hosting-ILLA können Sie flexiblere Update-Zeitpläne und andere Wartungsaktivitäten konfigurieren. Sie können die Schritte zur Einrichtung eines Wartungsfensters folgen und automatische Updates planen. So können Sie Ihre ILLA-Instanz auf eine für Sie und Ihre Organisation bequeme Art und Weise verwalten und aktualisieren.
