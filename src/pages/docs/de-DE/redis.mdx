## <h2 hidden>Was ist Redis?</h2>

Redis in Illa ist eine Datenbank-Integration, mit der Sie sich mit einer Redis-Datenbank verbinden und interagieren können. Redis ist ein beliebtes Open-Source-relationales Datenbank-Management-System, das für seine Geschwindigkeit, Zuverlässigkeit und Benutzerfreundlichkeit bekannt ist.

Mit der Redis-Integration in Illa können Sie Daten aus einer Redis-Datenbank mittels SQL-Befehlen abfragen, einfügen, aktualisieren und löschen. Sie können auch benutzerdefinierte SQL-Abfragen erstellen und innerhalb von Illa ausführen. Dies macht es einfach, datengesteuerte Anwendungen und Dashboards zu erstellen, die Echtzeitdaten aus einer Redis-Datenbank anzeigen.

### Erstelle Redis

Es gibt zwei Möglichkeiten, nach der Anmeldung in Ihrem Illa-Konto eine Ressource in Illa zu erstellen.

-   In Ressourcen erstellen

Melde dich in dein Illa Konto an, wähle `**Ressourcen**` oben auf der Seite und klicke auf `**Neu erstellen**` Button.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Wähle **`Redis`** aus der Datenbankliste.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Verbinden Sie sich mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

Nach der Erstellung einer Ressource werden die Redis wie angezeigt angezeigt.

![redis_ex_ready](https://cdn.illacloud.com/official-website/img/docs/redis_ex_ready.png)

-   Im Builder erstellen

Melde dich bei deinem Illa Account an, erstelle ein Projekt in Illa Builder auf der `**App**` Seite und navigieren Sie zur **`Action List`** am unteren Ende der Seite. Klicke auf `**new**`, dann wähle **`Redis`** aus der Datenbankliste. Verbinden Sie sich dann mit der Datenbank mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

### Verbindungseinstellungen

Hier müssen wir Informationen zur Verbindung mit der Redis-Datenbank zur Verfügung stellen.

![redis_config](https://cdn.illacloud.com/official-website/img/docs/redis_config.png)

| Eigenschaften      | Benötigt | Beschreibung                                                                                                                              |
| ------------------ | -------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| Name               | benötigt | Der Name für die Ressource beim Erstellen von Aktionen in der ILLA.                                                                       |
| Hostname           | benötigt | Die URL oder IP-Adresse für Ihre Datenbank                                                                                                |
| Port               | benötigt | Die Portnummer des Servers, die Sie verwenden sollten, um sich zu verbinden. Wenn Sie keinen Port angeben, ist der Standardport '6379'.   |
| Datenbank-Index    | optional | eine Datenstruktur, die zur Optimierung der Performance von Abfragen verwendet wird, die Daten aus einer Redis-Datenbank abrufen          |
| Datenbank          | benötigt | Der Name der Datenbank                                                                                                                    |
| Benutzername       | benötigt | den Benutzernamen den Sie beim Einloggen auf dem Redis-Server verwenden möchten.                                                          |
| Passwort           | benötigt | Dieses Passwort für die Authentifizierung verwenden.                                                                                      |
| SSL-Optionen       | optional | entscheidet, wie hoch eine sichere SSL-TCP/IP-Verbindung bei der Verhandlung mit dem Server priorisiert wird.                             |

### Aktionen erstellen

Wir haben eine Redis-Ressource erstellt. Wir können die Aktion hinzufügen, indem wir Redis aus der Aktionsliste auswählen und die **`Aktion erstellen`** Schaltfläche auswählen.

![redis_resource_list](https://cdn.illacloud.com/official-website/img/docs/redis_resource_list.png)

Jetzt haben wir den Redis-Server als Aktion zu unserer Bauseite hinzugefügt.

![redis](https://cdn.illacloud.com/official-website/img/docs/redis.png)

### Redis konfigurieren

| Eigenschaften  | Beschreibung                                                       |
| -------------- | ------------------------------------------------------------------ |
| abfrage        | Befehle zum Abrufen von Daten aus der Datenbank                    |
| Transformator  | Umwandlung von Daten in den Stil, den Sie mögen, mit JavaScript    |

Beispiel Verwendung:

-   KEYS

Dieser Befehl ruft eine Liste aller Schlüssel in der Redis-Datenbank ab. Sie können Platzhalter verwenden, um die Schlüssel nach Muster zu filtern.

```sql
KEYS *
KEYS user:*
```

-   ERHALTEN

Dieser Befehl ruft den Wert ab, der mit einer gegebenen Taste verbunden ist.

```sql
GET user:12345
```

-   HGETALL

Dieser Befehl ruft alle Felder und ihre Werte aus einer Redis-Hash-Datenstruktur ab.

```sql
HGETALL user:12345
```

-   ZRANGE

Dieser Befehl ruft einen Bereich von Werten aus einer Redis sortierten Datenstruktur basierend auf deren Punktzahlen ab.

```sql
ZRANGE High_scores 0 9 mit SCORES
```

-   LANGE

Dieser Befehl ruft einen Wertebereich von einer Redis-Datenstruktur ab.

```sql
LRANGE my_list 0 4
```

-   EXISTE

Dieser Befehl prüft, ob ein bestimmter Schlüssel in der Redis-Datenbank existiert.

```sql
EXISTS user:12345
```

-   TTL

Dieser Befehl ruft den Zeit-zu-Live-Wert (TTL) ab, der einem Schlüssel zugeordnet ist. Dieser legt fest, wie lange der Schlüssel in der Datenbank verbleibt, bevor er abläuft.

```sql
TTL user:12345
```
