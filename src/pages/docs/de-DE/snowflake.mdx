## <h2 hidden>Was ist Schneefloke?</h2>

In Illa können Sie mit dem Snowflake Abfrageeditor SQL-Abfragen gegen Ihr Snowflake Data Warehouse durchführen. Sie können sich mit Snowflake verbinden, indem Sie Ihre Anmeldedaten für Snowflake eingeben, einschließlich Kontoname, Benutzername, Passwort und Lagername. Sobald Sie sich mit Snowflake verbunden haben, können Sie den Abfrageeditor verwenden, um SQL-Abfragen zu schreiben und auszuführen, um Ihre Daten abzufragen, zu manipulieren und zu analysieren.

### Schneeflocken erstellen

Es gibt zwei Möglichkeiten, nach der Anmeldung in Ihrem Illa-Konto eine Ressource in Illa zu erstellen.

-   In Ressourcen erstellen

Melde dich in dein Illa Konto an, wähle `**Ressourcen**` oben auf der Seite und klicke auf `**Neu erstellen**` Button.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Wähle **`Schneefloke`** aus der Datenbankliste.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Verbinden Sie sich mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

Nach der Erstellung einer Ressource wird die fertige Schneeflocke wie gezeigt.

![snow_ex_ready](https://cdn.illacloud.com/official-website/img/docs/snow_ex_Ready.png)

-   Im Builder erstellen

Melde dich bei deinem Illa Account an, erstelle ein Projekt in Illa Builder auf der `**App**` Seite und navigieren Sie zur **`Action List`** am unteren Ende der Seite. Klicke `**neu**` und wähle dann **`Schneeflake`** aus der Datenbankliste. Verbinden Sie sich dann mit der Datenbank mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

### Verbindungseinstellungen

Hier müssen wir Informationen zur Verbindung mit der Snowflake Datenbank zur Verfügung stellen.

![snow_config](https://cdn.illacloud.com/official-website/img/docs/snow_config.png)

| Eigenschaften      | Benötigt | Beschreibung                                                                                                                                                                      |
| ------------------ | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Name               | benötigt | Der Name für die Ressource beim Erstellen von Aktionen in der ILLA.                                                                                                               |
| Kontoname          | benötigt | eindeutige Kennung, die jedem Snowflake Konto zugewiesen ist                                                                                                                      |
| Lagerhaus          | benötigt | ein Cluster von Rechenressourcen, die verwendet werden, um Abfragen zu bearbeiten und analytische Arbeitsbelastungen auszuführen                                                  |
| Datenbank          | benötigt | Der Name der Datenbank                                                                                                                                                            |
| Schema             | optional | ein Container, der Datenbankobjekte wie Tabellen, Ansichten und Prozeduren enthält                                                                                                |
| Rolle              | optional | eine benannte Gruppe von Berechtigungen, die einem oder mehreren Benutzern zugewiesen werden können                                                                               |
| Authentifizierung  | benötigt | sicherer Zugriff auf Ihre Daten und Ressourcen im Snowflake Datenlager (Basic Auth oder Key Pair (nur verschlüsselte private Schlüssel werden unterstützt))                       |
| Benutzername       | benötigt | den Benutzernamen den Sie beim Einloggen auf dem Snowflake Server verwenden möchten.                                                                                              |
| Passwort           | benötigt | Dieses Passwort für die Authentifizierung verwenden.                                                                                                                              |

### Aktionen erstellen

Wir haben eine Schneeflockenressource erstellt, wir können die Aktion hinzufügen, indem wir Schneeflocken aus der Aktionsliste auswählen und die **`Aktion erstellen`** Schaltfläche auswählen.

![snow_resource_list](https://cdn.illacloud.com/official-website/img/docs/snow_resource_list.png)

Jetzt haben wir den Snowflake Server als Aktion zu unserer Bauseite hinzugefügt.

![snow](https://cdn.illacloud.com/official-website/img/docs/snow.png)

### Schneeflocken konfigurieren

| Eigenschaften  | Beschreibung                                                                                  |
| -------------- | --------------------------------------------------------------------------------------------- |
| SQL-Abfrage    | sql Befehle zum Abrufen (auswählen), einfügen, aktualisieren, Daten aus der Datenbank löschen |
| Transformator  | Umwandlung von Daten in den Stil, den Sie mögen, mit JavaScript                               |

Beispiel Verwendung:

-   Auswählen

```sql
AUSWÄHLEN * AUS Benutzer VON Nachname;
```

-   Insert

```sql
INSERT INTO users

	(firstname, lastname, email)

VALUES

	(

		{{ NewUser.data.firstNameInput }},

		{{ NewUser.data.lastNameInput }},

		{{ NewUser.data.emailInput }}

	);
```

-   Aktualisieren

```sql
UPDATE users
  SET firstname = '{{form1.updatedProfile.first}}'
  WHERE id = {{ form1.updatedProfile.uid}};
```

-   Löschen

```sql
DELETE FROM users WHERE id = {{ form1.updatedProfile.uid }};
```
