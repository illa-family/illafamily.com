## <h2 hidden>Ce este Cascader?</h2>

Un cascadă este un tip de componentă de intrare care permite utilizatorilor să selecteze date ierarhice, cum ar fi o categorie sau o subcategorie dintr-un meniu derulant. Când un utilizator selectează o opțiune din meniul derulant, cascada va actualiza automat pentru a afișa subcategoriile sau opțiunile relevante într-un meniu derulant imbricat.

De exemplu, dacă aveți o componentă de intrare în cascader pentru selectarea unei categorii de produs, primul meniu derulant poate afișa toate categoriile disponibile, cum ar fi Electronicele, îmbrăcămintea și cărțile. Atunci când un utilizator alege "Electronica", cascada se va actualiza pentru a afișa subcategoriile de produse electronice, cum ar fi "Telefone", "Laptopuri" şi "Tablete". Acest lucru permite utilizatorilor să navigheze rapid și ușor prin date ierarhice și să aleagă opțiunea de care au nevoie.

### \***\*Proprietăți\*\***

Proprietățile disponibile ale Cascaderului. JavaScript poate fi scris pentru a citi sau a modifica informații despre componente.

| Proprietăți             | Descriere                                                                                                                                                                                                                |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Sursa de date           | Fiecare opțiune trebuie să aibă format JSON și să conțină două chei, valoare și etichetă. Folosind array-uri, puteţi crea opţiuni la acelaşi nivel. Iar opțiunile pentru copii pot fi create folosind atributul "copii". |
| Valoare implicită       | Valoarea inițială a componentei. Puteți schimba dinamic valoarea inițială tastând JavaScript în {{}}.                                                                                                                    |
| Substituent             | Valoarea va fi afișată când câmpul de intrare este gol.                                                                                                                                                                  |
| Etichetă                | Numele câmpului afişat utilizatorului                                                                                                                                                                                    |
| Legendă                 | O legendă folosită pentru a descrie câmpul în detaliu                                                                                                                                                                    |
| Etichetă ascunsă        | Setează dacă se afișează eticheta                                                                                                                                                                                        |
| Poziție                 | Setați poziția etichetei în raport cu componenta                                                                                                                                                                         |
| Aliniere                | Setați alinierea (aliniere la stânga sau la dreapta) a etichetei                                                                                                                                                         |
| Width                   | Când eticheta este pe partea stângă a componentei, setaţi raportul lăţimii etichetei.                                                                                                                                    |
| Gestionar de evenimente | Declanșează interogări, controlează componente sau apelează alte API-uri ca răspuns la evenimentele componentei.                                                                                                         |
| Dezactivat              | Controlează starea de dezactivare a componentei. Componenta nu poate fi modificată sau focalizată atunci când este dezactivată.                                                                                          |
| Doar citire             | Controlează starea dacă componenta este doar în citire. O componentă doar-în-citire poate fi selectată și focalizată, dar nu poate fi modificată.                                                                        |
| Afişare buton ştergere  | controlează dacă în câmpul de intrare este sau nu afișat un buton de ștergere                                                                                                                                            |
| Metoda de extindere     | Setează modul în care casierul se extinde (făcând clic sau plutire)                                                                                                                                                      |
| Tooltip                 | Utilizatorii pot introduce aici pontul componentei. Sfatul de instrumente va fi afișat când este concentrat. Formatul markdown este acceptat.                                                                            |
| Hidden                  | Controlează dinamic dacă componenta este ascunsă. Puteți schimba starea ascunsă prin valoarea booleană dinamică.                                                                                                         |

### \***\*Metode\*\***

Puteţi utiliza alte componente pentru a controla componenta. Sprijinim următoarele două metode:

-   **setValue**

Pentru a seta opțiunea selectată, de exemplu, {{”value1”}}

| Proprietăți | Descriere   |
| ----------- | ----------- |
| Valoare     | Input value |

-   **clearValue**

Pentru a șterge valoarea componentei selectate 

Exemplu de utilizare:

Cascader component care ascultă evenimentul `onChange` al altor componente folosind sistemul de evenimente încorporat. Configurează-l urmând acești pași exemplari:

1.  Adăugați un declanșator al evenimentului la componenta pe care doriți să o ascultați. De exemplu, dacă doriţi să ascultaţi evenimentul **`onChange`** al unei componente Radio Group, aţi adăuga un declanşator al evenimentului la acea componentă Radio Group.
2.  În gestionarul de editare a evenimentului, selectaţi **`Componenta Control `** în acţiune, selectaţi componenta **`Cascader`** pe care doriţi să o actualizaţi ca ţintă a evenimentului.
3.  Alegeți acțiunea **`SetValue`** și selectați valoarea potrivită pentru componenta Cascader. Aceasta va fi valoarea pe care componenta Cascader o are și o va actualiza atunci când evenimentul este declanșat.
4.  Salvați setările de declanșare a evenimentului și repetați procesul pentru orice alte componente pe care doriți să le ascultați.

Odată ce ați setat declanșatorul evenimentului, componenta de intrare a cascader se va actualiza automat ori de câte ori evenimentul **`onChange`** este declanșat pe celelalte componente. Acest lucru vă permite să creați interfețe dinamice care răspund la intrarea utilizatorului în timp real, facilitarea navigării și interacțiunii cu aplicația dvs.

### Gestionar de evenimente

| Eveniment  | Descriere                                                                                                              |
| ---------- | ---------------------------------------------------------------------------------------------------------------------- |
| Schimbă    | Când un utilizator schimbă valoarea opțiunii selectate                                                                 |
| Focalizare | Când un utilizator mută cursorul mouse-ului pe componenta cascader                                                     |
| Blur       | Atunci când un utilizator a terminat selectarea opţiunilor şi a renunţat să se concentreze asupra componentei cascader |

### Date

Componenta are niște date utilizate în mod obișnuit, care pot fi apelate prin **`{{componentName.propertyName}}`** în aplicație.

| Nume proprietate | Descriere                                             |
| ---------------- | ----------------------------------------------------- |
| date SursăMod    | modul sursei de date în cascader (dinamică implicită) |
| AfișeazăNume     | numele acestei componente (ie cascader1)              |
| Declanșator      | declanşează evenimente (clic)                         |
| valoare          | Valoare selectată                                     |
| etichetă         | valoare etichetă                                      |
| Aliniere         | aliniere etichetă cascadă (stânga sau dreapta)        |
| Pozitie          | eticheta casierului (stânga sau dreapta)              |
| Lățime etichetă  | integer reprezentând lăţimea etichetei                |
| substituent      | valoare substituent                                   |

Exemplu: {{cascader1.value[0]}

### Utilizare majusculă

Apoi, vom demonstra cum să cartografiem datele de la sursa de date la opțiuni în cascadă dinamic. 

#### Pasul 1 Adăugare acţiune

Haideţi să creăm un tabel în Supabase numit **`proiect`,** precum şi un subtabel numit **`terapeutist`** care sunt aliniate de **`project_id`**.

Apoi putem crea o nouă acțiune pentru Supabase din lista de acțiuni, enumerând toate datele din proiect și numind supabasedb1 

```sql
SELECT [project.id](http://project.id/) ca project_id, [therapist.name](http://therapist.name/) ca therapist_name from therapist join project on therapist.project_id = [project.id](http://project.id/)
```
