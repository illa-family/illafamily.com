## <Heading hidden>Запит</Heading>

Після створення ресурсу, вам потрібно витягти дані з ресурсу, вибирати, вибрати, фільтр, об’єднайте дані та помістіть їх в Маніпулятори за логічними вимогами. Ми називаємо це процесом обробки даних. Дії - це міст між даними та компонентами і є важливою частиною створення програми. Ми поділили дії на два категорії: дії та трансформери.

### SQL дія

#### Створити дію

Процес використання SQL дій розділений на наступні кроки:

1.  Спочатку потрібно знайти список дій в редакторі програми та натиснути кнопку "Додати".
2.  Потім виберіть тип джерела даних в полі вибору ресурсів. Ви можете знайти створені ресурси у відповідному списку ресурсів, ви також можете створити тут нові ресурси. Виберіть створений ресурс і натисніть кнопку "Підтвердити". SQL дія буде успішно створена.
3.  Створена дія буде відображатися в списку дій. Після стискання на дії буде показано нижче редактор дії. Ви можете змінити ресурс дій, писати звіт про дії, вміст відповіді, додати перетворювачі, обробники подій.
4.  Після заповнення списку дій, ви можете натиснути кнопку "Попередній перегляд", щоб перевірити, чи виконала дію яку ви написали. Конфігурація панелі відображення результату дії. Після редагування дії ви можете натиснути кнопку "Зберегти", щоб завершити процес. Якщо ви перейдете на іншу дію, не зберігаючи інформацію, система запропонує вам зберегти зміни.

#### Експлікація

1.  Дія: Ви можете ввести свої дії оператори в модулі дій, де ми додали редактор віддзеркалювання коду, щоб дозволити підсвітку дій та завершення коду. У наступних оновленнях, ми будемо надавати режим вводу графічного інтерфейсу для запису операторів дій, щоб користувачі не знайомі з операторами дії також могли завершити дії даних.
2.  Трансформер: Ви можете звучати логіку дій за допомогою команд JavaScript у перетворенні, наприклад, використовуючи функцію фільтра в JS замість функції в операторі дій.
3.  обробник подій: обробники подій дозволяють вам викликати дії, керувати вставками та іншими діями у відповідь на взаємодію користувача. Ви можете додати кілька обробників подій для відповіді на подію без додавання користувацьких команд JavaScript команд. Наприклад, коли користувач натискає кнопку, ви можете (1) викликати дію для запуску, під час (2) скидає стан кількох випадаючих списків і введених текстів.
4.  Відповідь: у модулі Відповіді ви можете налаштувати умови, за яких дія повертає неправильні результати, таким чином щоб забезпечити точність дій.
5.  Розширений переклад: Уточнені налаштування для таких дій, як очікування очікування бази даних, час обробки подій, затримки затримки тощо в розширених налаштуваннях.

### Дії API

#### Створити дію

Процес дії даних з використанням API подібний до нього для SQL.

1.  З самого початку, у користувача повинен знайти список дій унизу зліва від сторінки редагування додатку і натисніть на знак плюс в верхній правій частині списку дій.
2.  Далі виберіть відповідний список даних у полі вибору ресурсів, який спливає. Ви можете знайти створений ресурс у відповідному списку даних, або ви можете створити тут новий ресурс. Виберіть створений ресурс, натисніть "Створити" і дію API буде створено успішно.
3.  При натисканні на кнопку у списку дійових дій, що відображаються в нижній панелі та в загальній вкладці (Загально), з'являтимуться під кадром під вказівником. Ви можете зробити запит на будь-який HTTP API в ILLA. Будь ласка, перегляньте документацію API-інтерфейс для методів запиту даних.
4.  Як тільки ви заповнили свою дію, ви можете натиснути на кнопку "Попередній перегляд", щоб перевірити чи дія, яку ви написали, дійсна і панель конфігурації дасть вам результати. Після завершення редагування процесу натисніть кнопку Зберегти і вийдіть щоб завершити процес. Якщо ви перейдете на іншу операцію без збереження - спливаюче вікно запропонує вам завершити операцію збереження.

#### Експлікація

1.  Трансформер: Ви можете звучати логіку дій за допомогою JavaScript в перетворенні, наприклад, використовуючи функцію фільтру в JS замість функції в операторі.
2.  Обробники дій: обробники подій дозволяють вам викликати дії, керувати вставками та іншими діями у відповідь на взаємодію користувача. Ви можете додати кілька обробників подій для відповіді на подію без додавання користувацьких команд JavaScript команд. Наприклад, коли користувач натискає кнопку, ви можете (1) викликати дію для запуску, під час (2) скидає стан кількох випадаючих списків і введених текстів.
