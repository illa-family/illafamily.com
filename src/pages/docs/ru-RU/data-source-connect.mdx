Узнайте о подключении к источнику данных и вызовите его в вашем ILLA Builder.

ILLA позволяет вам подключать и интегрировать широкий спектр источников данных, включая базы данных и API, в вашу организацию для использования в качестве ресурсов.

## Источники данных

Источники данных - это места, где данные хранятся и обрабатываются. Примеры источников данных включают базы данных PostgreSQL, Twilio REST API и Amazon S3 buckets. В ILLA вы можете добавить несколько источников данных и сделать их доступными для пользователей для запроса.

### **Интеграции**

ILLA предоставляет встроенную интеграцию для многих популярных источников данных, требующих минимальной конфигурации, таких как SQL базы данных и Stripe. Эти интеграции, также известные как ресурсы, позволяют подключаться к источникам данных и запрашивать их с помощью Retool. Если какой-либо тип источника данных не поддерживается встроенной интеграцией, вы можете использовать REST, GraphQL или SOAP, чтобы подключаться практически к любому API. Это позволяет получить доступ к широкому кругу источников данных из ILLA-builder.

## Ресурсы

Ресурс в ILLA - это сохраненная конфигурация, которая определяет, как ILLA подключается к источнику данных. Чтобы использовать источник данных в ILLA, вы должны создать ресурс. Вы можете выбрать соответствующий ресурс для доступа к данным при написании запроса.

Каждый ресурс в ILLA имеет свои собственные параметры конфигурации, которые определяют, как ILLA подключается к соответствующему источнику данных. Эти параметры могут включать учетные данные и поток, настройки подключения к базе данных и API параметров, такие как базовый URL-адрес и заголовки по умолчанию.

### **Как работают ресурсы**

Ресурсы ILLA сохраняют необходимые параметры для безопасного подключения к вашему источнику данных. Это позволяет легко запрашивать данные без выявления источника данных непосредственно вашим пользователям. Когда запрос запускается для ресурса, Retool использует настройки ресурса для проксирования запроса к источнику данных на сервере. Это гарантирует, что только Retool имеет прямой доступ к источнику данных.

ILLA упрощает процесс подключения к источнику данных для ваших пользователей. Им только нужно выбрать соответствующий ресурс при написании запроса, не беспокоясь о деталях источника данных. Вы также можете добавить несколько ресурсов одного типа, таких как несколько сегментов Amazon S3, чтобы отличить их.

Этот подход скрывает ваши источники данных от ваших пользователей и гарантирует, что все взаимодействия с источником данных будут обработаны безопасно. ILLA избегает установления клиентских соединений с источником данных, которые могли бы раскрывать конфиденциальную информацию и использовать одни и те же условия для всех взаимодействий. Кроме того, ресурсы позволяют кэшировать запросы, что может сократить количество запросов, направляемых к источнику данных, и потенциально снизить затраты.
