## <Heading hidden>Запрос</Heading>

После создания ресурса необходимо извлечь данные из ресурса, выбрать, фильтровать, объединить данные и затем поместить их в Вставки, в соответствии с логическими требованиями. Мы называем этот процесс обработки данных действием. Действия являются мостом между данными и компонентами и являются важной частью построения приложения. Мы разделили действия на две категории: действия и преобразователи.

### Действие SQL

#### Создать действие

Процесс использования SQL-действий разделен на следующие этапы:

1.  Сначала вам нужно найти список действий в редакторе приложений и нажать на кнопку "Добавить".
2.  Затем выберите тип источника данных в окне выбора ресурса. Вы можете найти созданные ресурсы в соответствующем списке ресурсов, вы также можете создать новые ресурсы здесь. Выберите созданный ресурс и нажмите кнопку "Подтвердить". SQL действие будет успешно создано.
3.  Созданное действие будет отображаться в списке действий. Редактор действий будет показан под рамкой после того, как вы вытащите действие. Вы можете изменить ресурс действий, написать заявления действий, ответить содержание, добавить трансформаторы, обработчики событий.
4.  После заполнения списка действий Вы можете нажать на кнопку "Предварительный просмотр", чтобы проверить, является ли Ваше действие действительным. На панели конфигурации будет показан результат действия. После того, как действие будет отредактировано, вы можете нажать кнопку «Сохранить», чтобы завершить процесс. Если вы переключитесь на другое действие без сохранения информации, система предложит вам сохранить изменения.

#### Разбивка действий

1.  Действие: Вы можете ввести свои операторы действий в модуле Действий, где мы добавили редактор зеркал кода, чтобы включить подсветку заявления действий и завершение кода. В последующих обновлениях, мы обеспечим режим ввода графического интерфейса для написания операторов действий, чтобы пользователи, не знакомые с операторами действий, также могли выполнять полноту данных.
2.  Трансформатор: Вы можете уточнить логику действий с помощью операторов JavaScript в трансформаторе, Например, используя функцию фильтра в JS вместо того, чтобы использовать функцию в выражении действий.
3.  Обработчик событий: Обработчики событий позволяют вам инициировать действия, управляющие вставками и другими действиями в ответ на пользовательское взаимодействие. Вы можете добавить несколько обработчиков событий для ответа на событие без добавления пользовательских выражений JavaScript. Например, когда пользователь нажимает на кнопку, вы можете (1) запустить действие для запуска, в то время как сброс состояния нескольких выпадающих списков и текстовых вводов.
4.  Ответ: В модуле Response можно настроить условия, при которых действие возвращает неправильные результаты, таким образом, чтобы гарантировать точность действий.
5.  Дополнительно: Уточните подробные настройки для таких действий, как время ожидания ответа базы, задержки обработчика событий и т.д. в расширенных опциях.

### Действия API

#### Создать действие

Процесс действия данных с использованием API похож на процесс SQL.

1.  Во-первых, пользователю необходимо найти список действий в нижнем левом углу страницы редактирования приложения и нажать на знак плюса в верхнем правом углу списка действий.
2.  Затем выберите соответствующий список источников данных в появившемся окне выбора ресурса. Созданный ресурс можно найти в соответствующем списке источников данных или создать новый ресурс здесь. Выберите созданный ресурс, нажмите на кнопку "Создать" и действие API будет успешно создано.
3.  Успешно созданное действие будет отображаться в списке действий на нижней панели и вкладке Общее, при нажатии появится под рамкой. Вы можете сделать запрос на любой HTTP API в ILLA . Обратитесь к документации по API методам запроса данных.
4.  Как только вы заполнили ваше действие, вы можете нажать на кнопку "Предварительный просмотр", чтобы увидеть, является ли записанное действие действительным, и панель конфигурации даст вам результаты. После того, как вы закончили редактирование вашего действия, вы можете нажать Сохранить и выйти для завершения процесса. При переключении на другую операцию без сохранения всплывающее окно предложит завершить операцию сохранения.

#### Разбивка действий

1.  Трансформатор: Вы можете уточнить логику действий с помощью операторов JavaScript в трансформаторе, Например, используя функцию фильтра в JS вместо того, чтобы использовать функцию в выражении действий.
