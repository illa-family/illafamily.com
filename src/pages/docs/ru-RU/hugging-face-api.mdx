## <Heading hidden>API ограбления</Heading>

Hugging Face - это Github из сообщества машинного обучения, с сотнями тысяч предварительно подготовленных моделей и 10 000 наборов данных. Вы можете свободно получить доступ к моделям и наборам данных, которым обмениваются другие специалисты отрасли или хост и которые устанавливают ваши модели на Hugging лицом.

Некоторые примеры моделей, доступные в библиотеке "Головокружительные лица", включают:

1.  BERT (Двунаправленное представление кодировщиков от Transformers): BERT это модель на базе трансформаторов, разработанная Google для различных задач NLP. Она добилась самых современных результатов в понимании языка и заданиях машинного перевода.
2.  GPT (Generative Pre-training Transformer): GPT - это еще одна модель на базе трансформаторов, разработанная OpenAI. Он в первую очередь используется для задач генерации языков, таких как перевод и текстовое резюме.
3.  RoBERTa (Robustly Optimized BERT): RoBERTa является расширением модели BERT, которая была разработана для улучшения производительности BERT на различных задачах NLP.
4.  XLNet (eXtraordinary LanguageNet): XLNet - это модель на базе трансформаторов, разработанная Google для улучшения производительности моделей трансформаторов на языковых задачах.
5.  ALBERT (A Lite BERT): ALBERT - это версия модели BERT, которая была разработана для повышения эффективности и скорости обучения, сохраняя при этом хорошую производительность на задачах NLP.

На грани Huggingface.co/models](https://huggingface.co/models)]([https://huggingface.co/models)]([https://huggingface.co/models)]([https://huggingface.co/models]\(https://huggingface.co/models](https://huggingface.co/models)**. Кроме того, если вам понадобится решение для сценариев производства, вы можете использовать точки, которые могут быть развернуты и доступны в \[**<https://huggingface.co/docs/inference-endpoints/index>]([https://huggingface.co/endpoints/index)]([https://huggingface.co/docs/inference-endpoints/index](https://huggingface.co/docs/inference-endpoints/index))\*\*.

Builder ILLA предоставляет десятки часто используемых компонентов, позволяющих быстро строить различные интерфейсы на основе ваших конкретных потребностей. В то же время ILLA предлагает соединение с Hugging Face, позволяя быстро подключаться к API, отправлять запросы и получать возвращенные данные. Подключая API и внешние компоненты, вы можете реализовать требование, которое пользователи могут вводить контент через фронт-центр и отправлять его в API. API возвращает сгенерированное содержимое для отображения на фронт-энде.

### Настроить ресурс API Hugging Face

| Свойства     | Требуется    | Описание                                                                                              |
| ------------ | ------------ | ----------------------------------------------------------------------------------------------------- |
| Наименование | обязательный | Укажите имя ресурса, которое будет использоваться для отображения в ILLA                              |
| Токен        | обязательный | Доступ пользователя или маркер API. Вы можете получить его в https://huggingface.co/settings/tokens.  |

### Настроить действие

| Свойства      | Требуется    | Описание                                                                                                                                                                                                                                                 |
| ------------- | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID модели     | обязательный | Поиск моделей: https://huggingface.co/models                                                                                                                                                                                                             |
| Тип параметра | обязательный | Тип параметра вашей конечной точки. Например, если для вашей конечной точки вводится текст, выберите параметр «inputs» с текстом. Если ваша конечная точка нуждается в JSON, выберите заполните параметр «inputs» с помощью JSON или ключевого значения. |
| Параметр      | обязательный | Введите ваш параметр. Используйте {{ componentName.value }} , чтобы использовать данные компонентов.                                                                                                                                                     |

# Как использовать рожинг лица в ILLA Builder

### Шаг 1: Создайте интерфейс с компонентами на ILLA Builder

Основываясь на описанном вами сценарии использования, создайте внешний интерфейс. Например, вы можете использовать компоненты ввода и изображения, если ваш продукт принимает текст и выводит изображение. Если ваш продукт принимает текст и выводит сгенерированный текст, вы можете использовать ввод и текстовые компоненты.

Следующее изображение является примером страницы фронт-энда для продукта, который отвечает на вопросы в зависимости от контекста.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi1.png' alt="" />

### Шаг 2: Создайте ресурс с накручивания лица и настройте действия

Нажмите + New в списке действий и выберите Hugging Face Inference API.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi2.png' alt="" />

Заполните форму для подключения к вашей манере надежды:

Название: имя, которое будет отображаться в ILLA

Жетон: Зайдите в свое рогающее лицо [настройки профиля]([https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens))

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi3.png' alt="" />

Подтвердите информацию о модели в грани разрезания перед настройкой действий:

Выберите модель на [Странице модели ограждения]([https://huggingface.co/models](https://huggingface.co/models))

Давайте воспользуемся [deepset/roberta-base-squad2]([https://huggingface.co/deepset/roberta-base-squad2](https://huggingface.co/deepset/roberta-base-squad2)). Введите страницу деталей > нажмите Deploy > Щелкните Inference API

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi4.png' alt="" />

Параметры после «inputs» - это то, что вы должны заполнить в ILLA. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi5.png' alt="" />

В ILLA Builder нам следует заполнить ID модели и Parameter. Принимая вышеописанную модель в качестве примера, «inputs» является парой ключевого значения, поэтому мы можем заполнять ее ключевым значением или JSON. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi6.png' alt="" />

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi7.png' alt="" />

Мы также поддерживаем текстовый ввод и бинарный ввод, которые могут соответствовать всем соединениям API Hugging Face Inference API.

### Шаг 3: Подключение действий к компонентам

Чтобы передать пользовательский входной интерфейс интерфейса в API, вы можете использовать {{ для получения данных, введенных в компоненте. Например, input2 это компонент для ввода вопроса и input1 является компонентом входного контекста, мы можем заполнить `question` и `context` ключом и заполнить \`{{ input.value }}в значении:

```jsx
{
"вопрос": {{input2.value}},
"контекст": {{input1.value}}
}
```

Перед тем как отобразить выходные данные Действия в внешнем компоненте, мы должны подтвердить, в каком поле размещается вывод различных моделей. В качестве примера все еще взяв `deepset/roberta-base-squad2` следующие результаты:

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi8.png' alt="" />

Таким образом, ответ можно получить с помощью `{{ textQuestion.data[0].answer }}`textQuestion` (`textQuestion\` называется действие).
