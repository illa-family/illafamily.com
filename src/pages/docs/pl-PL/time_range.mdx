## <h2 hidden>Czym jest zakres czasowy?</h2>

Komponent zakresu czasowego jest komponentem interfejsu użytkownika, który umożliwia użytkownikom wybór zakresu czasowego z 24-godzinnego interfejsu, zwykle używane w formularzach lub interfejsach wprowadzania danych, w których użytkownik musi określić zakres czasu rozpoczęcia i zakończenia.

Gdy komponent zakresu czasowego jest dodany do płótna, wyświetla dwa 24-godzinne interfejsy obok siebie. Jeden interfejs jest używany do wyboru godziny rozpoczęcia, podczas gdy drugi jest używany do wyboru czasu zakończenia. Użytkownik może dostosować czas przesuwając kursor myszy lub używając strzałek górnych i dolnych.

Dostępne właściwości zakresu czasowego. JavaScript może być napisany w celu przeczytania lub zmiany informacji o komponentach.

| Godzina rozpoczęcia            | Domyślna wartość czasu rozpoczęcia komponentu                                                                                                                      |
| Czas zakończenia               | Domyślna wartość czasu końcowego komponentu                                                                                                                        |
| Formatowanie                   | Prawidłowy ciąg znaków formatu daty. Zob. https://day.js.org/docs/en/parse/string-format.                                                                          |
| Zacznij miejsce                | Wartość będzie wyświetlana, gdy pole czasu rozpoczęcia jest puste.                                                                                                 |
| Zastępca Kresu                 | Wartość będzie wyświetlana, gdy pole czasu zakończenia jest puste.                                                                                                 |
| Rozmiar kroku                  | rozmiar stopnia w jednostkach minut                                                                                                                                |
| Etykieta                       | Nazwa pola wyświetlanego użytkownikowi                                                                                                                             |
| Podpis                         | Podpis używany do szczegółowego opisu pola                                                                                                                         |
| Ukryta etykieta                | Ustaw czy wyświetlić etykietę                                                                                                                                      |
| Położenie                      | Ustaw położenie etykiety względem komponentu                                                                                                                       |
| Wyrównanie                     | Ustaw wyrównanie (wyrównanie do lewej lub prawej) etykiety                                                                                                         |
| Width                          | Kiedy etykieta znajduje się po lewej stronie składnika, ustaw stosunek szerokości etykiety.                                                                        |
| Obsługa zdarzeń                | Wyzwalacz zapytań, sterowanie komponentami lub wywoływanie innych interfejsów API w odpowiedzi na zdarzenia komponentów.                                           |
| Wyłączone                      | Kontroluj status czy komponent jest wyłączony. Komponent nie może być zmodyfikowany lub aktywowany, gdy jest wyłączony.                                            |
| Wskazówka                      | Użytkownicy mogą tutaj wprowadzić podpowiedź składową. Podpowiedź zostanie wyświetlona, gdy jest ostrością. Format Markdown jest obsługiwany.                      |
| Pokaż przycisk wyczyść         | kontroluj, czy przycisk wyczyść jest wyświetlany w polu wejściowym                                                                                                 |
| Pole wymagane                  | Ważne tylko wtedy, gdy przełącznik jest włączony.                                                                                                                  |
| Własna reguła                  | Utwórz tutaj swoją logikę sprawdzania. Reguły powinny być sporządzone w JavaScript i objęte {{}}.                                                                  |
| Ukryj komunikat potwierdzający | Możesz ukryć komunikat o błędzie przełączając status ukrycia, gdy wartość wejściowa jest nieprawidłowa. Możesz dynamicznie zmienić status ukryty przez JavaScript. |
| Klucz danych formularza        | Określ klucz komponentu opakowania formularza podczas konstruowania atrybutu danych.                                                                               |
| Hidden                         | Dynamicznie kontroluj czy komponent jest ukryty. Możesz zmienić ukryty status poprzez dynamiczną wartość logiczną.                                                 |
| Kolor motywu                   | Pozwala użytkownikom określić kolor tła i przezroczystość przycisku                                                                                                |

### Methods

Możesz użyć innych komponentów do kontroli komponentu. Popieramy następujące dwie metody:

-   **setStartTime**

Aby ustawić wartość daty początkowej, na przykład {{"value1"}}

| Właściwości | Opis                               |
| ----------- | ---------------------------------- |
| Wartość     | Wprowadź wartość czasu rozpoczęcia |

-   **setEndTime**

Aby ustawić wartość daty końcowej, na przykład {{"value2"}}

| Właściwości | Opis                               |
| ----------- | ---------------------------------- |
| Wartość     | Wprowadź wartość czasu zakończenia |

-   **clearValue**

Aby wyczyścić wartość wybranego komponentu 

-   **validate**

Sprawdzenie, czy informacje wejściowe są zgodne z prawem

-   **clearValidate**

Aby wyczyścić komunikat potwierdzający

### Obsługa zdarzeń

| Wydarzenie | Opis                                                                                                      |

| Zmiana     | Gdy użytkownik zmienia wybraną wartość czasu, wykonaj określone działanie dostosowane przez użytkowników. |

| Nazwa własności       | Opis                                                                |
| --------------------- | ------------------------------------------------------------------- |
| schemat kolorów       | właściwość koloru daty                                              |
| niestandardowa reguła | niestandardowa reguła określona przez użytkownika                   |
| wyłączone             | wartość logiczna wskazuje status wyłączony                          |
| nazwa wyświetlacza    | nazwa tego składnika (tj. kaskadowy1)                               |
| startTime             | wartość czasu rozpoczęcia                                           |
| endTime               | wartość czasu zakończenia                                           |
| Zastępca startowy     | wartość zastępcza dla czasu rozpoczęcia                             |
| endPlaceholder        | wartość zastępcza dla czasu zakończenia                             |
| wydarzenia            | wartość tablicy dla sekwencji zdarzeń                               |
| format                | postać daty i godziny                                               |
| formDataKey           | Formatowy klucz daty                                                |
| hidden                | wartość logiczna wskazuje ukryty status przycisku                   |
| ukryj Wiadomość       | wartość logiczna wskazuje, czy komunikat potwierdzający jest ukryty |
| etykieta              | wartość etykiety                                                    |
| Wyrównanie etykiety   | wyrównanie etykiety kaskadowej (lewa lub prawa)                     |
| pozycja etykiety      | pozycja etykiety kaskadowej (lewa lub prawa)                        |
| Szerokość etykiety    | liczba całkowita reprezentująca szerokość etykiety                  |
| krok minutowy         | wartość rozmiaru kroku minutowego                                   |
| wymagane              | wartość logiczna wskazuje wymagany status wejścia.                  |
| pokaż Wyczyść         | wartość logiczna wskazuje czy data zostanie wyczyszczona            |
| wartość               | Wybrana wartość                                                     |

Przykład: {{rangeTime1.value[0]}}

### Przypadek użycia

Następnie pokażemy jak mapować dane od źródła danych do zakresu czasowego i ustawić wartość końcową na bieżący czas za pomocą przycisku. 

#### Krok 1 Dodaj akcję

Utwórzmy tabelę w REST API o nazwie **`promoteCodes`,** zawierającą 6 kolumn: **`code`**, **`id`**, **`createdAt`**, **`expiredAt`**, **`startedAt`, `updatedAt`**. Będziemy używać startedPrzy, a czas wygaśnięcia do próbki. 

Następnie możemy utworzyć nową akcję dla REST API z listy działań, wymieniając wszystkie dane w promoteCodes i nazwane restapi1. 

Kliknij **`Zapisz`** i **Uruchom\`**, aby aktywować tę akcję.

#### Krok 2 Dodaj komponenty

Następnie możemy dodać komponent **`zakres czasowy`** i komponent **`przycisk`** do płótna. Dla domyślnej wartości czasu rozpoczęcia komponentu zakresu dat, używamy startedAt czasu pierwszego kodu promocyjnego z pozostałych danych api.

```sql
{{restapi1.data.promoteCodes[0].startedAt}}
```

Podobnie, dla domyślnej wartości czasu końcowego, używamy wygasłegoW momencie pierwszego promowania kodu z pozostałych danych api.

```sql
{{restapi1.data.promoteCodes[0].expiredAt}}
```

Oznaczaliśmy również przycisk jako "Teraz", jak pokazano poniżej.

![time_range_0](https://cdn.illacloud.com/official-website/img/docs/time_range_0.png)

#### Krok 3 Skonfiguruj komponent

Dla komponentu **`przycisk`** możemy go skonfigurować tak, aby ustawić wartość daty końcowej w komponencie **`zakres czasowy`** na dzisiejszą datę.

1.  W obsłudze zdarzeń edytuj wybierz **`składnik kontroli`** w akcji, wybierz składnik **`zakres czasowy`**, który chcesz zaktualizować jako cel wydarzenia.
2.  Wybierz działanie **`setEndTime`** i ustaw **`Koniec time`** na `{{currentUserInfo.updatedAt}}` 

![time_range_config](https://cdn.illacloud.com/official-website/img/docs/time_range_config.png)

#### Etap 4 Badanie

Teraz, gdy klikniesz przycisk "Teraz", czas zakończenia powinien zmienić się na dzisiejszą datę. Jak na razie mamy 20:03:05, ale może się to różnić podczas testowania.
