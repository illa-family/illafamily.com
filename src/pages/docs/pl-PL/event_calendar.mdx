## <h2 hidden>Co to jest kalendarz wydarzeń?</h2>

Komponent Kalendarz Wydarzeń w Chmurze ILLA jest potężnym narzędziem do organizowania i wyświetlania wydarzeń w ciekawy i przyjazny dla użytkownika. Pozwala użytkownikom na tworzenie, zarządzanie i prezentowanie wydarzeń z łatwością, udostępnianie funkcji takich jak nawigacja daty, kategoryzacja zdarzeń i szczegółowe informacje o zdarzeniach. Komponent Kalendarz Wydarzeń oferuje konfigurowalne układy, intuicyjne interfejsy tworzenia wydarzeń oraz opcje integracji z innymi systemami, uczynienie z niej istotnego narzędzia dla przedsiębiorstw i organizacji skutecznego zarządzania wydarzeniami i ich promowania.

### Właściwości

#### Opcje ręczne

Dodaj nowe opcje lub usuń opcje ręcznie i skonfiguruj opcje po nim. Właściwości opcji są następujące:

| Właściwości                    | Opis                                                                                                                                                                   |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Identyfikator wydarzenia       | jednoznacznie identyfikuje każde zdarzenie w kalendarzu                                                                                                                |
| Tytuł                          | tytuł lub nazwa wydarzenia                                                                                                                                             |
| Opis                           | Opis wydarzenia                                                                                                                                                        |
| Rozpocznij                     | data i godzina rozpoczęcia wydarzenia                                                                                                                                  |
| Koniec                         | data i godzina zakończenia zdarzenia                                                                                                                                   |
| ID zasobu                      | identyfikuje zasób lub podmiot powiązany z wydarzeniem                                                                                                                 |
| Tytuł zasobu                   | tytuł lub nazwa zasobu skojarzonego z wydarzeniem                                                                                                                      |
| Cały dzień                     | czy wydarzenie jest wydarzeniem całodniowym czy nie                                                                                                                    |
| Przeciągalny                   | czy zdarzenie może być przeciągnięte lub przeniesione w interfejsie kalendarza                                                                                         |
| Resizable                      | Czy zezwolić użytkownikom na aktualizację zakresu czasowego wydarzenia i zmianę przedziału czasowego, przesuwając granicę wydarzenia do wydłużenia lub skrócenia czasu |

#### Opcje mapowane

Możesz również dodać opcje ze źródła danych poprzez mapowanie etykiety i właściwości wartości do odpowiednich pól danych. Możesz uzyskać opcje dynamicznie z bazy danych w ten sposób. Na końcu tego artykułu zademonstrowaliśmy, jak wykorzystać go przykładowo.

| Właściwości                    | Opis                                                                                                                                                                            |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Źródło danych                  | Ustaw źródło danych opcji                                                                                                                                                       |
| Identyfikator wydarzenia       | Unikalny identyfikator używany do identyfikacji zdarzenia                                                                                                                       |
| Tytuł                          | Tytuł wydarzenia                                                                                                                                                                |
| Opis                           | Opis wydarzenia                                                                                                                                                                 |
| Rozpocznij                     | Godzina rozpoczęcia wydarzenia                                                                                                                                                  |
| Koniec                         | Czas zakończenia wydarzenia                                                                                                                                                     |
| ID zasobu                      | Zasoby mogą być osobą, salą posiedzeń itp. Możesz ustawić ID zasobu i podłączyć go do wydarzenia.                                                                               |
| Tytuł zasobu                   | Nazwa zasobu wyświetlana w widoku zasobów                                                                                                                                       |
| Cały dzień                     | Czy to całodzienne wydarzenie                                                                                                                                                   |

| Resizable                      | Czy zezwolić użytkownikom na aktualizację zakresu czasowego wydarzenia i zmianę przedziału czasowego, przesuwając granicę wydarzenia do wydłużenia lub skrócenia czasu          |

### Przypadek użycia

Następnie pokażemy jak dodać nowe wydarzenia wyświetlane w kalendarzu wydarzeń za pomocą przycisku.

#### Krok 1 Dodaj akcję

Utwórzmy tabelę w Supabase o nazwie `events`. Zawiera 9 kolumn: `title`, `description`, `resource_id`, `resource_title`, `all_day`, `draggable`, `size timzable`, `start_time`, i `end_time`.

Następnie możemy utworzyć nową akcję dla Supabase z listy działań, wymieniając wszystkie dane z fragmentem kodu poniżej i nazywaną `supabasedb1.`

```sql
wybiera * ze zdarzeń;
```

Zmień z 'Uruchom działanie tylko po ręcznym uruchomieniu' na 'Uruchom działanie automatycznie po zmianie wprowadzania' w górnym prawym rogu poniżej przycisku 'Zapisz'.

Kliknij **`Zapisz`** i **Uruchom\`**, aby aktywować tę akcję.

![event_data](https://cdn.illacloud.com/official-website/img/official-site/components/event_data.jpg)

Aby dodać nowe wydarzenie, potrzebujemy również innej czynności. Wybierz ten sam zasób Supabase z listy akcji o nazwie `supabasedb2` i umieść fragment kodu poniżej.

```sql
INSERT INTO events (id, title, description, start_time, end_time, resource_id, resource_title, all_day, draggable, resizable)
VALUES ((SELECT MAX(id) + 1 FROM events), '{{input1.value}}', '{{input2.value}}', '{{dateTime1.value}}', '{{dateTime2.value}}', {{numberInput1.value}}, '{{input4.value}}', {{switch1.value}}, {{switch2.value}}, {{switch3.value}});
```

Wpis, numerInput, dateTime i przełączanie składników są z składników modalnych. Poniżej przedstawiono bardziej szczegółowe opisy tych komponentów.

#### Krok 2 Konfiguruj Akcje

W przypadku supabasedb2 chcemy wyświetlić powiadomienie, gdy informacje o nowym wydarzeniu zostaną pomyślnie dodane do bazy danych.

W Akcjach kliknij `supabasedb2`, na stronie `General`, przewiń w dół do `Event handler`.

1.  Kliknij **`+ Nowość`**, aby dodać program obsługi zdarzeń.
2.  W obsłudze zdarzeń edytuj wybierz **`Pokaż powiadomienie`** w akcji, **`title`** "Sukces" i **`opis`** "Pomyślnie dodaj nowe wydarzenie. Wybierz `Sukces` dla **`type`**, wprowadź `{{2000}}` na **`czas trwania`** (2 sekundy)

![event_action_success](https://cdn.illacloud.com/official-website/img/official-site/components/event_action_success.jpg)

#### Krok 2 Dodaj komponenty

Następnie możemy dodać komponent **`kalendarz wydarzeń`**, komponent **przycisk\`** i komponent `**modal**` do płótna. Oznacz komponent przycisku do `Dodaj nowe wydarzenie`.

![event_layout](https://cdn.illacloud.com/official-website/img/official-site/components/event_layout.jpg)

W składniku modalnym zmieniamy tekst w lewym górnym rogu na "Nowe wydarzenie". Następnie dodajemy komponenty wejściowe `Title` i `Description`, `Start time` i `End time `, elementy czasowe czasu, `Resource ID` , i komponenty wejściowe `Tytuł Zasobów`, komponenty "Draggable`, i "Resizable` i opisuj je w razie potrzeby.

![event_modal_layout](https://cdn.illacloud.com/official-website/img/official-site/components/event_modal_layout.jpg)

#### Krok 3 Skonfiguruj komponent

Dla komponentu `**kalendarz zdarzeń**` możemy mapować dane z `supabasedb1` do wyświetlenia. Najpierw wybieramy `mapped` z opcji, następnie umieszczamy `{{supabasedb1.data}}` w źródle danych i wypełnimy pola `Mapped Option` jak pokazano poniżej.

![event_data_source](https://cdn.illacloud.com/official-website/img/official-site/components/event_data_source.jpg)

Dla komponentu **`przycisk`** możemy go skonfigurować do otwarcia modalnego po kliknięciu.

1.  Kliknij komponent **`przycisk`, aby otworzyć jego stronę kontrolną. W **`Obsługa zdarzeń`**, kliknij \`**+ Nowa\*\*\`.
2.  W obsłudze zdarzeń edytuj wybierz **`składnik kontroli`** w akcji, i wybierz składnik **`modal`**, który chcesz zaktualizować jako cel wydarzenia.
3.  Wybierz metodę **`openModal`**

![event_add_new_button](https://cdn.illacloud.com/official-website/img/official-site/components/event_add_new_button.jpg)

Dla komponentu **modal\`** możemy go skonfigurować do odświeżenia wyświetlanych wydarzeń w `**kalendarzu zdarzeń**` po zamknięciu modu.

1.  Kliknij komponent **`modal`**, aby otworzyć jego stronę kontrolną. W **`Obsługa zdarzeń`**, kliknij `**+ Nowa**`.
2.  W obsłudze zdarzeń edytuj wybierz **`Wyzwalacz akcji`** w akcji i wybierz komponent **`supabasedb`**, który chcesz wyzwalać dla `Nazwa akcji`. W naszym przypadku jest to `supabasedb1`.

![event_modal](https://cdn.illacloud.com/official-website/img/official-site/components/event_modal.jpg)

Dla komponentu "Anuluj" **`button`** możemy go skonfigurować do zamknięcia modu, jeśli kliknięto na niego

1.  Kliknij komponent **`przycisk`, aby otworzyć jego stronę kontrolną. W **`Obsługa zdarzeń`**, kliknij \`**+ Nowa\*\*\`.
2.  W obsłudze zdarzeń edytuj wybierz **`składnik kontroli`** w akcji, i wybierz składnik **`modal`**, który chcesz zaktualizować jako cel wydarzenia.
3.  Wybierz metodę **closeModal\`**

![close_modal](https://cdn.illacloud.com/official-website/img/docs/close_modal.jpg)

Dla "Potwierdź" **`przycisk`**, chcemy, aby wprowadziła nowe dane wypełnione w modalu do tabeli **`events`** i zamknij modal.

1.  Kliknij komponent **`przycisk`, aby otworzyć jego stronę kontrolną. W **`Obsługa zdarzeń`**, kliknij \`**+ Nowa\*\*\`.
2.  W obsłudze zdarzeń edytuj wybierz **`Wyzwalacz akcji`** dla akcji i wybierz `supabasedb2` dla nazwy akcji

![event_db2](https://cdn.illacloud.com/official-website/img/official-site/components/event_db2.jpg)

1.  Kliknij **`+ Nowość`**, aby dodać program obsługi zdarzeń.
2.  W obsłudze zdarzeń edytuj wybierz **`składnik kontroli`** w akcji, i wybierz składnik **`modal`**, który chcesz zaktualizować jako cel wydarzenia.
3.  Wybierz metodę **closeModal\`**

![close_modal](https://cdn.illacloud.com/official-website/img/docs/close_modal.jpg)
