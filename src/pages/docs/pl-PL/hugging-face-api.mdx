## <Heading hidden>Interfejs API twarzy</Heading>

Ostre twarze to Github społeczności nauki maszyn, z setkami tysięcy wstępnie przeszkolonych modeli i 10 000 zbiorów danych. Możesz swobodnie uzyskać dostęp do modeli i zbiorów danych udostępnianych przez innych ekspertów branżowych lub hostów i wdrażać swoje modele na twarzy.

Przykłady modeli dostępnych w bibliotece Hugging Face obejmują:

1.  BERT (Bidirectional Encoder reprezentations from Transformers): BERT jest modelem opartym na transformatorach opracowanym przez Google dla różnych zadań NLP. Osiągnięto najnowsze wyniki w zakresie zrozumienia języków i zadań związanych z tłumaczeniem maszynowym.
2.  GPT (Generative pre-training Transformer): GPT jest kolejnym modelem opartym na transformatorach opracowanym przez OpenAI. Jest on stosowany przede wszystkim do zadań związanych z generowaniem języków, takich jak tłumaczenie i streszczenie tekstu.
3.  RoBERTa (Zoptymalizowany BERT): RoBERTa stanowi rozszerzenie modelu BERT opracowanego w celu poprawy funkcjonowania BERT w odniesieniu do różnych zadań NLP.
4.  XLNet (eXtraordinary LanguageNet): XLNet jest modelem opartym na transformatorach opracowanym przez Google, mającym na celu poprawę wydajności modeli transformatorów w zakresie zadań związanych z interpretacją języka.
5.  ALBERT (A Lite BERT): ALBERT jest wersją modelu BERT, który opracowano w celu zwiększenia wydajności i przyspieszenia szkolenia przy jednoczesnym utrzymaniu dobrych wyników w zakresie zadań NLP.

### Co możesz zrobić z wyciskiem Face w konstruktorze ILLA

W twarzy twarzy ponad 130 000 modeli nauki maszyn jest dostępnych za pośrednictwem publicznego interfejsu API, z którego możesz korzystać i testować za darmo na \[**<https://huggingface.co/models>](<https://huggingface.co/models>**. Ponadto, jeśli potrzebujesz rozwiązania dla scenariuszy produkcji, możesz użyć punktów końcowych preferencyjnych Hugging Face, które można zastosować i uzyskać dostęp do \[**<https://huggingface.co/docs/inference-endpoints/index>](<https://huggingface.co/docs/inference-endpoints/index>**.

Kreator ILLA dostarcza dziesiątki powszechnie używanych komponentów front-end, co pozwala na szybką budowę różnych interfejsów front-end w oparciu o Twoje konkretne potrzeby. Jednocześnie ILLA oferuje połączenie z twarz Hugging Face, umożliwiając szybkie połączenie się z API, wysyłanie żądań i odbieranie zwróconych danych. Podłączając interfejs API i komponenty front-end, możesz zaimplementować wymaganie, aby użytkownicy mogli wprowadzać zawartość przez główny koniec i przesyłać ją do API. API zwraca wygenerowaną zawartość do wyświetlenia na stronie głównej.

### Skonfiguruj zasób Face API

| Właściwości | Wymagane | Opis                                                                                           |
| ----------- | -------- | ---------------------------------------------------------------------------------------------- |
| Nazwisko    | wymagane | Zdefiniuj nazwę zasobu, która będzie używana do wyświetlania w ILLA                            |
| Token       | wymagane | Dostęp użytkownika lub token API. Możesz to uzyskać w https://huggingface.co/settings/tokens.  |

### Konfiguruj akcję

| Właściwości          | Wymagane | Opis                                                                                                                                                                                                                                                         |
| -------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Identyfikator modelu | wymagane | Szukaj modeli: https://huggingface.co/models                                                                                                                                                                                                                 |
| Typ parametru        | wymagane | Typ parametru punktu końcowego. Na przykład, jeśli Twój punkt końcowy potrzebuje wprowadzenia tekstu, wybierz parametr "wejść" tekstem. Jeśli twój punkt końcowy potrzebuje danych wejściowych JSON, wybierz parametr "wejść" z JSON lub wartością kluczową. |
| Parametr             | wymagane | Wprowadź swój parametr. Użyj {{ componentName.value }} aby użyć danych komponentów.                                                                                                                                                                          |

# Jak używać wycisk twarzy w budownictwie ILLA

### Krok 1: Zbuduj interfejs użytkownika z komponentami na konstruktorze ILLA

W oparciu o opisany przez Ciebie scenariusz użytkowania zbuduj interfejs front-end. Na przykład możesz użyć komponentów wejściowych i obrazów, jeśli produkt przyjmuje tekst i wyśle obraz. Jeśli Twój produkt przyjmuje wygenerowany tekst tekstowy i wynikowy, możesz użyć komponentów wejściowych i tekstowych.

Poniższy obraz jest przykładem strony front-end dla produktu, który odpowiada na pytania oparte na kontekście.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi1.png' alt="" />

### Krok 2: Utwórz Ostre Zasoby i skonfiguruj działania

Kliknij + Nowe na liście akcji i wybierz API Ociosania Face Inference API.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi2.png' alt="" />

Wypełnij formularz, aby połączyć się z twoją twardą twarzy:

Nazwa: nazwa, która będzie wyświetlana w ILLA

Token: Uzyskaj swoją twarz [ustawienia profilu]([https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens))

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi3.png' alt="" />

Potwierdź informacje modelowe w twarzy przed skonfigurowaniem akcji:

Wybierz model w [Stronie modelu twarzy]([https://huggingface.co/models](https://huggingface.co/models))

Jako przykład użyjmy [głęboko głęboko set/roberta-base-squad2]([https://huggingface.co/deepset/roberta-base-squad2](https://huggingface.co/deepset/roberta-base-squad2)). Wprowadź stronę szczegółów > kliknij Rozmieść > Kliknij API

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi4.png' alt="" />

Parametry po `"inputs"` to co powinieneś wypełnić ILLA. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi5.png' alt="" />

W konstruktorze ILLA powinniśmy wypełnić identyfikator modelu i parametry. Przyjmując za przykład powyższy model, `"wejście"` jest parą klucz-wartość, więc możemy wypełnić ją wartością kluczową lub JSON. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi6.png' alt="" />

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi7.png' alt="" />

Wspieramy również dane wejściowe i dane wejściowe w systemie binarnym, które mogą spotkać się ze wszystkimi połączeniami interfejsu API Ostrzania Face.

### Krok 3: Połącz akcje z komponentami

Aby przekazać dane wejściowe użytkownika do API, możesz użyć {{ aby pobrać dane wprowadzone w komponencie. Na przykład dane wejściowe2 to komponent do wprowadzenia pytania, a dane wejściowe1 to kontekst wejściowy, możemy wypełnić `pytanie` i `context` w kluczu i wypełnić `{{ input.value }}`:

```jsx
{
"pytanie": {{input2.value}},
"kontekst": {{input1.value}}
}
```

Przed wyświetleniem danych wyjściowych działania w komponencie front-end powinniśmy potwierdzić, w którym polu znajduje się dane wyjściowe różnych modeli. Wciąż przyjmując za przykład `deepset/roberta-base-squad2` wyniki są następujące:

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi8.png' alt="" />

Tak więc możemy uzyskać odpowiedź z `{{ textQuestion.data[0].answer }}`(`textQuestion` to nazwa akcji). 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi9.png' alt="" />
