## <Heading hidden>Query</Heading>

Dopo aver creato la risorsa, è necessario estrarre i dati nella risorsa, selezionare, filtrare, unire i dati e quindi inserirli negli inserti secondo i requisiti logici. Chiamiamo questo processo di elaborazione dei dati di un'azione. Le azioni sono il ponte tra dati e componenti e costituiscono una parte essenziale della costruzione di un'applicazione. Abbiamo suddiviso le azioni in due categorie: azioni e Transformers.

### Azione SQL

#### Crea azione

Il processo di utilizzo delle azioni SQL è diviso nei seguenti passaggi:

1.  In primo luogo, è necessario trovare l'elenco di azioni nell'editor di app e fare clic sul pulsante "Aggiungi".
2.  Quindi, selezionare un tipo di sorgente dati nella casella di selezione della risorsa. Puoi trovare le risorse create nell'elenco delle risorse corrispondenti, puoi anche creare nuove risorse qui. Selezionare la risorsa creata e fare clic sul pulsante "Conferma". Un'azione SQL verrà creata con successo.
3.  L'azione creata verrà visualizzata nell'elenco delle azioni. Un editor di azioni verrà mostrato sotto il frame dopo aver calcato l'azione. È possibile modificare la risorsa delle azioni, scrivere dichiarazioni di azione, contenuto di risposta, aggiungere trasformatori, gestori di eventi.
4.  Dopo aver compilato l'elenco delle azioni, è possibile fare clic sul pulsante "Anteprima" per verificare se l'azione scritta è valida. Il pannello di configurazione mostrerà il risultato dell'azione. Dopo che l'azione è stata modificata, è possibile fare clic sul pulsante "Salva" per terminare il processo. Se si passa a un'altra azione senza salvare le informazioni, il sistema ti chiederà di salvare la modifica.

#### Ripartizione Azione

1.  Azione: Puoi inserire le tue azioni nel modulo Azione dove abbiamo aggiunto l'editor di Code Mirror per abilitare la dichiarazione di azione evidenziando e il completamento del codice. Negli aggiornamenti successivi, forniremo una modalità di input GUI per la scrittura di istruzioni di azione in modo che gli utenti che non hanno familiarità con le istruzioni di azione possano anche completare le azioni di dati.
2.  Trasformatore: È possibile perfezionare la logica di azione con istruzioni JavaScript nel trasformatore, ad esempio utilizzando la funzione di filtro in JS invece della funzione dove si trova la dichiarazione di azione.
3.  Gestore eventi: i gestori degli eventi consentono di attivare azioni, inserti di controllo e altre azioni in risposta all'interazione dell'utente. È possibile aggiungere più gestori di eventi per rispondere a un evento senza aggiungere istruzioni JavaScript personalizzate. Ad esempio, quando un utente fa clic su un pulsante, è possibile (1) attivare un'azione per eseguire, mentre (2) resettare lo stato di più elenchi a discesa e input di testo.
4.  Risposta: Nel modulo Risposta, è possibile personalizzare le condizioni in cui l'azione restituisce risultati errati, in modo da garantire l'accuratezza dell'azione.
5.  Avanzate: affinare le impostazioni dettagliate per azioni come tempi di risposta timeout del database, ritardi del processore degli eventi, ecc. all'interno delle opzioni avanzate.

### Azioni API

#### Crea azione

Il processo per l'azionamento dei dati utilizzando l'API è simile a quello per SQL.

1.  In primo luogo, l'utente ha bisogno di trovare l'elenco delle azioni in basso a sinistra della pagina di modifica dell'applicazione e fare clic sul segno più in alto a destra della lista delle azioni.
2.  Successivamente, selezionare l'elenco dei dati corrispondenti nella casella di selezione delle risorse che si apre. È possibile trovare la risorsa creata nell'elenco di origine dei dati corrispondente, o è possibile creare una nuova risorsa qui. Seleziona la risorsa creata, clicca su "Crea" e un'azione API verrà creata con successo.
3.  L'azione creata con successo verrà visualizzata nell'elenco Azioni nel pannello inferiore e la scheda Generale per l'azione verrà visualizzata sotto il riquadro quando si fa clic. È possibile effettuare una richiesta a qualsiasi API HTTP in ILLA . Consultare la documentazione API per i metodi di richiesta di dati.
4.  Una volta che hai riempito la tua azione, puoi cliccare sul pulsante "Anteprima" per vedere se l'azione che hai scritto è valida e il pannello di configurazione ti darà i risultati. Una volta terminata la modifica dell'azione, è possibile fare clic su Salva ed uscire per terminare il processo. Se si passa a un'altra operazione senza salvare, un popup ti chiederà di completare l'operazione di salvataggio.

#### Ripartizione Azione

1.  Trasformatore: È possibile perfezionare la logica di azione con istruzioni JavaScript in un trasformatore, ad esempio utilizzando la funzione di filtro in JS invece della funzione dove in una dichiarazione di azione.
2.  Gestori eventi: I gestori di eventi consentono di attivare azioni, inserti di controllo e altre azioni in risposta all'interazione dell'utente. È possibile aggiungere più gestori di eventi per rispondere a un evento senza aggiungere istruzioni JavaScript personalizzate. Ad esempio, quando un utente fa clic su un pulsante, è possibile (1) attivare un'azione per eseguire, mentre (2) resettare lo stato di più elenchi a discesa e input di testo.
