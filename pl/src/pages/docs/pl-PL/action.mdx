## <Heading hidden>Zapytanie</Heading>

Po utworzeniu zasobu, musisz wyodrębnić dane z zasobu, wybrać, filtrować, złącz dane, a następnie umieść je w wejściach zgodnie z logicznymi wymaganiami. Ten proces przetwarzania danych nazywamy działaniem. Działania są pomostem między danymi i komponentami oraz stanowią zasadniczy element budowy aplikacji. Pogrupowaliśmy działania na dwie kategorie: działania i transformatory.

### Akcja SQL

#### Utwórz akcję

Proces używania akcji SQL jest podzielony na następujące kroki:

1.  Najpierw musisz znaleźć listę akcji w edytorze aplikacji i kliknąć przycisk "Dodaj".
2.  Następnie wybierz typ źródła danych w polu wyboru zasobu. Możesz znaleźć utworzone zasoby na odpowiedniej liście zasobów, możesz również tworzyć nowe zasoby tutaj. Wybierz utworzony zasób i kliknij przycisk "Potwierdź". Akcja SQL zostanie pomyślnie utworzona.
3.  Utworzona akcja zostanie wyświetlona na liście działań. Edytor akcji zostanie wyświetlony pod ramką po wyrzuceniu akcji. Możesz zmienić zasób akcji, zapisywać polecenia akcji, zawartość odpowiedzi, dodawać transformatory, obsługi zdarzeń.
4.  Po wypełnieniu listy akcji możesz kliknąć przycisk "Podgląd", aby sprawdzić, czy akcja, którą napisałeś, jest prawidłowa. Panel konfiguracyjny wyświetli wynik działania. Po edycji akcji, możesz kliknąć przycisk "Zapisz", aby zakończyć proces. Jeśli przełączysz się na inną akcję bez zapisywania informacji, system poprosi Cię o zapisanie modyfikacji.

#### Podział akcji

1.  Akcja: Możesz wprowadzić instrukcje działania w module Akcja, w którym dodaliśmy edytor lustrzanego kodu, aby włączyć podświetlanie i uzupełnianie kodu instrukcji działania. W kolejnych aktualizacjach zapewnimy tryb wprowadzania interfejsu graficznego do pisania instrukcji działania, tak aby użytkownicy nieznani z instrukcjami akcji mogli również wykonywać działania danych.
2.  Transformer: Możesz udoskonalić logikę akcji za pomocą instrukcji JavaScript w transformatorze, na przykład poprzez zastosowanie funkcji filtra w JS zamiast funkcji w instrukcji działania.
3.  Obsługa zdarzeń: Obsługa zdarzeń pozwala na uruchamianie akcji, kontrolę wstawiania i innych działań w odpowiedzi na interakcje użytkownika. Możesz dodać wiele obsługi zdarzeń, aby odpowiedzieć na wydarzenie bez dodawania niestandardowych instrukcji JavaScript. Na przykład, gdy użytkownik kliknie przycisk, możesz (1) uruchomić akcję do uruchomienia, podczas (2) resetowanie stanu wielu rozwijanych list i danych tekstowych.
4.  Odpowiedź: W module Odpowiedzi możesz dostosować warunki, w których działanie zwraca nieprawidłowe wyniki, w sposób zapewniający dokładność działania.
5.  Zaawansowane: Udoskonalij szczegółowe ustawienia dla działań, takich jak czas odpowiedzi bazy danych, opóźnienia procesora zdarzeń itp. w zaawansowanych opcjach

### Akcje API

#### Utwórz akcję

Proces aktywacji danych za pomocą API jest podobny do procesu SQL.

1.  Po pierwsze, użytkownik musi znaleźć listę akcji w lewym dolnym rogu strony edycji aplikacji i kliknąć znak plusa w prawym górnym rogu listy działań.
2.  Następnie wybierz odpowiednią listę źródeł danych w okienku wyboru zasobów, które wyskakują. Utworzony zasób można znaleźć na odpowiedniej liście źródeł danych lub możesz utworzyć tutaj nowy zasób. Wybierz utworzony zasób, kliknij "Utwórz", a akcja API zostanie utworzona pomyślnie.
3.  Pomyślnie utworzona akcja zostanie wyświetlona na liście Akcji w dolnym panelu, a zakładka Ogólna dla tej akcji pojawi się pod ramką po kliknięciu. Możesz złożyć żądanie do dowolnego HTTP API w ILLA . Proszę zapoznać się z dokumentacją API dla metod żądania danych.
4.  Po wykonaniu czynności możesz kliknąć na przycisk "Podgląd", aby sprawdzić, czy akcja, którą napisałeś, jest prawidłowa, a panel konfiguracyjny da Ci wyniki. Po zakończeniu edycji swojej akcji możesz kliknąć Zapisz i wyjść, aby zakończyć proces. Jeśli przełączysz się na inną operację bez zapisywania, wyskakujące okno poinformuje Cię o zakończeniu operacji zapisu.

#### Podział akcji

1.  Transformer: Możesz udoskonalić logikę akcji za pomocą instrukcji JavaScript w transformatorze, na przykład poprzez zastosowanie funkcji filtra w JS zamiast funkcji w instrukcji działania.
2.  Obsługa zdarzeń: Obsługa zdarzeń pozwala na uruchamianie akcji, kontrolę wstawiania i innych działań w odpowiedzi na interakcję użytkownika. Możesz dodać wiele obsługi zdarzeń, aby odpowiedzieć na wydarzenie bez dodawania niestandardowych instrukcji JavaScript. Na przykład, gdy użytkownik kliknie przycisk, możesz (1) uruchomić akcję do uruchomienia, podczas (2) resetowanie stanu wielu rozwijanych list i danych tekstowych.
