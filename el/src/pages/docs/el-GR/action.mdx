## <Heading hidden>Ερώτημα</Heading>

Μετά τη δημιουργία του πόρου, θα πρέπει να εξαγάγετε τα δεδομένα στον πόρο, να επιλέξετε, να φιλτράρετε, συγχωνεύστε τα δεδομένα και στη συνέχεια να τα βάλετε στα Ένθετα σύμφωνα με τις λογικές απαιτήσεις. Αποκαλούμε αυτή τη διαδικασία επεξεργασίας δεδομένων μια Δράση. Οι ενέργειες είναι η γέφυρα μεταξύ των δεδομένων και των στοιχείων και αποτελούν ουσιαστικό μέρος της κατασκευής μιας εφαρμογής. Έχουμε διαιρέσει τις δράσεις σε δύο κατηγορίες: δράσεις και Transformers.

### Ενέργεια SQL

#### Δημιουργία ενέργειας

Η διαδικασία χρήσης SQL ενεργειών χωρίζεται στα ακόλουθα βήματα:

1.  Πρώτα, πρέπει να βρείτε τη λίστα ενεργειών στον επεξεργαστή εφαρμογών και να κάνετε κλικ στο κουμπί "Προσθήκη".
2.  Στη συνέχεια, επιλέξτε έναν τύπο πηγής δεδομένων στο πλαίσιο επιλογής πόρου. Μπορείτε να βρείτε τους πόρους που δημιουργήθηκαν στην αντίστοιχη λίστα πόρων, μπορείτε επίσης να δημιουργήσετε νέους πόρους εδώ. Επιλέξτε τον δημιουργημένο πόρο και κάντε κλικ στο κουμπί "Επιβεβαίωση". Μια ενέργεια SQL θα δημιουργηθεί με επιτυχία.
3.  Η ενέργεια που δημιουργείται θα εμφανιστεί στη λίστα ενεργειών. Ένας επεξεργαστής ενεργειών θα εμφανιστεί κάτω από το πλαίσιο αφού κλωτσήσει την ενέργεια. Μπορείτε να αλλάξετε τον πόρο των ενεργειών, να γράψετε δηλώσεις ενέργειας, περιεχόμενο απάντησης, να προσθέσετε μετασχηματιστές, χειριστές γεγονότων.
4.  Αφού συμπληρώσετε τη λίστα ενεργειών, μπορείτε να κάνετε κλικ στο κουμπί "Προεπισκόπηση" για να ελέγξετε αν η ενέργεια που γράψατε είναι έγκυρη. Ο πίνακας ρυθμίσεων θα εμφανίζει το αποτέλεσμα της ενέργειας. Μετά την επεξεργασία της ενέργειας, μπορείτε να κάνετε κλικ στο κουμπί "Αποθήκευση" για να τερματίσετε τη διαδικασία. Αν κάνετε εναλλαγή σε άλλη ενέργεια χωρίς αποθήκευση των πληροφοριών, το σύστημα θα σας ζητήσει να αποθηκεύσετε την τροποποίηση.

#### Κατανομή Ενεργειών

1.  Ενέργεια: Μπορείτε να εισάγετε τις δηλώσεις ενεργειών σας στο πρόσθετο Ενέργειας, όπου έχουμε προσθέσει τον επεξεργαστή Κώδικα Mirror για να ενεργοποιήσετε την επισήμανση ενέργειας και την ολοκλήρωση κώδικα. Σε επόμενες ενημερώσεις, θα παράσχουμε έναν τρόπο εισαγωγής GUI για τη σύνταξη δηλώσεων δράσης, έτσι ώστε οι χρήστες που δεν είναι εξοικειωμένοι με τις δηλώσεις δράσης να μπορούν επίσης να ολοκληρώσουν τις ενέργειες δεδομένων.
2.  Μετασχηματιστής: Μπορείτε να βελτιώσετε τη λογική δράσης με δηλώσεις JavaScript στον μετασχηματιστή, για παράδειγμα, χρησιμοποιώντας τη λειτουργία φίλτρου στο JS αντί της λειτουργίας όπου βρίσκεται η εντολή δράσης.
3.  Χειριστής συμβάντων: Οι χειριστές συμβάντων σας επιτρέπουν να ενεργοποιείτε ενέργειες, να ελέγχετε ένθετα και άλλες ενέργειες ως απάντηση στην αλληλεπίδραση του χρήστη. Μπορείτε να προσθέσετε πολλαπλούς χειριστές συμβάντων για να απαντήσετε σε ένα συμβάν χωρίς να προσθέσετε προσαρμοσμένες εντολές JavaScript. Για παράδειγμα, όταν ένας χρήστης κάνει κλικ σε ένα κουμπί, μπορείτε (1) να ενεργοποιήσετε μια ενέργεια για εκτέλεση, ενώ (2) επαναφορά της κατάστασης των πολλαπλών αναπτυσσόμενων λιστών και των εισροών κειμένου.
4.  Απάντηση: Στη μονάδα απόκρισης, μπορείτε να προσαρμόσετε τις συνθήκες υπό τις οποίες η ενέργεια επιστρέφει εσφαλμένα αποτελέσματα, με τέτοιο τρόπο ώστε να εξασφαλίζεται η ακρίβεια της δράσης.
5.  Για προχωρημένους: Εκλεπτυσμένες λεπτομερείς ρυθμίσεις για ενέργειες όπως χρόνος απόκρισης της βάσης δεδομένων, καθυστερήσεις επεξεργαστή συμβάντων κλπ. εντός των προηγμένων επιλογών.

### Ενέργειες API

#### Δημιουργία ενέργειας

Η διαδικασία για την ενεργοποίηση δεδομένων χρησιμοποιώντας το API είναι παρόμοια με αυτή για SQL.

1.  Πρώτον, ο χρήστης πρέπει να βρει τη λίστα ενεργειών στο κάτω αριστερά μέρος της σελίδας επεξεργασίας της εφαρμογής και κάντε κλικ στο σύμβολο συν στο πάνω δεξιά μέρος της λίστας ενεργειών.
2.  Στη συνέχεια, επιλέξτε την αντίστοιχη λίστα προέλευσης δεδομένων στο πλαίσιο επιλογής πόρων που εμφανίζεται επάνω. Μπορείτε να βρείτε τον δημιουργημένο πόρο στην αντίστοιχη λίστα πηγής δεδομένων, ή μπορείτε να δημιουργήσετε έναν νέο πόρο εδώ. Επιλέξτε τον δημιουργημένο πόρο, κάντε κλικ στο "Δημιουργία" και μια ενέργεια API θα δημιουργηθεί με επιτυχία.
3.  Η ενέργεια που δημιουργήθηκε με επιτυχία θα εμφανιστεί στη λίστα ενεργειών στο κάτω μέρος του πίνακα και η καρτέλα Γενικά για την ενέργεια θα εμφανιστεί κάτω από το πλαίσιο όταν πατηθεί. Μπορείτε να υποβάλετε αίτημα σε οποιοδήποτε HTTP API στο ILLA . Παρακαλούμε δείτε την τεκμηρίωση APIs για τις μεθόδους αίτησης δεδομένων.
4.  Μόλις συμπληρώσετε τη δράση σας, μπορείτε να κάνετε κλικ στο κουμπί "Προεπισκόπηση" για να δείτε αν η ενέργεια που έχετε γράψει είναι έγκυρη και ο πίνακας ρυθμίσεων θα σας δώσει τα αποτελέσματα. Μόλις ολοκληρώσετε την επεξεργασία της ενέργειας σας, μπορείτε να κάνετε κλικ στο κουμπί Αποθήκευση και έξοδο για να τερματίσετε τη διαδικασία. Αν κάνετε εναλλαγή σε άλλη λειτουργία χωρίς αποθήκευση, ένα αναδυόμενο παράθυρο θα σας ζητήσει να ολοκληρώσετε τη λειτουργία αποθήκευσης.

#### Κατανομή Ενεργειών

1.  Μετασχηματιστής: Μπορείτε να βελτιώσετε τη λογική δράσης με δηλώσεις JavaScript σε έναν μετασχηματιστή, για παράδειγμα, χρησιμοποιώντας τη λειτουργία φίλτρου στο JS αντί για τη λειτουργία όπου βρίσκεται μια εντολή δράσης.
2.  Χειριστές συμβάντων: Οι χειριστές συμβάντων σας επιτρέπουν να ενεργοποιείτε ενέργειες, να ελέγχετε ένθετα και άλλες ενέργειες ως απάντηση στην αλληλεπίδραση του χρήστη. Μπορείτε να προσθέσετε πολλαπλούς χειριστές συμβάντων για να απαντήσετε σε ένα συμβάν χωρίς να προσθέσετε προσαρμοσμένες εντολές JavaScript. Για παράδειγμα, όταν ένας χρήστης κάνει κλικ σε ένα κουμπί, μπορείτε (1) να ενεργοποιήσετε μια ενέργεια για εκτέλεση, ενώ (2) επαναφορά της κατάστασης των πολλαπλών αναπτυσσόμενων λιστών και των εισροών κειμένου.
