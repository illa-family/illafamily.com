**Du musst Docker haben, bevor du den ILLA CLI nutzen kannst, um eine selbst gehostete Installation bereitzustellen. Wenn du ILLA CLI verwendest, solltest du den aktuellen Benutzer zur Docker-Gruppe hinzufügen oder in Root verwenden.**

## Lade ILLA CLI herunter

Wenn Sie Rust installiert haben, können Sie ganz einfach die neueste ILLA CLI Version mit **cargo**:

```
> cargo install illa
```

Das **Cargo** Tool wird das Herunterladen des ILLA CLI mit seinen Quell-Abhängigkeiten beenden bauen und installieren Sie es in den Frachtschrank-Pfad, so dass wir ihn ausführen können.
Nach der Installation können Sie das ILLA CLI mit dem **illa** Befehl ausführen.

Alternativ können Sie auch den ILLA CLI für Ihr Betriebssystem hier herunterladen:

-   [Windows](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-win.zip)
-   [Linux](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-linux.tar.gz)
-   [Mac](https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-macos.tar.gz)

Wenn Sie von den obigen Links herunterladen, sind die Schritte etwas anders als das Herunterladen des ILLA CLI von **cargo**. Nach dem Download des CLI müssen Sie ihn ausführbar machen. Nehmen wir das illa-linux zum Beispiel:

```
## lade die illa cli binäre Datei
> wget https://github.com/illacloud/illa/releases/latest/download/illa-x86_64-linux.tar. z

## entpacken Sie die Datei
> tar -zxvf illa-x86_64-linux.tar. z

> cd illa-x86_64-linux

> chmod +x illa

## führen Sie den illa cli
> ./illa
 aus
```

### Überprüfung der Voraussetzungen für selbstgehostete Installation

**Diese Ports 5432, 9999 und 8000 sollten für den ILLA Builder verbleiben**

Der folgende Befehl prüft die Voraussetzungen für die selbstgehostete Installation auf Ihrem Betriebssystem:

```
> illa doctor
```

### ILLA-Builder wird eingesetzt

Sobald der ILLA CLI installiert ist, kannst du eine selbst gehostete oder [Cloud-Installation](https://cloud.illacloud.com/).

Schnellstart

```
> illa Deploy --self --port=10000
```

Sie können einfach einen selbst gehosteten ILA Builder bereitstellen, indem Sie den folgenden Befehl ausführen:

```
# der Port, auf den der ILLA Builder zugegriffen werden kann, kann geändert werden
# der <MOUNT_PATH> ist der benutzerdefinierte Mount-Pfad des ILLA Builders, der Standardwert ist ein Pfad im Home-Verzeichnis des Benutzers, wenn nicht ausgefüllt
> illa Deploy --self --port=10000 --mount=<MOUNT_PATH>
```

Jetzt können Sie auf den ILLA Builder zugreifen: http://localhost:10000

## Aktualisiere ILLA Builder

Um den ILLA Builder mit den neuesten Docker-Bildern zu aktualisieren, verwenden Sie folgenden Befehl:

```
> illa update --self
```

## Andere Operationen

```
# den ILLA Builder auflisten
> illa list --self

# den ILLA Builder stoppen
> illa stop --self

# den ILLA Builder entfernen
> illa remove --self

# erzwungene Entfernung des ILLA Builders und persistente Daten
> sudo illa remove --force --self --data

# den ILLA Builder neustarten
> illa Neustart --self

# Hilfeinformationen
> illa Hilfe 
 > illa
```
