## <Heading hidden>Abfrage</Heading>

Nach dem Erstellen der Ressource müssen Sie die Daten in der Ressource extrahieren, auswählen, filtern, die Daten zusammenführen und dann entsprechend den logischen Anforderungen in die Beilagen einfügen. Wir nennen diesen Prozess der Datenverarbeitung eine Aktion. Aktionen sind die Brücke zwischen Daten und Komponenten und sind ein wesentlicher Bestandteil des Aufbaus einer Anwendung. Wir haben Aktionen in zwei Kategorien unterteilt: Aktionen und Transformatoren.

### SQL-Aktion

#### Aktion erstellen

Der Prozess der Verwendung von SQL-Aktionen ist in die folgenden Schritte unterteilt:

1.  Zuerst müssen Sie die Aktionsliste im App-Editor finden und auf die Schaltfläche "Hinzufügen" klicken.
2.  Wählen Sie dann einen Datentyp in der Ressourcen-Auswahlbox aus. Sie können die erzeugten Ressourcen in der zugehörigen Ressourcenliste finden. Hier können Sie auch neue Ressourcen anlegen. Wählen Sie das erstellte Werk aus und klicken Sie auf "Bestätigen". Eine SQL-Aktion wird erfolgreich erstellt.
3.  Die erstellte Aktion wird in der Aktionsliste angezeigt. Nach dem Auswerfen der Aktion wird ein Aktionseditor unterhalb des Rahmens angezeigt. Sie können die Ressource von Aktionen, Beschreibungen von Aktionen, Antwortinhalt, Transformatoren, Eventhandler ändern.
4.  Nachdem Sie die Aktionsliste ausgefüllt haben, können Sie auf die Schaltfläche "Vorschau" klicken, um zu überprüfen, ob die von Ihnen geschriebene Aktion gültig ist. Das Konfigurationsfenster zeigt das Ergebnis der Aktion an. Nachdem die Aktion bearbeitet wurde, können Sie auf "Speichern" klicken, um den Prozess zu beenden. Wenn Sie zu einer anderen Aktion wechseln, ohne die Informationen zu speichern, wird das System Sie auffordern, die Änderung zu speichern.

#### Aktionsaufteilung

1.  Aktion: Sie können Ihre Aktionserklärungen in das Aktionsmodul eingeben, wo wir den Code Mirror-Editor hinzugefügt haben, um das Hervorheben und Vervollständigen von Code zu aktivieren. In folgenden Updates, werden wir einen GUI-Eingabemodus für das Schreiben von Aktionserklärungen bereitstellen, so dass Benutzer, die mit Aktionserklärungen nicht vertraut sind, auch Datenaktionen abschließen können.
2.  Transformer: Sie können die Handlungslogik mit JavaScript-Anweisungen im Transformator verfeinern. z.B. durch die Verwendung der Filterfunktion in JS anstelle der Funktion Wo in der Action-Anweisung.
3.  Event-Handler: Eventhandler erlauben es Aktionen, Einfügeoperationen und andere Aktionen als Reaktion auf die Benutzerinteraktion zu steuern. Sie können mehrere Eventhandler hinzufügen, um auf ein Ereignis zu reagieren, ohne benutzerdefinierte JavaScript-Anweisungen hinzuzufügen. Zum Beispiel, wenn ein Benutzer auf eine Schaltfläche klickt, können Sie (1) eine Aktion auslösen, die ausgeführt werden soll, während (2) den Status von mehreren Dropdown-Listen und Texteingaben zurücksetzen.
4.  Antwort: Im Antwortmodul können Sie die Bedingungen anpassen, unter denen die Aktion falsche Ergebnisse liefert, so dass die Richtigkeit der Handlung gewährleistet ist.
5.  Erweitert: Detaillierte Einstellungen für Aktionen wie Datenbank-Timeout-Antwortzeiten, Eventprozessorverzögerungen usw. innerhalb der erweiterten Optionen verfeinern.

### API-Aktionen

#### Aktion erstellen

Der Prozess zur Aktionierung von Daten mit der API ist ähnlich wie bei SQL.

1.  Erstens, der Benutzer muss die Aktionsliste unten links auf der Seite zur Bearbeitung der Anwendung finden und auf das Plus-Zeichen oben rechts in der Aktionsliste klicken.
2.  Als nächstes wählen Sie die zugehörige Datenquellen-Liste im Auswahlfeld für Ressourcen aus. Sie können die erstellte Ressource in der zugehörigen Datenquellen-Liste finden, oder Sie können hier eine neue Ressource erstellen. Wählen Sie die erstellte Ressource aus, klicken Sie auf "Erstellen" und eine API-Aktion wird erfolgreich erstellt.
3.  Die erfolgreich erstellte Aktion wird in der Aktionen-Liste im unteren Fenster angezeigt und die Registerkarte Allgemein für die Aktion wird beim Klicken unterhalb des Rahmens angezeigt. Sie können eine Anfrage an jede HTTP-API in ILLA stellen. Bitte lesen Sie die APIs Dokumentation für Datenabfragemethoden.
4.  Sobald Sie Ihre Aktion ausgefüllt haben Sie können auf die Schaltfläche "Vorschau" klicken, um zu sehen, ob die von Ihnen geschriebene Aktion gültig ist und das Konfigurationsfenster Ihnen die Ergebnisse liefert. Sobald Sie Ihre Aktion bearbeitet haben, können Sie auf Speichern klicken und beenden, um den Prozess zu beenden. Wenn Sie zu einer anderen Operation wechseln ohne zu speichern, wird ein Popup Sie zum Abschluss des Speicherns auffordern.

#### Aktionsaufteilung

1.  Transformer: Sie können die Handlungslogik mit JavaScript-Anweisungen in einem Transformator verfeinern, z.B. durch die Verwendung der Filterfunktion in JS statt der Funktion Wo in einer Aktionsanweisung.
2.  Ereignis-Handler: Ereignishandler erlauben es Aktionen, Einfügeoperationen und andere Aktionen als Reaktion auf die Benutzerinteraktion zu steuern. Sie können mehrere Eventhandler hinzufügen, um auf ein Ereignis zu reagieren, ohne benutzerdefinierte JavaScript-Anweisungen hinzuzufügen. Zum Beispiel, wenn ein Benutzer auf eine Schaltfläche klickt, können Sie (1) eine Aktion auslösen, die ausgeführt werden soll, während (2) den Status von mehreren Dropdown-Listen und Texteingaben zurücksetzen.
