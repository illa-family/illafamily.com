## <Heading hidden>Hugging Gesichts-API</Heading>

Hugging Face ist der Github der Maschinenlern-Community mit Hunderttausenden vorausgebildeten Modellen und 10.000 Datensätzen. Sie können frei auf Modelle und Datensätze anderer Branchenexperten zugreifen oder Ihre Modelle auf Hugging Face.

Einige Beispiele für Modelle, die in der Hugging Face Bibliothek verfügbar sind, sind:

1.  BERT (Bidirectional Encoder Representations from Transformers): BERT ist ein Transformer-basiertes Modell, das Google für verschiedene NLP-Aufgaben entwickelt hat. Sie hat im Bereich des Sprachverständnisses und der maschinellen Übersetzung hochmoderne Ergebnisse erzielt.
2.  GPT (Generativer Pre-Training Transformer): GPT ist ein weiteres Transformer-basiertes Modell, das von OpenAI entwickelt wurde. Es wird in erster Linie für die Erstellung von Sprachen wie Übersetzung und Zusammenfassung von Texten verwendet.
3.  RoBERTa (Robustly Optimized BERT): RoBERTa ist eine Erweiterung des BERT-Modells, das entwickelt wurde, um die Leistung von BERT bei verschiedenen NLP-Aufgaben zu verbessern.
4.  XLNet (eXtraordinary LanguageNet): XLNet ist ein Transformer-basiertes Modell, das von Google entwickelt wurde, um die Leistung von Transformatormodellen beim Sprachverstehen zu verbessern.
5.  ALBERT (A Lite BERT): ALBERT ist eine Version des BERT-Modells, das entwickelt wurde, um effizienter und schneller trainieren zu können und dabei eine gute Performance bei NLP-Aufgaben zu erhalten.

### Was du mit Hugging Face im ILLA Builder machen kannst

In Hugging Face, sind über 130.000 Modelle mit maschinenlernendem Lernen über die öffentliche API verfügbar, die du unter \[**[https://huggingface.co/models](https://huggingingface.co/models)](<https://huggingingface.co/models>**. Wenn du eine Lösung für Produktionsszenarien benötigst, kannst du außerdem die Inferenz-Endpunkte von Hugging Face verwenden, die unter \[**[https://huggingface.co/docs/inference-endpoints/index](https://huggingingingface.co/docs/inference-endpoints/index)]([https://huggingingingingface.co/docs/inference-endpoints/index](https://huggingingingface.co/docs/inference-endpoints/index)**.

ILLA Builder bietet Dutzende von gängigen Frontend-Komponenten an, die es Ihnen ermöglichen, verschiedene Frontend-Schnittstellen auf Basis Ihrer spezifischen Bedürfnisse schnell zu erstellen. Gleichzeitig bietet ILLA eine Verbindung zu Hugging Face, die es Ihnen ermöglicht, sich schnell mit der API zu verbinden, Anfragen zu senden und zurückgegebene Daten zu erhalten. Durch das Verbinden der API und der Front-End-Komponenten können Sie die Anforderung implementieren, dass Benutzer Inhalte über das Frontend eingeben und an die API senden können. Die API gibt den generierten Inhalt zurück, der im Frontend angezeigt wird.

### Hugging Face-API-Ressource konfigurieren

| Eigenschaften | Benötigt | Beschreibung                                                                                                |
| ------------- | -------- | ----------------------------------------------------------------------------------------------------------- |
| Name          | benötigt | Legen Sie einen Ressourcennamen fest, der für die Anzeige in der ILLA verwendet wird                        |
| Token         | benötigt | Der Benutzerzugriff oder API-Token. Du kannst es unter https://huggingingface.co/settings/tokens bekommen.  |

### Aktion konfigurieren

| Eigenschaften | Benötigt | Beschreibung                                                                                                                                                                                                                                                    |
| ------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Modell-ID     | benötigt | Modelle suchen: https://huggingingface.co/models                                                                                                                                                                                                                |
| Parametertyp  | benötigt | Der Parametertyp des Endpunkts. Wenn zum Beispiel der Endpunkt eine Texteingabe benötigt, geben Sie den Parameter „Eingabe“ mit Text ein. Wenn Ihr Endpunkt eine JSON-Eingabe benötigt, geben Sie den Parameter „Eingabetaste“ mit JSON oder Schlüsselwert ein. |
| Parameter     | benötigt | Geben Sie Ihren Parameter ein. Verwenden Sie {{ componentName.value }} , um Daten von Komponenten zu verwenden.                                                                                                                                                 |

# Verwendung von Hugging Face im ILLA Builder

### Schritt 1: Build UI mit Komponenten auf ILLA Builder

Basierend auf dem erwarteten Einsatzszenario, das Sie beschrieben haben, erstellen Sie eine Front-End-Schnittstelle. Zum Beispiel können Sie Eingabe- und Bildkomponenten verwenden, wenn Ihr Produkt Text nimmt und ein Bild ausgibt. Wenn Ihr Produkt Text und erzeugte Ausgabe annimmt, können Sie Eingabe- und Textkomponenten verwenden.

Das folgende Bild ist ein Beispiel für eine Front-End-Seite für ein Produkt, das Fragen basierend auf Kontext beantwortet.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi1.png' alt="" />

### Schritt 2: Erstelle eine Hugging Face Ressource und konfiguriere die Aktionen

Klicken Sie + Neu in der Aktionsliste und wählen Sie Hugging Face Inference API.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi2.png' alt="" />

Füllen Sie das Formular aus, um sich mit Ihrem Hugging Gesicht zu verbinden:

Name: Der Name, der in ILLA angezeigt wird

Token: Gehe in dein Hugging Gesicht [Profileinstellungen]([https://huggingingface.co/settings/tokens](https://huggingingface.co/settings/tokens))

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi3.png' alt="" />

Bestätigen Sie die Modellinformationen in Hugging Face bevor Sie die Aktionen konfigurieren:

Wähle ein Modell in [Hugging Face Model Page]([https://huggingface.co/models](https://huggingingface.co/models))

Lass uns [deepset/roberta-base-squad2]([https://huggingingface.co/deepset/roberta-base-squad2](https://huggingingface.co/deepset/roberta-base-squad2)) als Beispiel nutzen. Geben Sie die Detailseite ein > Deploy > Click Inference API

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi4.png' alt="" />

Die Parameter nach \`“inputs” sind das, was du ILLA ausfüllen solltest. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi5.png' alt="" />

Im ILLA Builder sollten wir die Modell-ID und Parameter ausfüllen. Wenn man das obige Modell als Beispiel nimmt, ist das \`“inputs”-Paar ein Schlüssel-Wert-Paar, so dass wir es mit Schlüssel-Wert oder JSON füllen können. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi6.png' alt="" />

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi7.png' alt="" />

Und wir unterstützen auch die Texteingabe und Binäreingabe, die alle Hugging Face Inference API-Verbindungen erfüllen können.

### Schritt 3: Aktionen mit Komponenten verbinden

Um die Benutzereingabe an die API zu übergeben, können Sie {{ verwenden, um die in der Komponente eingegebenen Daten abzurufen. Zum Beispiel ist input2 die Komponente, um die Frage einzugeben und input1 ist die Komponente zum Input-Kontext, wir können `question` und `context` im Schlüssel ausfüllen und `{{ input.value }}` im Wert ausfüllen:

```jsx
{
"Frage": {{input2.value}},
"context": {{input1.value}}
}
```

Bevor die Ausgabedaten der Aktion in der Front-End-Komponente angezeigt werden, sollten wir bestätigen, in welchem Feld die Ausgabe verschiedener Modelle platziert ist. Nehmen Sie immer noch `deepset/roberta-base-squad2` als Beispiel, die Ergebnisse sind wie folgt:

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi8.png' alt="" />

Also können wir die Antwort mit `{{ textQuestion.data[0].answer }}`(Die `textQuestion` ist der Name der Aktion). 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi9.png' alt="" />
