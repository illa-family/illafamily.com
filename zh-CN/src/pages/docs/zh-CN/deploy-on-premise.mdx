# <Heading hidden>ILLA的私有化部署</Heading>

自托管ILLA 意味着在您自己的服务器或机器上托管和管理ILLA 平台，而不是使用第三方服务提供商。 这种方法常常被认为更加安全和私密，因为您对您的数据及其使用拥有完全的控制权。

ILLA将提供云托管版本，并提供在各种平台上设置实例的选项。

## 自主机

### 安全

通过在您自己的基础设施上部署ILLA，您可以将数据保存在本地，并确保您的API密钥或安全凭证永远不会离开您的服务器。这允许您对ILLA应用与您服务器上任何其他应用程序相同的安全规则。您使用ILLA构建的应用程序将托管在您的基础设施和公司网络内。您还可以选择在防火墙之后的内部网络上托管ILLA，并将其配置使用您自定义的域和安全策略。 这允许您对ILLA应用相同的安全规则，您可以在服务器上的任何其他应用上应用相同的安全规则。 您使用 ILLA 构建的应用程序将保存在您的基础设施和公司网络中。 您也可以选择在防火墙后面的内部网络上托管ILLA，并配置它来使用您的自定义域名和安全政策。

### 内部数据库

通过自动托管ILLA，您可以连接到一个在同一机器或内联网上运行的数据库，并使用它来构建应用程序。 ILLA 使用 MongoDB 存储应用数据，Redis 存储会话信息和非关键缓存。 通过私有化部署ILLA，您可以连接到在同一台机器或内网上运行的数据库，并使用它来构建应用程序。ILLA使用MongoDB存储应用程序数据，使用Redis存储会话信息和非关键缓存。如果您想将ILLA与部署在您基础设施上的自定义MongoDB或Redis服务器集成，您将需要修改环境变量的配置并将它们指向您自定义的MongoDB/Redis实例。
