## <Heading hidden>查询</Heading>

在创建资源后，您需要提取资源中的数据，选择、过滤。 合并数据，然后按照逻辑要求将其放入插入插件。 我们将这一数据处理过程称为行动。 行动是数据和组件之间的桥梁，是构建应用程序的一个基本部分。 我们将行动分为两类：行动和改革。

### SQL查询

#### 创建动作

使用SQL查询的过程主要分为以下几步：

1.  首先，用户需要在应用编辑页面的左下方找到Action列表，在Action列表的右上方点击加号。
2.  然后在资源选择框中选择一个数据源类型。 您可以在相应的资源列表中找到创建的资源，您也可以在这里创建新的资源。 选择创建的资源，然后点击“确认”按钮。 SQL 操作将成功创建。
3.  创建的操作将显示在操作列表中。 踢出动作后，动作编辑器将显示在框架下。 您可以更改动作、写动作语句、响应内容、添加变压器、事件处理器。
4.  填写操作列表后，您可以点击“预览”按钮来检查您写的操作是否有效。 配置面板将显示动作结果。 编辑操作后，您可以点击“保存”按钮结束操作。 如果您切换到另一个动作而不保存信息，系统会提示您保存修改。

#### 动作分解

1.  动作：您可以在我们添加代码镜像编辑器的行动模块中输入您的动作语句以启用高亮和代码补全。 在随后的更新中， 我们将提供写动作语句的GUI输入模式，以便不熟悉动作语句的用户也可以完成数据操作。
2.  转换器（Transformer）：您可以在转换器中用JavaScript语句完善查询逻辑，例如在JS中使用filter函数代替查询语句中的where函数。
3.  事件处理程序：事件处理程序允许您触发动作，控制插入和其他动作响应用户交互。 事件处理器（event handler）：事件处理器允许您触发查询、控制组件等行为以响应用户交互。您可以在不添加自定义JavaScript语句的前提下添加多个事件处理程序以响应一个事件。例如，当用户单击一个按钮时，您可以(1)触发一个查询来运行，同时(2)重置多个下拉列表和文本输入的状态。 例如，当用户点击按钮时，您可以(1) 触发一个动作来运行。 当(2) 重置多个下拉列表和文本输入的状态。
4.  响应（Response）：在响应模块中，您可以自定义查询返回结果错误的条件，通过这样的方式保证查询的准确性。
5.  高级（Advanced）：在高级选项内完善查询的详细设置，例如数据库的超时响应时间、事件处理器延迟等。

### API查询

#### 创建动作

使用API数据查询的过程与SQL大致相同：

1.  首先，用户需要在应用编辑页面的左下方找到Action列表，在Action列表的右上方点击加号。
2.  接下来，在资源选择框中选择相应的数据源列表。 您可以在相应的数据源列表中找到创建的资源，或者您可以在这里创建一个新的资源。 选择创建的资源，点击“创建”并成功创建 API 动作。
3.  成功创建的动作将会显示在底部面板中的动作列表中，当点击时，动作会在框架下拉亮。 您可以在 ILLA 中向任何 HTTP API发出请求。 请查看 API 文档以获取数据请求方法。
4.  一旦您填写了您的动作， 您可以点击“预览”按钮来查看您所写的动作是否有效，配置面板将给您提供结果。 一旦完成编辑您的操作，您可以单击保存并退出以结束进程。 如果您切换到另一个操作而不保存，弹出窗口会促使您完成保存操作。

1.  转换器（Transformer）：您可以在转换器中用JavaScript语句完善查询逻辑，例如在JS中使用filter函数代替查询语句中的where函数。
