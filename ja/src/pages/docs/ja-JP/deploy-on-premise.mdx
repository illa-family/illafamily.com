# <Heading hidden>オンプレミスのデプロイ</Heading>

セルフホスティングILLAとは、サードパーティのサービスプロバイダを使用する代わりに、独自のサーバーまたはマシン上でILLAプラットフォームをホスティングおよび管理することを意味します。 データとその使用を完全に制御できるため、このアプローチはしばしば安全でプライベートなものとみなされます。

ILLAは、クラウド型のバージョンと、さまざまなプラットフォームでインスタンスをセットアップするオプションをすぐに提供します。

## セルフホスト

独自のインフラストラクチャ上でセルフホスティングILLAを使用すると、別の構成を維持する必要なく、既存のセットアップとシームレスに統合できます。 これにより、既存のインフラストラクチャの機能と機能を使用し、ILLAまで拡張することができます。 ILLAをインフラストラクチャに導入することの利点は以下のとおりです。

### セキュリティ

独自のインフラストラクチャにILLAをホストすることで データをオンプレミスに保持し、API キーやセキュリティ認証情報がサーバーから離れないようにすることができます。 これにより、サーバー上の他のアプリに適用する ILLA にも同じセキュリティルールを適用できます。 ILLAを使用してビルドするアプリケーションは、お客様のインフラストラクチャおよび企業ネットワーク上でホストされます。 また、ファイアウォールの背後にある内部ネットワークで ILLA をホストし、カスタムドメインおよびセキュリティポリシーを使用するように設定することもできます。

### 内部データベース

ILLAをセルフホスティングすることで、同じマシンまたはイントラネット上で実行されているデータベースに接続し、アプリを構築するためにそれを使用することができます。 ILLAはMongoDBを使用してアプリデータを保存し、Redisはセッション情報や重要でないキャッシュを保存します。 インフラ上でホストされているカスタムMongoDBまたはRedisサーバーとILLAを統合したい場合。 環境変数の設定を変更し、カスタムMongoDB/Redis インスタンスを指定する必要があります。

### 更新

ILLAを自己ホスティングすることで、より柔軟な更新スケジュールやその他のメンテナンス活動を構成できます。 提供された手順に従ってメンテナンスウィンドウを設定し、自動更新をスケジュールすることができます。 これにより、お客様と組織にとって便利な方法で ILLA インスタンスを維持および更新することができます。
