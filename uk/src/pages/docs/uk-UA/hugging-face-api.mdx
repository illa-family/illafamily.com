## <Heading hidden>Гортаючий API</Heading>

Підключення до Github цієї спільноти машинного навчання з сотні тисяч дотренованих моделей і 10 000 знайомств на даний момент доступні в даний час. Ви можете вільно отримати доступ до моделей та даних спільно з іншими фахівцями з галузі або хостом і розгорнути свої моделі на Hugging Face.

Деякі приклади моделей, доступних у бібліотеці Hugging Face включають:

1.  BERT (Біблійні представники енкодеру від Transformers): BERT є моделлю, розробленою Google для різних задач NLP. Він досяг найсучасніших результатів в розумінні мови та завданнях машинного перекладу.
2.  GPT (Генеративна переднавчання передачі): GPT це інша модель, розроблена OpenAI. Для створення мов використовується головним чином для виконання завдань, таких як переклад і підсумок тексту.
3.  RoBERTa (зручно оптимізований БЕРТ): RoBERTa є розширенням БЕРТ моделі, яка була розроблена для покращення роботи BERT на різних завданнях NLP.
4.  XLNet (eXtraordinary LanguageNet): XLNet є моделлю, розробленою компанією Google, яка розроблена для вдосконалення швидкодії трансформаторських моделей на завданнях з розуміння мови.
5.  ALBERT (A Lite BERT): ALBERT є версією виробленої BERT моделі, яка була більш ефективною та швидкою для тренування, зберігаючи хорошу продуктивність на завданнях NLP.

### Те, що ви можете зробити з обіймом обличчя в конструкторі ILLA

У Hugging Face, більш ніж 130 000 моделей механічного навчання доступні через публічний API, який ви можете використовувати і тестувати безкоштовно \[<https://huggingface.co/models>](<https://huggingface.co/models>**. Крім того, якщо вам потрібне рішення для виробничих сценаріїв, ви можете використовувати кінцеві точки Фашинга, які можна розгорнути та отримати за адресою \[**<https://huggingface.co/docs/inference-endpoints/index>]([https://huggingface.co/docs/inference-points/index](https://huggingface.co/docs/huggingface.co/docs/inference-endpoints/index)\*\*.

ILLA Builder надає десятки часто використовуваних фронт-компонентів, що дозволяє вам будувати різні інтерфейси переднього кінця на основі ваших конкретних потреб. У той же час, ILLA пропонує підключення до Hugging Face, що дозволяє швидко підключитися до API, відправити запити і отримати повернуті дані. З'єднуючи API та передні компоненти, ви можете реалізувати вимогу, яку користувачі можуть ввести через фронт-енд і надати його API. API повертає згенерований вміст, який буде відображатися в першій частині.

### Налаштування API ресурсів Hugging

| Властивості | Обов'язково | Опис                                                                                                  |
| ----------- | ----------- | ----------------------------------------------------------------------------------------------------- |
| Ім'я        | обов’язково | Визначте назву ресурсу, яка буде використовуватися для відображення в ILLA                            |
| Маркер      | обов’язково | Доступ користувача або API маркер. Ви можете отримати його в https://huggingface.co/settings/tokens.  |

### Налаштувати дію

| Властивості   | Обов'язково | Опис                                                                                                                                                                                                                                                 |
| ------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID моделі     | обов’язково | Пошук моделей: https://huggingface.co/models                                                                                                                                                                                                         |
| Тип параметра | обов’язково | Тип параметра вашої кінцевої точки. Наприклад, якщо ваша кінцева точка потребує введення тексту, виберіть параметр "input" з текстом. Якщо ваша кінцева точка потребує введення JSON, оберіть заповніть параметр "input" з JSON або значенням ключа. |
| Параметр      | обов’язково | Введіть ваш параметр. Використовувати дані компонентів з {{ componentName.value }}.                                                                                                                                                                  |

# Як використовувати обличчя Hugging в ILLA Builder

### Крок 1: Побудуйте інтерфейс з компонентами на конструкторі ILLA

На основі очікуваного використання ви описали, побудувати інтерфейс переднього кінця. Наприклад, ви можете використовувати компоненти вводу і зображень, якщо ваш продукт приймає в текст і виводить зображення. Якщо ваш товар бере на себе текст і виводиться текст, ви можете використовувати вхідні і текстові компоненти.

Наступне зображення є прикладом головної сторінки товару для відповідей, що базується на контексті.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi1.png' alt="" />

### Крок 2: Створіть горючий ресурс і налаштуйте дії

Натисніть + Нову з переліку дій та виберіть API.

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi2.png' alt="" />

Заповніть форму для підключення до вашого Hugging Face:

Ім'я, яке буде відображатися в ILLA

Токен: Завантажте в ваш HuggingFace [налаштування профілю]([https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens))

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi3.png' alt="" />

Підтвердіть інформацію про модель в Hugging Face перед налаштуванням дій:

Виберіть модель в [Hugging Face Model Page]([https://huggingface.co/models](https://huggingface.co/models))

Скористаймося [deepset/roberta-base-squad2]([https://huggingface.co/deepset/roberta-base-squad2](https://huggingface.co/deepset/roberta-base-squad2)). Введіть детальну сторінку > клікніть Розгортання > Натисніть кнопку API

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi4.png' alt="" />

Параметри після `“введення”` - це те, що ви повинні заповнити в ILLA. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi5.png' alt="" />

В ILLA Builder, ми повинні заповнити ID та параметр моделі. Візьмемо вищевказану модель як приклад, "вхідні дані" - це пара ключ-значення, щоб ми могли заповнити її ключовим значенням або JSON. 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi6.png' alt="" />

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi7.png' alt="" />

І ми також підтримуємо введення тексту і бінарний вхідний текст, який може задовольнити всі API-з'єднання, що приймають мишку.

### Крок 3: Підключити дії до компонентів

Переходити передній кінець користувача до API, ви можете використовувати {{ для отримання даних, введених в компоненті. Наприклад, вхід 2 є компонентом для вводу питання і введення 1 компонента для вхідного контексту, ми можемо заповнити `question` та `context` в ключі, і заповнити `{{ input.value }}` в значенні:

```jsx
{
"question": {{input2.value}},
"context": {{input1.value}}
}
```

Перед відображенням даних вихідної дії в зовнішньому компоненті, ми повинні підтвердити, які поля виході різних моделей розміщені. Все ще беручи до уваги "deepset/roberta-base-squad2" як приклад, результати є наступними:

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi8.png' alt="" />

Таким чином, ми можемо отримати відповідь за допомогою{{ textQuestion.data[0].answer }}`(`textQuestion\` - це назва дії). 

<img src='https://cdn.illacloud.com/official-website/img/docs/connect/hfapi9.png' alt="" />
